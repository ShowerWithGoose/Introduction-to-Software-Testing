**Problem Description**

Design and implement a syntax analysis program based on the given grammar. The program should identify the corresponding syntax components from the source program. The input, output, and processing requirements are as follows:

For a correct source program, use the recursive procedure method to analyze the syntax components defined in the grammar.
Output the information of each word line by line in the order recognized by lexical analysis.
Before completing the analysis of any syntax components in the grammar (except for <BlockItem>, <Decl>, <BType>), output the name of the current syntax component on a new line, in the format of “<Stmt>” (Note: Syntax components that are not required to be output should still be analyzed but not displayed).

For an erroneous source program, the program should identify the errors and output the line number of the error and the error category code.

**Syntax Grammar**
The symbol [...] indicates that the content inside the square brackets is optional.
The symbol {...} indicates that the content inside the curly braces can be repeated zero or more times.

CompUnit → {Decl} {FuncDef} MainFuncDef
Decl → ConstDecl | VarDecl
ConstDecl → 'const' BType ConstDef { ',' ConstDef } ';'
BType → 'int' | 'char'
ConstDef → Ident [ '[' ConstExp ']' ] '=' ConstInitVal
ConstInitVal → ConstExp | '{' [ ConstExp { ',' ConstExp } ] '}' | StringConst
VarDecl → BType VarDef { ',' VarDef } ';'
VarDef → Ident [ '[' ConstExp ']' ] | Ident [ '[' ConstExp ']' ] '=' InitVal
InitVal → Exp | '{' [ Exp { ',' Exp } ] '}' | StringConst
FuncDef → FuncType Ident '(' [FuncFParams] ')' Block
MainFuncDef → 'int' 'main' '(' ')' Block
FuncType → 'void' | 'int' | 'char'
FuncFParams → FuncFParam { ',' FuncFParam }
FuncFParam → BType Ident ['[' ']']
Block → '{' { BlockItem } '}'
BlockItem → Decl | Stmt
Stmt → LVal '=' Exp ';' | [Exp] ';' | Block | 'if' '(' Cond ')' Stmt [ 'else' Stmt ] | 'for' '(' [ForStmt] ';' [Cond] ';' [ForStmt] ')' Stmt | 'break' ';' | 'continue' ';' | 'return' [Exp] ';' | LVal '=' 'getint''('')'';' | LVal '=' 'getchar''('')'';' | 'printf''('StringConst {','Exp}')'';'
ForStmt → LVal '=' Exp
Exp → AddExp
Cond → LOrExp
LVal → Ident ['[' Exp ']']
PrimaryExp → '(' Exp ')' | LVal | Number | Character
Number → IntConst
Character → CharConst
UnaryExp → PrimaryExp | Ident '(' [FuncRParams] ')' | UnaryOp UnaryExp
UnaryOp → '+' | '−' | '!'
FuncRParams → Exp { ',' Exp }
MulExp → UnaryExp | MulExp ('*' | '/' | '%') UnaryExp
AddExp → MulExp | AddExp ('+' | '−') MulExp
RelExp → AddExp | RelExp ('<' | '>' | '<=' | '>=') AddExp
EqExp → RelExp | EqExp ('==' | '!=') RelExp
LAndExp → EqExp | LAndExp '&&' EqExp
LOrExp → LAndExp | LOrExp '||' LAndExp
ConstExp → AddExp


**Input Format**
The test program is provided in **testfile.txt**. It may be a correct source program or an erroneous source program.

**Output Format**
For correct source programs, output the word category code and the word's character/string form or the syntax component name to `parser.txt` in the order they are read.

word category code word's character/string form (separated by a single space)  
or
syntax component name

For incorrect source programs, output the line number and error category code to `error.txt`, sorted by line number in ascending order, starting from line number 1.

error line number error category code (separated by a single space)


**Special Reminder**
A terminal token needs to be output, but what is output is its lexical component. Compared to lexical analysis, syntax analysis requires the additional output of non-terminal components, which consist of multiple lexical tokens and are represented in the form of <Non-terminal>.

**Possible Errors in Syntax Analysis**

| error category code | explanation | example | 
|------|------|------|
| a | The symbols "&" and "\|" appearing individually | if(1 & 2) |
| i | Missing semicolon. The error line number corresponds to the line where the last non-terminal before the semicolon is located. | a = b + c |
| j | Missing right parenthesis ')'. The error line number corresponds to the line where the last non-terminal before the right parenthesis is located. | c = getchar(; |
| k | Missing right square bracket ']'. The error line number corresponds to the line where the last preceding non-terminal symbol appears before the right square bracket. | i = a[1;|
