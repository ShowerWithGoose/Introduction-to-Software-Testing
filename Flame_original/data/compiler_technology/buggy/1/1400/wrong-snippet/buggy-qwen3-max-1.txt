Based on the problem description and code review, the main issue appears to be in the `parseCompUnit()` method in Parser.java. The current implementation has a logic error in how it processes declarations and function definitions - it only adds the last parsed node to the children list instead of adding all parsed nodes. This would cause incorrect AST construction and missing output for valid programs.

    public Node parseCompUnit() {
        ArrayList<Node> children = new ArrayList<>();
        Node node = null;
        while (true) {
            if (curToken == null) {
                break;
            }
            /* root node*/
            if (tokens.preRead(1).getType() == TokenType.MAINTK) {
                node = parseMainFunDef();
            } else if (tokens.preRead(2).getType() == TokenType.LPARENT) {
                node = parseFuncDef();
            } else if (curToken.getType() == TokenType.CONSTTK) {
                node = parseConstDecl();
            } else if (curToken.getType() == TokenType.INTTK || curToken.getType() == TokenType.CHARTK) {
                node = parseVarDecl();
            } else {
                break;
            }
            children.add(node); // This line is missing in the original code!
        }
        return new Node(children, SyntaxType.CompUnit);
    }
