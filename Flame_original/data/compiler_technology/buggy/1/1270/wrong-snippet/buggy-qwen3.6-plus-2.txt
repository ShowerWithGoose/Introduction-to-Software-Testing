**Summary of Errors:**
1. **`Parser.java` - `lookBehind()` method**: The boundary check `if (lookbehindIndex < tokens.size())` lacks a lower bound check (`>= 0`). When `index` is `0`, `lookbehindIndex` becomes `-1`, causing an `IndexOutOfBoundsException` when calling `tokens.get(-1)`.
2. **`Parser.java` - `parseBlock()` method**: In the `else` branch (when the current token is neither a declaration nor a statement), the parser prints an error but fails to advance the token index. This results in an infinite loop when encountering unexpected tokens inside a block.
3. **`Token.java` - `tokenize()` method**: When handling single `&` or `|` characters (error type `a`), the code clears `currentToken` before creating the error token. Consequently, `currentToken.toString()` is an empty string `""` instead of `"&"` or `"|"`, leading to malformed tokens being passed to the parser.
