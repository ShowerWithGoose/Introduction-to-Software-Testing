Based on the problem description and code review, the issue appears to be in the token output handling for syntax components. The problem is that the parser isn't correctly outputting the syntax component names before analyzing their contents (except for <BlockItem>, <Decl>, <BType>), and the leaf node output format may be incorrect.

Looking at the ParserSuccessPrint class, which handles the output formatting, there's a logic error in how it traverses and outputs the syntax tree nodes.

<util/ParserSuccessPrint.java>

public class ParserSuccessPrint {
    public void run(SyntaxTreeNode root, StringBuilder sb, ParserSuccessPrint psp) {
        if(root.getName().equals("leaf")) {
            sb.append(root.getToken().getType()).append(" ").append(root.getToken().getName()).append("\n");
        }
        else {
            for(SyntaxTreeNode child: root.getChildren()) {
                psp.run(child, sb, psp);
            }
            if(!root.getName().equals("<BlockItem>") && !root.getName().equals("<Decl>") && !root.getName().equals("<BType>")) {
                sb.append(root.getName()).append("\n");
            }
        }
    }
}

The issue is that the current implementation outputs the syntax component name AFTER processing all children, but according to the problem description, it should output the syntax component name BEFORE processing its contents (except for the excluded components). This means the order of operations in the else block is incorrect.
