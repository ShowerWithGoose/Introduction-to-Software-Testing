**Error Analysis Summary:**
1. **`parseCompUnit` Loop Condition for `FuncDef`**: `while (preRead() == LexType.IDENFR)` incorrectly expects function definitions to start with an identifier. According to the grammar, `FuncDef` starts with `FuncType` (`void`, `int`, or `char`). This condition will never be true for valid function definitions, causing the parser to skip all `FuncDef` nodes.
2. **`parseFuncDef` Parent Assignment**: `funcType.setParentNode(funcType);` sets the parent of the `funcType` node to itself, creating a circular reference and breaking the AST structure. It should be `funcType.setParentNode(funcDef);`.
3. **`lastNum` Iterator Misuse**: `iter.previous()` is called twice consecutively. The first call moves the cursor back one position, and the second call moves it back another position, retrieving the line number of a token *two* steps behind instead of the immediately preceding token. This leads to incorrect error line reporting.
