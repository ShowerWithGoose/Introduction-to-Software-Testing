The primary error in the program is the incorrect order of outputting syntax component names. According to the problem description, the name of the current syntax component must be output **before** completing its analysis (i.e., before processing its children), but the current AST print methods output the component name **after** all children have been printed. This causes the output order to be reversed, which is likely the reason the program produces no output (or wrong output) for correct source programs.

The error is located in the `print()` methods of AST classes. For example, in `CompUnit.print()`:

<AST/AST.java>
...
        public void print() {
            for (Decl decl : declList){
                decl.print();
            }
            for (FuncDef funcDef : funcDefList){
                funcDef.print();
            }
            mainFuncDef.print();
            InputOutput.write(getType()); // <-- Component name printed at the end
        }
...

The component name (`<CompUnit>`) should be printed **before** the children, not after. This pattern is repeated in all other syntax components (e.g., `Block`, `Stmt`, `ConstDecl`, etc.) that are required to output their names.

Thus, the root cause is that the order of outputting non‑terminal names is reversed. To fix the program, the `write(getType())` calls should be moved to the beginning of each `print()` method, before processing the children.
