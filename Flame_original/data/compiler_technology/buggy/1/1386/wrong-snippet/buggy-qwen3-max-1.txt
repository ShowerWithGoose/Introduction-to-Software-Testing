Based on the problem description and code review, I can see that the test case input shows a compilation error where `g++` is not found, but this appears to be an environment issue rather than a code issue. However, looking at the actual code implementation, there's a clear bug in the `ConstInitValNode::createNode` method in AST.cpp.

The issue is in the handling of array initialization with braces. When encountering a `{` token, the code incorrectly creates an `LBRACKNode` (left square bracket) instead of an `LBRACENode` (left brace). This is a critical error that would cause incorrect parsing of array initializers.
