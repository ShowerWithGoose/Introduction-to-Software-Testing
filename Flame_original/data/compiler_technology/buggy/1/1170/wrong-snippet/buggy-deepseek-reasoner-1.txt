```java
package frontend.Syntax.Children;

import frontend.Syntax.Syntax;

public class AddExp {
    static void AddExpAnalysis() {
        MulExp.MulExpAnalysis();
        while (Tools.LookNextTK().tk.equals("PLUS") || Tools.LookNextTK().tk.equals("MINU")) {
            Tools.WriteLine(Syntax.NodeType.AddExp, Tools.GetNowTK().id);
            CompUnit.count++; // +-
            MulExp.MulExpAnalysis();
        }

        Tools.WriteLine(Syntax.NodeType.AddExp, Tools.GetNowTK().id);
    }
}
```
The error is that `AddExp` is output both inside the loop and at the end, leading to multiple or incorrect output of the non-terminal name. This pattern appears in multiple expression-handling classes (e.g., `MulExp`, `RelExp`, `EqExp`, `LAndExp`, `LOrExp`). The correct behavior per the problem description is to output the name once, after completing the analysis of the entire syntax component.
