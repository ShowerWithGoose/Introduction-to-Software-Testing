Based on the problem description and code review, the main issue appears to be in the `parseCompUnit()` method in `Parser.java`. The logic for distinguishing between declarations and function definitions is flawed - it uses lookahead tokens (`preRead` and `prePreRead`) but doesn't properly handle all cases, particularly when determining whether a type token (`int`/`char`) starts a variable declaration or a function definition.

The current implementation may incorrectly classify function definitions as declarations or vice versa, leading to parsing errors. Additionally, the error handling for missing main function is incomplete.
