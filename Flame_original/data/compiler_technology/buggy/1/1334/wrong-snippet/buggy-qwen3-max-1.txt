Based on the problem description and code review, I can see several critical errors in the parser implementation. The main issues are:

1. In `MainFuncDef()`, when handling missing right parenthesis error, it incorrectly adds `RBRACK` (]) instead of `RPARENT` ())
2. Similar error in `PrimaryExp()` and `UnaryExp()` where missing right parenthesis is handled by adding `RBRACK` instead of `RPARENT`
3. The parser doesn't properly handle EOF conditions, which could cause null pointer exceptions
4. The `CompUnit()` method has logic issues in determining whether to parse FuncDef or Decl

The most critical errors are in the error recovery code where wrong token types are being added.
