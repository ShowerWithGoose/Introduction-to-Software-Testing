### Summary
The task requires implementing a recursive descent parser for a specific grammar. The provided test case input (`/bin/sh: 1: javac: not found`) indicates an environment compilation issue, but analyzing the source code reveals a clear logical error in the parser implementation. 

In `Parser.java`, the `parseInitVal()` method incorrectly handles the grammar rule `InitVal → ... | '{' [ Exp { ',' Exp } ] '}' | ...`. The square brackets `[...]` indicate that the `Exp` inside the braces is **optional**. However, the code unconditionally calls `parseExp()` immediately after consuming `{`, which will cause parsing failures or incorrect AST generation for empty initializer lists (e.g., `int arr[2] = {};`). This contrasts with `parseConstInitVal()`, which correctly checks `if (peek(0) != TokenType.RBRACE)` before parsing the first expression.
