### Summary of Issues
1. **Incorrect Output File Routing (`output.java`)**: The specification requires that erroneous source programs output error information to `error.txt`. However, the code unconditionally initializes and writes to `lexer.txt`, regardless of the `sign` (error flag) value.
2. **State Variable Scope & Logic Confusion (`lexer.java`)**: The constructor uses both the parameter `sign2` and the instance variable `sign1` to track multi-line comment states. `sign2` is modified locally (`sign2 = 0;`) but this change is lost when the constructor returns, while `Compiler.java` relies on `getSign1()` to update its state. The `while` condition mixing `sign2` and `sign1` leads to broken cross-line comment skipping and incorrect state propagation.
3. **Enum Typo (`Token.java`)**: `COSNTTK` is misspelled (should be `CONSTTK`). While not directly causing a crash in the current flow, it violates the token category specification and will cause compilation/runtime errors if referenced.
