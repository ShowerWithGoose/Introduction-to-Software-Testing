## Summary

**Problem:** Lexical analyzer for a custom language that tokenizes source code and detects errors (specifically error type `a`: single `&` or `|` appearing alone).

**Test failure:** The compilation itself fails — `javac: not found` — meaning the code doesn't compile. This is a build environment issue, but there's also a clear **logical bug** in the source that would cause a runtime/compilation error:

In `frontend/Lexer.java`, the keyword map registers `"getin"` instead of `"getint"` for `GETINTTK`. This is a semantic error — `getint` would never be recognized as a keyword and would instead be tokenized as an identifier `IDENFR`.

Additionally, the `isNewLine()` method has a side effect (incrementing `lineno`) that is called inside a `while` condition in the whitespace-skipping loop, which means every newline increments `lineno` correctly — but this is fragile and worth noting.

The most clear-cut bug is the typo in the keyword map:
