### Summary
- **Problem Description**: Implement a lexical analyzer that tokenizes source code into specified categories. For erroneous programs, it must detect lexical errors (specifically error type `a` for individual `&` or `|` symbols) and output the line number and error code.
- **Test Case Input**: `/bin/sh: 1: javac: not found` (Indicates an environment/compilation failure in the grading system, preventing runtime output generation).
- **Buggy/Expected Output**: Not generated due to the compilation error, but static analysis of the source code reveals clear logical flaws.
- **Error Location**: In `frontend/Lexer.java`, the character constant parsing block incorrectly checks for double quotes (`isDQuotation()`) instead of single quotes (`isSQuotation()`) in the `while` loop condition. This typo will cause the lexer to incorrectly consume characters until a `"` is found, leading to wrong tokenization or infinite loops for `char` constants. Additionally, in the `isLOP()` block, the error handling for single `&` or `|` incorrectly appends a duplicate character to the token (`token += '&'`/`'|'`) after `catToken()` has already added the first one.
