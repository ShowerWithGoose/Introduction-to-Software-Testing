### Summary
- **Problem Description**: Implement a lexical analyzer that reads a source program, identifies tokens (keywords, identifiers, constants, operators, etc.), and outputs them to `lexer.txt`. If lexical errors occur (specifically error type `a` for standalone `&` or `|`), output the line number and error code to `error.txt`.
- **Test Case Input**: `/bin/sh: 1: g++: not found` (This indicates a compilation/environment failure in the test runner, but the code itself contains logical flaws that would cause runtime errors or incorrect outputs).
- **Buggy/Expected Output**: Empty (due to compilation failure or incorrect logic preventing proper execution).
- **Identified Issues**: The primary logical and semantic errors are located in `lexer.cpp`:
  1. `skip_multi_line_comment()` does not increment `line_number` when encountering `\n`, causing incorrect line tracking for all subsequent tokens/errors.
  2. `chrcon()` incorrectly handles character literals. For escape sequences, it pushes **two** separate tokens instead of one, and lacks bounds checking, which will corrupt the token stream.
  3. `next()` accesses `source[pos + 1]` for multi-character operators (`&`, `|`, `/`, `!`, `<`, `>`, `=`) without verifying `pos + 1 < source.length()`, leading to out-of-bounds memory access if the operator appears at the end of the file.
  4. `skip_single_line_comment()` may access `source[pos]` out of bounds if a single-line comment ends exactly at EOF without a trailing newline.

void Lexer::skip_multi_line_comment() {
    State state = State::S1; // start state
    while (pos < source.length()) {
        switch (state) {
            case State::S1:
                if (source[pos] == '*')
                    state = State::S2;
                break;
            case State::S2:
                if (source[pos] == '/')
                    state = State::S3;
                else if (source[pos] == '*')
                    state = State::S2;
                else
                    state = State::S1;
                break;    
            case State::S3:
                break;
            default:
                break;
        }
        // ERROR: Missing line_number++ when source[pos] == '\n' inside comments
        pos++;
        if (state == State::S3) break;
    }
}
