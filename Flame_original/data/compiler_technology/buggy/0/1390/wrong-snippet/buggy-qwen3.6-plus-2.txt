### Summary of Issues
1. **IndexOutOfBoundsException Risk**: In `frontend/Lexer.java`, when processing `&` or `|`, the code increments `pos` and immediately calls `input.charAt(pos)` without checking if `pos` is still within bounds. If the source program ends with a single `&` or `|`, this will throw a `StringIndexOutOfBoundsException`.
2. **Incorrect Output File for Errors**: According to the problem description, erroneous programs should output to `error.txt`, while correct programs should output to `lexer.txt`. However, `Compiler.java` unconditionally writes all output to `lexer.txt`, violating the specification.
