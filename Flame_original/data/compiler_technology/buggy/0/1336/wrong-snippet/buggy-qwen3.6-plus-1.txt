**Summary**
- **Problem Description**: Implement a lexical analyzer that reads `testfile.txt`, identifies tokens, and outputs them to `lexer.txt`. If lexical errors occur (specifically standalone `&` or `|`, error type `'a'`), it should output the line number and error code to `error.txt`.
- **Test Case Input**: `/bin/sh: 1: javac: not found` (Indicates the test environment failed to compile/run the program, typically caused by an infinite loop or unhandled termination condition in the lexer).
- **Buggy/Expected Output**: Empty (The program hangs or crashes before producing any output).
- **Error Location**: `frontend/Lexer.java` contains two critical logical flaws that cause infinite loops:
  1. The EOF boundary check `if (index == line.length())` does not account for `index` being incremented past `line.length()` by `getCharNow()`. This prevents `nextLine()` from being called at the end of the file, trapping the program in an infinite loop.
  2. When a single `&` or `|` is detected, the code adds an error to the list but does not `return` the token or advance the index. The lexer then re-reads the same character on the next iteration, causing an infinite loop on the erroneous input.
