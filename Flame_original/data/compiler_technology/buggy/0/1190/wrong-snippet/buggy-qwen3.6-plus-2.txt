### Summary of Issues
Based on the provided code and problem description, the program contains the following critical errors:
1. **`IndexOutOfBoundsException` in `peekNextChar()`**: The boundary check `if (pos >= text.length())` is incorrect. When `pos` points to the last character of the string (`pos == text.length() - 1`), `pos + 1` equals `text.length()`, causing `text.charAt(pos + 1)` to throw an exception. It should check `pos + 1 >= text.length()`.
2. **Incorrect comment skipping in `handleComment()`**: When a block comment `/*` is detected, the code calls `advance()` three times. This incorrectly skips the `*` and the first two characters of the actual comment content. It should only call `advance()` twice to consume `/` and `*`.
3. **Dependency on flawed `peekNextChar()` in `handleSymbol()`**: The `tmp` string is constructed using `peekNextChar()`, which will crash on the last character of the input. Additionally, `tmp` is evaluated after the first `advance()`, but since `currentChar` has already moved to the second character, the logic accidentally works for valid two-char operators, yet remains vulnerable to the bounds error.
