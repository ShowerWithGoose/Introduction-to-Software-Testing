**Summary of the Issue:**
The problem specification requires that when a lexical error occurs (specifically for individual `&` or `|` symbols), the program must output the error category code `a`. However, the `outputError` method in the provided code dynamically generates the error code by taking the first lowercase character of the `ErrorType` enum name (`errorType.name().toLowerCase().charAt(0)`). Since the enum is named `INVALID_CHAR`, this results in the character `i` being output instead of the required `a`, causing a mismatch with the expected output format.
