### Summary of the Issue
The program fails due to a `NullPointerException` in `frontend/Lexer/Lexer.java`. When the lexer processes single-character operators (e.g., `+`, `-`, `*`, `/`, `%`, `=`, `!`, `<`, `>`), the `tokenType` variable is not explicitly assigned within the `if-else` chain and remains `null`. The code then attempts to resolve the token type by calling `tokenType.getType(currentToken)`. Since `getType` is a `static` method, calling it on a `null` reference (`tokenType`) throws a `NullPointerException`, crashing the program before any output is generated. The correct call should be `TokenType.getType(currentToken)`.
