### Summary of the Issue
The lexical analyzer fails to correctly handle the case where the symbols `&` or `|` appear as the **last character on a line**. In the current implementation, after incrementing `curpos`, the code checks `if (curpos < line.length())`. If the symbol is at the end of the line, this condition evaluates to `false`, causing the entire block to be skipped. As a result, the program neither sets the `name`/`curLexType` nor records the required error type `'a'`. This leads to missing error reports or incorrect token generation for standalone `&` or `|` at line endings.
