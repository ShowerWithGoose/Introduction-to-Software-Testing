Based on the problem description and test case, the issue is related to error handling for invalid single '&' or '|' symbols (error type 'a'). The program should output the error to **error.txt**, but the current implementation prints it to `std::cout` and continues processing as if it were a valid token.

The key problems are:
1. Error output goes to `std::cout` instead of **error.txt**
2. The lexer continues processing after an error instead of stopping
3. The main function doesn't handle error output redirection

However, focusing on the most direct error location as requested, the issue is in the error handling within the lexer's `next` method where it creates and throws an error but then incorrectly assigns a valid token type.
