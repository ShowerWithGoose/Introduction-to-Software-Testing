**Summary**
The lexical analyzer uses a singleton pattern (`getInstance`) to create and manage the `LexicAnalysis` object. However, the static `instance` variable and its internal state (`WordList`, `ErrorList`, `curRow`, `curCol`, etc.) are never reset or cleared when a new test case is processed. If the testing environment executes multiple test cases within the same JVM session, the analyzer will carry over tokens, errors, and cursor positions from previous runs, causing incorrect outputs or logical failures for subsequent inputs. The `getInstance` method should either avoid the singleton pattern for this use case or explicitly reset/reinitialize the internal state when called with new input.
