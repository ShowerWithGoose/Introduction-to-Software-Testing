## Summary

The error is a **compilation error**: `javac: not found`, meaning the Java compiler isn't available in the environment. However, looking at the source code itself, there is also a clear **logical bug** in `frontend/Lexer.java`:

The entry for `>=` in the `singles` map has the wrong value string — it maps `">="` to `"EQL"` with value `"=="` instead of `"GEQ"` with value `">="`. This is a copy-paste error.

Additionally, the whitespace handling block reads a new character and then checks `if ((char) ch == '\n')` — but it already consumed the `\n` to detect whitespace via `Character.isWhitespace`, so the `line_num` increment logic is misplaced (the newline check happens on the *next* character, not the whitespace character that triggered the branch).

The two suspicious locations:

1. `singles.put(">=", new Single("==", "EQL"));` — the key is `">="` but the `Single` is constructed with `"=="` and `"EQL"`. It should be `new Single(">=", "GEQ")`.

2. The whitespace block increments `line_num` based on the character *after* the whitespace, not the whitespace character itself. If the whitespace character was `'\n'`, `line_num` should be incremented before reading the next character.
