**Summary of the Issue:**
The lexical analyzer is designed to detect error type `a` (individual `&` or `|` symbols). In the `nextSymbol()` method, the code calls `getChar()` to peek at the next character before checking if it forms a valid `&&` or `||` token. However, `getChar()` updates the `line` counter immediately when it encounters a newline (`\n`). If the character following `&` or `|` is a newline, `line` will have already been incremented by the time `errorLine = line;` is executed. This causes the reported error line number (and the token's line number) to be off by one. The line number should be captured *before* calling `getChar()` to accurately reflect the line where the error occurs.
