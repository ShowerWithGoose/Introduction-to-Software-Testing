**Summary**
The provided lexical analyzer contains critical logical errors in two main areas:
1. **Integer Constant Parsing (`Digit` state)**: The loop condition `Character.isDigit(peekChar())` incorrectly checks the *next* character instead of the *current* one. Since `peekChar()` advances the position and updates `current`, the loop appends the *previous* character, causing the last digit of an integer to be dropped and earlier digits to be duplicated. Additionally, if `peekChar()` returns `null` at EOF, it will throw a `NullPointerException` due to auto-unboxing.
2. **Error Handling for `&` and `|`**: When a single `&` or `|` is encountered, `popErrorA()` correctly logs the error, but the code fails to consume the erroneous character or reset the `state` to `Start`. This leaves `state` as `Operation` and `current` unchanged, causing an infinite loop during lexical scanning.
