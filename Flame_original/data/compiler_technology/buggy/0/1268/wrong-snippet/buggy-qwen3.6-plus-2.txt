### Summary
The provided lexical analyzer reads source code line-by-line to tokenize identifiers, constants, keywords, and operators, or to report lexical errors. The critical issue lies in `frontend/Lexer.java`. When detecting two-character operators (such as `&&`, `||`, `<=`, `>=`, `!=`, `==`, `//`, `/*`), the code directly accesses `line.charAt(columnNumber + 1)` **without verifying if `columnNumber + 1` is within the string's bounds**. 

If a single `&` or `|` (which should trigger error type `a`) appears at the very end of a line, `columnNumber + 1` will equal `line.length()`, causing a `StringIndexOutOfBoundsException` and crashing the program instead of correctly logging the error. The same bounds-checking flaw exists for `<`, `>`, `!`, `=`, `/`, and the string/character constant parsers.
