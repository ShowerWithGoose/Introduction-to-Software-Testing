### Summary
The lexical analyzer contains a logical flaw in how it handles lookahead for two-character operators (`||`, `&&`, `!=`, `==`, `<=`, `>=`) and error detection for single `|` and `&`. The code uses `if (!reader.ready())` to check for the end of the stream before reading the next character. `PushbackReader.ready()` is not a reliable EOF indicator and can behave unpredictably. More critically, regardless of the `ready()` check, the code unconditionally calls `reader.read()` immediately afterward. This causes **double-counting**: if `!reader.ready()` evaluates to true, it adds a token or error, then proceeds to read (often getting `-1` or blocking), fails the subsequent character comparison, and adds the token/error *again*. This results in duplicate entries in `lexer.txt` or `error.txt`. The `if (!reader.ready())` blocks are unnecessary and should be removed; the logic should rely solely on reading the next character and branching based on its value.
