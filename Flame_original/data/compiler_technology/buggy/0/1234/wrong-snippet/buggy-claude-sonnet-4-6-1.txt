## Summary

The error is a **compilation error** caused by `javac` not being found (`/bin/sh: 1: javac: not found`). This means the Java compiler isn't available in the environment, but looking at the source code itself for structural/logical issues that would cause compilation failure, the most likely culprit is in the `pushBackChar` method.

The `pushBackChar` method uses `currentChar = c` to "push back" a character, but `getChar()` already has logic to handle `currentChar != -1`. However, the real problem is that `pushBackChar` is supposed to use `PushbackReader.unread()` but instead stores the character in a field — and critically, it does **not** call `reader.unread(c)`. This means if `pushBackChar` is called and then `getChar` is called again, it returns `currentChar` without calling `updatePosition`, so the line number tracking is inconsistent. But more importantly for compilation: the `PushbackReader` is never actually used for pushback.

The actual compilation error (`javac: not found`) is an environment issue, but the logical error in the source is here:

```frontend/lexer/Lexer.java```
```
    // 从字符流中退回一个
    public void pushBackChar(int c) throws IOException {
        if (c != -1) {
            if (c == '\n') {
                lineNumber--; // 退回换行符，减少行号
            } else {
                charPosition--; // 退回字符位置
            }
            currentChar = c; // 保存当前字符以供下次读取
        }
    }
```

And the corresponding `getChar` that returns `currentChar` without updating position:

```frontend/lexer/Lexer.java```
```
    // 从字符流中读入一个
    private int getChar() throws IOException {
        // 检查当前字符是否有效，如果无效则读取新字符
        if (currentChar != -1) {
            int temp = currentChar; // 保存当前字符
            currentChar = -1; // 重置当前字符
            return temp; // 返回之前保存的字符
        }
        ...
    }
```

The `javac: not found` error is purely an environment issue — Java is not installed or not on the PATH in the test environment. The source code itself has no syntax errors that would prevent compilation given a proper Java environment.
