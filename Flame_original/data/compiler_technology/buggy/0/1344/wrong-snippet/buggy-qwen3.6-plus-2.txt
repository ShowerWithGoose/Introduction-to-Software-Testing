Based on the problem description and code analysis, here is a summary of the identified logical and semantic errors:

1. **Macro Precedence Bug (`isSpace`)**: The `isSpace` macro is missing outer parentheses. When expanded in `while (isSpace(cur_line[index]) && index < ...)`, C++ operator precedence causes `&&` to bind only to the `\t` check. This results in an infinite loop or out-of-bounds access when the lexer encounters a space or `\r`.
2. **Line Number Off-by-One**: In `Lexer::work()`, `line_num` is incremented *before* calling `parseLine()`. This causes the first line of the source file to be incorrectly tracked as line 2 instead of 1.
3. **Missing Bounds Check in `next()`**: After the whitespace-skipping loop, the code directly accesses `cur_line[index]` without checking if `index` has reached `cur_line.length()`. This triggers undefined behavior on lines that end with whitespace.
4. **Out-of-Bounds Access in `skip_comment()`**: The code accesses `cur_line[i + 1]` without verifying that `i + 1 < cur_line.length()`. This will cause a crash or undefined behavior if a line ends with a single `/`.
