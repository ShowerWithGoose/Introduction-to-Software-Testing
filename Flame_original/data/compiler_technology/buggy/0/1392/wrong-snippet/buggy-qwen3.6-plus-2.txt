**Summary**
- **Problem Description**: Design a lexical analyzer that reads `testfile.txt`, tokenizes correct source code into `lexer.txt`, or detects lexical errors (specifically error type `a` for standalone `&` or `|`) and outputs them to `error.txt`.
- **Test Case Input**: `/bin/sh: 1: javac: not found` (Indicates the program failed to compile in the testing environment).
- **Buggy/Expected Output**: Empty (due to compilation failure).
- **Error Analysis**: 
  1. **Compilation Error**: The code uses Java 14+ switch expression syntax (`case "keyword" ->`), which is incompatible with Java 8 environments commonly used in automated grading systems. This directly causes the `javac` compilation failure.
  2. **Logical/Runtime Error**: In the string constant parsing block, `next = source.charAt(i + 1);` lacks a boundary check. If a string literal reaches the end of the file without a closing quote, this will throw a `StringIndexOutOfBoundsException`.

            case "char" -> tokens.add(new Token(TokenType.CHARTK, lineNum, string));
