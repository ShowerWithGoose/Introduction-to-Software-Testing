### Summary of Information
- **Problem**: Implement a lexical analyzer that reads `testfile.txt`, identifies tokens, and outputs them to `lexer.txt`. If lexical errors occur (specifically error type `a` for standalone `&` or `|`), output the line number and error code to `error.txt`.
- **Input/Output**: Reads from `testfile.txt`. Outputs tokens as `TokenType value` to `lexer.txt`, or errors as `lineNumber errorCode` to `error.txt`.
- **Identified Errors**:
  1. **`nextDigital()` Logic Error**: The method uses `patternIdentAll` (`[0-9a-zA-Z_]`) to continue consuming characters. This incorrectly allows letters and underscores to be parsed as part of an integer constant. It should strictly consume only digits.
  2. **`isSpecialStringNotIdent()` Bounds Error**: The method calls `codeFile.substring(pos + len, pos + len + 1)` without checking if `pos + len` exceeds `fileLength`. This will throw a `StringIndexOutOfBoundsException` when a reserved keyword appears at the exact end of the file.
  3. **`Compiler.java` Formatting Issue**: The error output string is missing a trailing `\n`, which will cause multiple error entries to be written on the same line in `error.txt`.
