## Summary

**Problem:** Implement a lexical analyzer that tokenizes source code and detects errors (specifically, single `&` or `|` characters).

**Test Case Failure:** The program fails to compile because `javac` is not found in the environment (`/bin/sh: 1: javac: not found`). This is a compilation error, not a runtime logic error.

However, looking at the source code carefully, there are also logical bugs that would cause incorrect behavior even if it compiled. Let me identify the code locations with errors:

1. **`inMultiLineComment` is declared inside the loop** — it resets to `false` on every new line, breaking multi-line comment handling.
2. **Identifier parsing loop uses `currentChar` instead of `inputLine.charAt(i)`** — the condition `currentChar == '_'` never advances past underscores mid-word correctly.
3. **`operators` map uses wrong keys for brackets** — `"LBRACKET"`/`"RBRACKET"` instead of `"LBRACK"`/`"RBRACK"`.
4. **Error token for `&`/`|` is added to `errors` but the operator token is still added to `tokens`** — the invalid symbol token gets added to both lists.
