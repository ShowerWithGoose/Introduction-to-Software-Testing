**Summary of Information:**
- **Problem Description:** Design a lexical analyzer that tokenizes source programs or detects lexical errors (specifically error type `a` for standalone `&` or `|`). Correct programs output tokens to `lexer.txt`; erroneous programs output line numbers and error codes to `error.txt`.
- **Test Case Input:** `/bin/sh: 1: javac: not found` (Indicates the execution environment lacks a Java compiler, but the provided source code contains logical flaws that would produce incorrect results even if successfully compiled).
- **Buggy/Expected Output:** Empty (due to environment failure or logical bugs preventing correct execution).
- **Identified Logical/Semantic Errors:**
  1. **Missing End-of-Line Buffer Flush:** In `LexicalMain.runLexer()`, after processing each line, `curStr` is not flushed. Tokens ending exactly at the line boundary are lost or incorrectly merged with the next line's content.
  2. **Incorrect Quote Boundary Condition:** The checks `if(i > 0 && curLine.charAt(i-1) != '\\' ...)` for both `"` and `'` skip processing when a quote appears at the start of a line (`i == 0`), causing string/char constants at column 0 to be misparsed.
  3. **Swapped Row/Column Arguments:** `curLineNum` (line number) is consistently passed as the `col` parameter, and `i` (column index) as the `row` parameter. While not breaking the current token output format, it semantically misaligns with the `Word` and `Error` class definitions.
