### Summary of the Issue
The program incorrectly handles the lexical error type `a` (individual `&` or `|` symbols). In `CLexicalAnalyzer.cpp`, the `NextToken()` method initializes the token string to `"&&"` or `"||"` and unconditionally sets `m_nTokenType` to `AND` or `OR` **before** verifying whether the second character actually exists. When a single `&` or `|` is encountered, the code throws a `LexicalError` carrying the wrong lexeme (`"&&"`/`"||"` instead of `"&"`/`"|"`) and an incorrect token type. This causes malformed output in both `lexer.txt` and `error.txt` for this specific error case.
