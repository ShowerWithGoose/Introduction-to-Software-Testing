## Summary

**Problem:** Lexical analyzer that tokenizes source code and detects errors (like lone `&` or `|`).

**Test Case Input:** `/bin/sh: 1: javac: not found`

**Buggy Output:** (empty)

**Expected Output:** (empty — but the program should compile and run)

**Root Cause:** The program fails to compile because `javac` is not found on the system. However, looking at the source code itself for logical/semantic errors that would cause incorrect behavior:

The character constant handling has a bug. After matching a char token like `'\n'`, the code extracts `characterValue = token.substring(1, token.length() - 1)` which gives `\n` (two chars: backslash + n). But the switch cases compare against single characters like `"0"`, `"t"`, `"n"` — they never match because the extracted value includes the leading backslash (e.g., `\n` not `n`).

Additionally, the comment removal regex strips comments before tracking line numbers, which can cause line number mismatches for error reporting.
