### Summary of Information
- **Problem**: Implement a lexical analyzer that reads a source file (`testfile.txt`), identifies tokens, and outputs them to `lexer.txt`. If lexical errors occur, it must output the line number and error category code to `error.txt`.
- **Error Output Format Requirement**: `error line number error category code` (e.g., `1 a`)
- **Identified Issues**:
  1. **`io/Conponent/ErrorToken.java`**: The `toString()` method returns `errorType + " " + lineNum`, which reverses the required output format (`lineNum` should come first).
  2. **`frontend/Analyzer.java`**: The `throwError()` method ignores the `type` parameter and hardcodes `'a'` when instantiating `ErrorToken`. While currently only error `'a'` is defined, this violates the method's contract and will cause incorrect error reporting if other error types are added or passed.
  3. **`frontend/Analyzer.java` (Potential Runtime Error)**: The string (`"`) and character (`'`) parsing blocks access `line.charAt(i)` without verifying `i < line.length()` first. If a quote is unterminated, this will throw an `IndexOutOfBoundsException`.
