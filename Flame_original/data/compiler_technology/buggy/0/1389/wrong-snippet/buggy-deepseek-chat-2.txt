The error is likely in the `Compiler.java` file where the output file `lexer.txt` is opened in append mode (`true`). This means that if the file already exists from a previous run, the new output will be appended to the old content, causing incorrect results. The file should be opened in overwrite mode (without the `true` flag) to ensure fresh output for each run.
