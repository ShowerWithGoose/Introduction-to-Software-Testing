### Summary
The lexical analyzer is designed to detect error type `a` (individual `&` or `|` symbols). However, in the `getToken` function within `lexer_func.h`, when an individual `&` or `|` is encountered, the code sets `*end = p` and immediately returns `LEXER_ERROR_A` **without advancing the input pointer `p`**. Consequently, in the `initLexer` loop, `current` is updated to `next` (which still points to the exact same `&` or `|` character). This causes the lexer to repeatedly process the same erroneous character in an infinite loop, continuously recording the same error until the error table overflows or the program crashes. The pointer must be advanced past the erroneous character before returning the error code.
