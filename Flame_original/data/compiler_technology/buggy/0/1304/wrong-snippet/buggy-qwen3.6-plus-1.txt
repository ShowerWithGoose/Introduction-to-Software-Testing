### Summary of Issues
Based on the problem description and the provided source code, there are two critical logical/semantic errors in the lexical analyzer:

1. **Missing Error Reporting for Invalid Tokens (`&` and `|`)**: The problem specifies that standalone `&` or `|` should trigger error type `a`. In `getSymbol()`, these characters correctly result in an `INVALID` token type, but `doLexerAnalysis()` blindly adds all tokens to `tokenList` without checking for `TokenType.INVALID`. Consequently, the program never calls `ErrorHandler.ERROR_HANDLER.addError()` to output the required error to `error.txt`.
2. **Index Out of Bounds in `getWord()`**: The `do-while` loop in `getWord()` accesses `sourceCode.charAt(pos)` *before* verifying `pos < sourceCode.length()` in the loop condition. If an identifier or keyword appears at the very end of the file, `pos` will equal the string length after `pos++`, causing a `StringIndexOutOfBoundsException` and crashing the program (which aligns with the `/bin/sh: 1: javac: not found` or runtime failure observed in the test environment).

    private Token getSymbol() {
        char ch = sourceCode.charAt(pos);
        StringBuilder sb = new StringBuilder().append(ch);
        switch (ch) {
            case '&':
            case '|':
                if (pos + 1 < sourceCode.length() && sourceCode.charAt(pos + 1) == ch) {
                    sb.append(ch);
                    pos++;
                }
                break;
            case '<':
            case '>':
            case '=':
            case '!':
                if (pos + 1 < sourceCode.length() && sourceCode.charAt(pos + 1) == '=') {
                    sb.append('=');
                    pos++;
                }
                break;
            default:
        }
        pos++;
        String symbol = sb.toString();
        return new Token(symbol, Token.symbolToTokenType(symbol), line);
    }
