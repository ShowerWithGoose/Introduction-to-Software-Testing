**Summary of Information:**
- **Problem Description:** Design a lexical analyzer that reads `testfile.txt`, identifies tokens according to the specified grammar, and outputs them to `lexer.txt`. If lexical errors occur (specifically standalone `&` or `|`), output the line number and error code `a` to `error.txt`.
- **Test Case Input:** `/bin/sh: 1: javac: not found` (Indicates the testing environment failed to compile/run the program, likely due to runtime exceptions or logical flaws in the lexer).
- **Buggy/Expected Output:** Both are empty, confirming the program crashes or terminates prematurely before generating any output files.
- **Identified Errors:** The core issues reside in `frontend/Lexer.java`:
  1. `isArithmeticOperator`: The bounds check `pos + 1 >= input.length()` is placed *after* `input.charAt(pos + 1)`, causing a `StringIndexOutOfBoundsException` when `!` is the last character.
  2. `skipComments`: The block comment loop checks `input.charAt(pos + 1)` without verifying bounds first, risking an `IndexOutOfBoundsException`. It also stops at `*` without consuming the closing `/`, leaving `*` to be misparsed as `MULT`.
  3. `getLogicalOperator`: After detecting a single `&` or `|` and logging error `a`, the code unconditionally calls `move()` (if not at EOF), which consumes the next valid character and causes token loss/cascading errors.
