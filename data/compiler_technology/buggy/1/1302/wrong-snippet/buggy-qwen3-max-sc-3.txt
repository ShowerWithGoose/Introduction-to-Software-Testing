Based on the problem description and code review, the main issue appears to be in the `parseCompUnit` method in `Parser.java`. The current implementation incorrectly determines whether a declaration is a function definition or main function definition by looking ahead at tokens, which can lead to misclassification. Specifically, the condition `peekOffset(1).getLexType() == LexType.MAINTK` for detecting main function is flawed because it doesn't properly check that the current token is `INTTK` before `MAINTK`.

    public Node parseCompUnit() {
        Node root = new CompUnit(); root.setBeginLine(curToken.getLineNum());
        while (curToken != null) {
            Node child = null;
            if (peekOffset(1).getLexType() == LexType.MAINTK) {
                child = parseMainFuncDef();
            } else if(peekOffset(2).getLexType() == LexType.LPARENT) {
                child = parseFuncDef();
            } else if (curToken.getLexType() == LexType.CONSTTK) {
                child = parseConstDecl();
            } else if (isBType(curToken.getLexType())) {
                child = parseVarDecl();
            } else {
                System.out.println("Error: parseCompUnit error");
            }

            if (child != null) {
                root.addChild(child);
            }
        }
        root.setEndLine(peekOffset(-1).getLineNum());
        return root;
    }
