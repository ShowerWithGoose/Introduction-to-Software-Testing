Based on the problem description and code review, the main issue appears to be in the error handling implementation. The `Error` class uses a static list that is shared across all instances, but errors are being added incorrectly by creating a new `Error` instance and calling `addErrors()` on it, which adds the error to the static list but also creates redundant instances.

The critical error is in the `Lexer.java` file where single '&' or '|' characters are handled - it adds an error but then still adds a token for the same character, which is incorrect. Also, in `Parser.java`, the error handling methods create new `Error` instances and call `addErrors()` on them, which is redundant since the constructor already adds to the static list.
