Based on the problem description and code review, I can see several issues in the parser implementation. The main problems are:

1. In the `compUnit()` method, the loop conditions for detecting Decl and FuncDef are incorrect
2. In the `stmt()` method, the FOR loop handling uses wrong tokens (LBRACE/RBRACE instead of LPARENT/RPARENT)
3. In the `lAndExp()` and `lOrExp()` methods, the condition checks use `getValue()` instead of `getType()`
4. Several print methods in node classes have incorrect output patterns

The most critical error appears to be in the `compUnit()` method where the loop conditions are flawed, causing incorrect parsing of declarations and function definitions.
