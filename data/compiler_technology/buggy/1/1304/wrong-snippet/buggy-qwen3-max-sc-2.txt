Based on the problem description and code review, the main issue appears to be in the `parseCompUnit()` method in Parser.java. The current logic for determining when to parse declarations vs function definitions is flawed - it uses `peek(2) != TokenType.LPARENT` which doesn't correctly identify the boundary between declarations and function definitions according to the grammar.

The grammar specifies:
CompUnit → {Decl} {FuncDef} MainFuncDef

But the current implementation incorrectly assumes that any token sequence without LPARENT at position 2 must be a declaration, which can lead to incorrect parsing.

    private CompUnitNode parseCompUnit() {
        //Decl
        ArrayList<DeclNode> declNodes = new ArrayList<>();
        while (peek(2) != TokenType.LPARENT) {
            declNodes.add(parseDecl());
        }

        //FuncDef
        ArrayList<FuncDefNode> funcDefNodes = new ArrayList<>();
        while (peek(1) != TokenType.MAINTK) {
            funcDefNodes.add(parseFuncDef());
        }

        //MainFuncDef
        MainFuncDefNode mainFuncDefNode = parseMainFuncDef();

        return new CompUnitNode(declNodes, funcDefNodes, mainFuncDefNode);
    }
