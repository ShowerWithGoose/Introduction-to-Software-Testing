### Summary of Issues
The program fails primarily due to **unsafe token lookahead and missing boundary checks** in the parser, which lead to `NullPointerException` or `IndexOutOfBoundsException` when the token list is short or at the end of input. Specifically:
1. In `Parser.java`'s `parseCompUnit()` and `parseUnaryExp()`, `next()` is called unconditionally before checking if tokens exist. If `pos` exceeds the token list size, `getToken()` returns `null`, and subsequent `.getType()` calls crash the program.
2. In `Lexer.java`'s `skipComment()`, the `while(true)` loop for multi-line comments lacks an `isEnd()` check. If a comment is not properly closed before EOF, the lexer enters an infinite loop, causing the program to hang or crash.
3. The lookahead logic in `parseCompUnit()` is fundamentally flawed for recursive descent parsing. It manually advances and rewinds the token stream instead of using a safe peek mechanism, making it highly fragile and prone to state corruption.
