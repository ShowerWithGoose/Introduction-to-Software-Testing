### Summary of Errors
1. **Unsafe Lookahead & Index Out of Bounds in `CompUnit()`**: The `while` condition accesses `this.tokens.get(i+1)` and `this.tokens.get(i+2)` without checking if `i+1` or `i+2` are within the bounds of the `tokens` list. This will throw an `IndexOutOfBoundsException` on short or edge-case inputs. Additionally, the logic for distinguishing `Decl` from `FuncDef`/`MainFuncDef` using fixed lookahead offsets is fragile and incorrect for the given grammar.
2. **Invalid Index Access & Wrong Character in `consumeToken()`**: When a token is missing, the code attempts to report the error using `this.tokens.get(i-1).lineno`. If `i == 0` (i.e., the error occurs at the very beginning of the file), this causes an `IndexOutOfBoundsException`. Furthermore, for a missing `RBRACK`, the dummy token returns a full-width right parenthesis `"）"` instead of the correct `"]"`.
3. **Incorrect Semicolon Index in `StmtNode.output()`**: In the `For` statement output logic, `semicns.get(0)` is used twice. The second semicolon in a `for` loop should be retrieved using `semicns.get(1)`, otherwise the output will incorrectly repeat the first semicolon and miss the second one.
