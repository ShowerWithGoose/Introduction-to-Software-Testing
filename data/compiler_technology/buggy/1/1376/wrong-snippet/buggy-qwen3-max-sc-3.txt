The method doesn't properly handle the output format requirements - it should output terminal tokens (with their category code and value) AND non-terminal syntax components (in `<Component>` format), but the current implementation has logic issues in how it traverses and outputs the AST nodes.

2. The handling of when to output non-terminal component names is incorrect

    public static String astToString(ASTNode node) {
        if (node.getType() instanceof TokenType) {
            return node.getType().toString()+" "+node.getValue()+"\n";
        }
        StringBuilder sb = new StringBuilder();
        for (Object child : node.getChildren()) {
            if (child instanceof ASTNode) {
                sb.append(astToString((ASTNode) child));
            }
            else {
                sb.append(child.toString()).append(" ").append(getWordName((TokenType) child)).append("\n");
            }
        }
        if (node.getType() == NodeType.BlockItem || node.getType() == NodeType.Decl || node.getType() == NodeType.BType) {
            return sb.toString();
        }
        sb.append("<").append(node.getType().toString()).append(">").append("\n");
        return sb.toString();
    }
