Based on the problem description and code review, the main issue appears to be in the `compUnit()` method in Parser.java. The logic for determining when to stop parsing declarations and function definitions is flawed - it uses `tokens.get(index + 1)` and `tokens.get(index + 2)` which can cause IndexOutOfBoundsException and incorrect parsing decisions. Additionally, the parser doesn't properly handle the case where there are no declarations or function definitions before the main function.

    private CompUnitNode compUnit() {
        // CompUnit -> {Decl} {FuncDef} MainFuncDef
        List<DeclNode> declNodes = new ArrayList<>();
        List<FuncDefNode> funcDefNodes = new ArrayList<>();
        MainFuncDefNode mainFuncDefNode = null;
        // Decl
        while ((!tokens.get(index + 1).getType().equals(TokenType.MAINTK)) && (!tokens.get(index + 2).getType().equals(TokenType.LPARENT))) {
            // 下一个不是main也不是(，就保证了不是函数定义也不是主函数
            DeclNode decl = Decl();
            declNodes.add(decl);
        }
        // FuncDef
        while ((!tokens.get(index + 1).getType().equals(TokenType.MAINTK))) {
            // 没有Decl之后，下一个不是main就是普通函数定义
            FuncDefNode funcDefNode = FuncDef();
            funcDefNodes.add(funcDefNode);
        }
        // MainFuncDef
        mainFuncDefNode = MainFuncDef();
        return new CompUnitNode(declNodes, funcDefNodes, mainFuncDefNode);
    }
