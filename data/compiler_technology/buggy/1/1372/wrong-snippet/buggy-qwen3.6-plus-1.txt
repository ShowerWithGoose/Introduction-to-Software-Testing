### Summary of Identified Issues
Based on the provided grammar, error reporting requirements, and recursive descent parsing logic, the following critical errors are present in the code:

1. **`NullPointerException` on Lookahead & EOF**: Multiple methods (`handleCompUnit`, `handleUnaryExp`, `handleStmt`) call `tokens.preview(n).getLexType()` without checking if `preview()` returns `null`. When the parser reaches the end of the token list or looks ahead beyond available tokens, this causes a crash.
2. **Missing EOF Handling in `handleTerminal` & `verifyTerminal`**: `tokens.getThis()` returns `null` at EOF. Both `handleTerminal` and `verifyTerminal` directly call `.getLexType()` or `.getLineNum()` on the returned token without null checks, leading to runtime crashes when expected terminals are missing at the end of a file.
3. **Infinite Loop in Lexer String Parsing**: The string literal parsing loop in `Lexer.java` does not check for `EOF`. If a string is unterminated, `currentChar` becomes `(char)-1`, the loop condition remains true, and the lexer enters an infinite loop.
4. **Fragile Lookahead in `handleCompUnit`**: Using `preview(1)` and `preview(2)` to distinguish `MainFuncDef`, `FuncDef`, and `Decl` is highly sensitive to token positioning and will fail/crash on edge cases or malformed inputs.
