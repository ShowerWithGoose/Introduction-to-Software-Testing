Based on the problem description and code review, the main issue appears to be in the parser's handling of syntax component output. According to the requirements, certain syntax components should NOT be output (<BlockItem>, <Decl>, <BType>), but the current implementation outputs all non-terminals.

Looking at the Node.print() method in frontend/Node/Node.java, it handles special cases for some expression types but doesn't exclude the specified components from being output.

However, the most critical error is in the Parser.java file where the grammar parsing logic has issues with lookahead and backtracking, particularly in the parse_CompUnit method and statement parsing.

The key problematic area is in the parse_Stmt method where the lookahead logic is flawed, causing incorrect parsing decisions.
