Based on the problem description and code review, the issue appears to be in the error handling for missing semicolons in declaration statements. The test case input shows a compilation error related to javac not being found, but the core issue is likely in how the parser handles missing semicolons.

Looking at the code, in both `ConstDeclParser.java` and `VarDeclParser.java`, when a semicolon is missing, the error line number is set to the token before the expected semicolon. However, according to the problem description, for error type 'i' (missing semicolon), "the error line number corresponds to the line where the last non-terminal before the semicolon is located."

The issue is in the error reporting logic where it uses `t6.getLineNum()` (which is the token before the expected semicolon) but should use the line number of the last meaningful token in the declaration.
