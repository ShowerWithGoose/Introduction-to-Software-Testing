Based on the problem description and code review, the main issue appears to be in the error handling for missing semicolons in declaration statements. The test case input shows a compilation error related to javac not being found, but the actual issue in the code is in the declaration parsers where missing semicolots are not being handled correctly according to the error specification.

The error is most likely in the `ConstDeclParser.java` and `VarDeclParser.java` files where the semicolon error handling doesn't properly account for the line number requirement specified in error type 'i': "The error line number corresponds to the line where the last non-terminal before the semicolon is located."

Looking at the code, in both parsers when a semicolon is missing, they use the current token's line number rather than the line number of the last non-terminal before the expected semicolon.
