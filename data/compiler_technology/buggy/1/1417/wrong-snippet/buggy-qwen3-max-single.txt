Based on the problem description and code review, the main issue appears to be in the error handling mechanism. The `Error` class uses a static `isError` flag and static `tokenErrors` queue, but in the `Compiler.java` main method, a local `Error error = new Error();` is created which doesn't actually track errors because the static fields are shared across all instances.

The critical error is in the `Compiler.java` file where error checking is performed incorrectly, and also in the output generation logic for the parser.
