Based on the problem description and code review, the main issue appears to be in the `parseCompUnit` method in `Parser.java`. The method incorrectly uses `getNextToken(1)` and `getNextToken(2)` to look ahead, but doesn't properly handle the case when the token list is exhausted (returns null), which can cause NullPointerException or incorrect parsing decisions. Additionally, the logic for distinguishing between Decl and FuncDef is flawed - it checks `getNextToken(2).getType() == TokenType.LPARENT` but this doesn't reliably distinguish function definitions from variable declarations.

    public CompUnit parseCompUnit() throws IOException {
        CompUnit compUnit = new CompUnit();
        while (getNextToken(1).getType() != TokenType.MAINTK) {
            if (getNextToken(2).getType() == TokenType.LPARENT) {
                compUnit.addFuncDef(parseFuncDef());
            } else {
                compUnit.addDecl(parseDecl());
            }
        }
        compUnit.setMainFuncDef(parseMainFuncDef());
        if (isOutput) {
//        System.out.println("<CompUnit>");
            parserWriter.write("<CompUnit>\n");
        }
        return compUnit;
    }
