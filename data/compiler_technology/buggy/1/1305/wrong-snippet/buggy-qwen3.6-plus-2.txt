### Summary of Issues
1. **Compilation Error (Likely Cause)**: The `switch` statement in `Node/StmtNode.java` uses Java 14+ syntax (`case Label -> { ... }`). If the grading environment uses an older JDK (e.g., Java 8), this will directly cause a **Compilation Error**.
2. **Logical/Semantic Errors**:
   - In `Node/StmtNode.java`, the `For` case prints `semicolon1` twice instead of printing `semicolon2` for the second semicolon in the `for` loop header.
   - In `Frontend/Parser.java`, the `findToken` method incorrectly returns a token of type `TokenTypes.RBRACE` when recovering from a missing `RPARENT` error. It should return `TokenTypes.RPARENT`.
   - In `Frontend/Parser.java`, `compUnitAnalyze` accesses `tokens.get(index + 1)` and `tokens.get(index + 2)` without verifying that the list has enough elements, which will cause an `IndexOutOfBoundsException` at runtime when parsing ends.
