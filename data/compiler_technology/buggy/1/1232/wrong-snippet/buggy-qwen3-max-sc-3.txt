Based on the problem description and code review, the main issue appears to be in the lexer's handling of single '&' and '|' characters. According to the error specification, these should generate error type 'a', but the current implementation incorrectly sets the token type to AND/OR while setting an error flag.

The error is in the Lexer.java file in the handling of '&' and '|' characters:
