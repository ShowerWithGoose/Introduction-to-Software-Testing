1. **Off-by-one indexing in `Compiler.java`**: The `printResult` method iterates through tokens using a 0-based index `i`, but checks `nodes.containsKey(i + 1)`. Since `curTokenIndex` in `Parser.java` is initialized to `0` and `addNode` uses this 0-based index, the `nodes` map contains 0-based keys. This mismatch prevents syntax component names from ever being matched and outputted correctly.
2. **Incorrect `{` handling in `Parser.java`**: The `parseBlock` method consumes the `{` token with `curToken = tokens.get(++curTokenIndex);`, leaving `curToken` as `{`. It then immediately enters a `do-while` loop. Since `curToken` is `{`, it falls into the `else if (curToken.getType() != TokenType.RBRACE)` branch and calls `parseStmt()`. `parseStmt` sees `{` and recursively calls `parseBlock`, causing token misalignment and incorrect parsing of block structures (especially nested blocks).
