Based on the problem description and code review, the main issue appears to be in the error handling and node output logic in the Parser class. The key problems are:

1. The `addNode()` method uses `curTokenIndex` as the key, but nodes should be associated with token positions correctly
2. Error checking methods (`checkSemicn`, `checkRParent`, `checkRBreak`) don't advance the token pointer when errors occur, potentially causing infinite loops
3. The node output logic doesn't properly handle the ordering requirements specified in the problem

The most critical error is in the `parseCompUnit` method where the loop termination condition is incorrect - it only breaks when finding main function but doesn't handle end of file properly.

                funcDefs.add(parseFuncDef());
