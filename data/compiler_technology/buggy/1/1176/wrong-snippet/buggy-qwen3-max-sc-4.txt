Based on the problem description and code review, the main issue appears to be in the **CompUnit** parsing logic in Parser.java. The current implementation doesn't properly handle the grammar rule `CompUnit → {Decl} {FuncDef} MainFuncDef` because:

1. It checks for `MainFuncDef` inside the same loop as Decl/FuncDef
2. It doesn't ensure MainFuncDef appears exactly once at the end
3. The condition for detecting MainFuncDef is checked repeatedly in the loop

This would cause incorrect parsing when a valid program has declarations followed by main function.

    private void CompUnit(){

        //curNode = root;
        while((isConst() || isFuncType()) && !end()){
            if(isConst()){
                Decl(root);
            } else if (isFuncType()){
                if(lexList.get(curTokenIndex + 1).lexType.equals(Lexer.LexType.IDENFR)){
                    if(lexList.get(curTokenIndex + 2).token.equals("(")){
                        FuncDef(root);
                    } else if (!curLex.lexType.equals(Lexer.LexType.VOIDTK)){
                        Decl(root);
                    }
                }
            }
            if(curLex.lexType.equals(Lexer.LexType.INTTK) && lexList.get(curTokenIndex + 1).lexType.equals(Lexer.LexType.MAINTK)) {
                MainFuncDef(root);
            }
        }
    }
