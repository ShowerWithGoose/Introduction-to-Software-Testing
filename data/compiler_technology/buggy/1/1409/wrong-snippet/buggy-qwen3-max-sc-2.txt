Based on the problem description and code review, the main issue appears to be in the error handling logic for missing semicolons, right parentheses, and right square brackets. The `Judge_Error_i`, `Judge_Error_j`, and `Judge_Error_k` methods in the Parser class incorrectly handle error reporting by using `currTokenPos-1` which may not correspond to the correct line number for the error.

    public Token Judge_Error_i(Token currToken)
    {
        if (isMatch(currToken, LexType.SEMICN)) {
            return get_new_Token();
        } else {
            Token beforeToken = tokens.get(currTokenPos-1);
            errors.add(new Error(beforeToken.getLineNum(), ErrorType.i));
            return currToken;
        }
    }
