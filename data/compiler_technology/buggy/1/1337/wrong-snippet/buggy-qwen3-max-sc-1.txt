Based on the problem description and code review, the main issue appears to be in the error reporting mechanism. The test case shows a compilation error where g++ is not found, but the actual syntax analysis errors should be reported with proper line numbers and error codes.

Looking at the error handling code, the key issue is in the `printErrorInfo` function in Util.cpp, which incorrectly uses `Token::tokenIndex - 1` instead of the passed `lineNum` parameter. This would cause incorrect line number reporting for errors.

<util/Util.cpp>
...
void Util::printErrorInfo(int lineNum, const char *error) {
    std::ofstream errorFile("error.txt", std::ios::app);
    errorFile << Token::tokenList[Token::tokenIndex - 1].lineNum << " " << error << std::endl;
    errorFile.close();
}
...

The function should use the `lineNum` parameter that's passed to it, not the previous token's line number. This affects all error reporting throughout the parser.
