Based on the problem description and code review, the main issue appears to be in the error handling for missing semicolons (error type 'i'). The problem is that when an error is detected, the code reports the current token's line number instead of the line number of the last non-terminal before the semicolon as required by the specification.

Looking at multiple parser files, I can see this pattern repeated where `Util::printErrorInfo(token.lineNum, "i")` is used, but according to the specification, it should use the line number of the last non-terminal before the semicolon.

The most critical instance is in the VarDecl.cpp file:

<parser/declaration/variable/VarDecl.cpp>
...
    if (token.name == ";") {
        token = Token::getNextToken();
    } else {
        Util::printErrorInfo(token.lineNum, "i");
    }
...

Similarly in ConstDecl.cpp:

<parser/declaration/const/ConstDecl.cpp>
...
    if(token.name == ";"){
        token = Token::getNextToken();
    }
    else{
        Util::printErrorInfo(token.lineNum, "i");
    }
...

And in Stmt.cpp for various statements:

<parser/statement/Stmt.cpp>
...
        if (token.name == ";") {
            token = Token::getNextToken();
        } else {
            Util::printErrorInfo(token.lineNum, "i");
        }
...
