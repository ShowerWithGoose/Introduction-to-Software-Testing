Based on the problem description and code review, the main issue appears to be in the `compUnit()` method in the Parser class. The logic for determining when to stop parsing declarations and function definitions is flawed - it uses `tokens.get(index+1)` and `tokens.get(index+2)` which can cause IndexOutOfBoundsException and doesn't properly handle the transition to MainFuncDef.

        List<Decl> declList=new ArrayList<>();
        List<FuncDef> funcDefList=new ArrayList<>();
        MainFuncDef mainFuncDef=null;
        while(tokens.get(index+1).getLexType()!=LexType.MAINTK && tokens.get(index+2).getLexType()!=LexType.LPARENT){
            Decl decl=decl();
            declList.add(decl);
        }
        while(tokens.get(index+1).getLexType()!=LexType.MAINTK){
            FuncDef funcDef=funcDef();
            funcDefList.add(funcDef);
        }
        mainFuncDef=mainFuncDef();
        return new CompUnit(declList,funcDefList,mainFuncDef);
