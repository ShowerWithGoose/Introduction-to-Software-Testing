The current implementation incorrectly assumes that when it encounters a token followed by '(', it must be a function definition, but it doesn't properly handle the MainFuncDef case which has a specific structure ('int' 'main' '(' ')' Block).

The error is in the initial parsing logic that separates declarations from function definitions. The condition `while (!tokens.peek(2).is(TokenType.LPARENT))` is flawed because it doesn't account for the specific MainFuncDef pattern.
