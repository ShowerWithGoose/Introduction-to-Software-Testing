Based on the problem description and code review, the main issue appears to be in the parsing logic for the compilation unit structure. The parser incorrectly assumes that declarations end when it sees a token followed by LPARENT, but this doesn't properly handle the MainFuncDef which starts with "int main()".

The error is in the `parse()` method of Parser.java where it separates declarations from function definitions using an incorrect condition.

<frontend/ast/Parser.java>

    public void parse() {
        //int a
        //const int a
        //int a()
        while (!tokens.peek(2).is(TokenType.LPARENT)) {
            if (tokens.peek().is(TokenType.CONSTTK)) {
                compUnit.addDecl(parseConstDecl());
            } else {
                compUnit.addDecl(parseVarDecl());
            }
        }
        while (tokens.peek() != null) {
            compUnit.addFuncDef(parseFuncDef());
        }
    }
