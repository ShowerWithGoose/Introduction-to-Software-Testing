Based on the problem description and code review, the key issue appears to be in the `parseVarDecl` method in `Parser.java`. The method incorrectly includes `CONSTTK` in its token check when it should only check for `INTTK` or `CHARTK` (since `BType → 'int' | 'char'`). This would cause parsing errors when encountering variable declarations.

    private ParsedTree parseVarDecl() throws IOException {
        //VarDecl → BType VarDef { ',' VarDef } ';' // i
        LinkedList<ParsedTree> subTree = new LinkedList<>();
        peekAndAdd(subTree, INTTK, CONSTTK);
        subTree.add(parseVarDef());
        while (peekToken(COMMA)) {
            subTree.add(curToken);
            subTree.add(parseVarDef());
        }
        if (peekToken(SEMICN)){
            subTree.add(curToken);
        } else {
            addError("i");
        }
        return new ParsedTree("VarDecl", subTree);
    }
