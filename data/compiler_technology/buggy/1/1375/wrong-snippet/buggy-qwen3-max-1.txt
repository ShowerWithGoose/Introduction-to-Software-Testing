Based on the problem description and code review, the main issue appears to be in the error handling for missing semicolons (error type 'i'). The error line number should correspond to "the line where the last non-terminal before the semicolon is located", but the current implementation uses the line number of the expression/first element instead of the correct location.

Looking at multiple parser files, I can see this pattern repeated where semicolon errors are handled incorrectly. The most representative example is in the VarDeclParser:

<frontend/parser/declaration/variable/VarDeclParser.java>
...
        if(!token.type.equals(TokenType.SEMICN))
        {
            this.iterator.unReadToken(1);
            this.semicn = new Token(TokenType.SEMICN,this.btype.token.linenum,";");
            this.error_tokens.add(new Token(TokenType.SEMICN, first.my_line_num(), "i"));
        } else{
            this.semicn = token;
        }
...

The error token is being created with `first.my_line_num()` which may not be the correct line for the missing semicolon error according to the specification. Similar issues exist in other parsers like ConstDeclParser, StmtAssignParser, etc.
