Based on the problem description and the reported "Compilation Error", the primary issue lies in the use of modern Java syntax that may not be supported by the grading environment's compiler version. Specifically, the `expect` method in `Parser.java` uses Java 14+ switch expression syntax (`case X -> ...`). If the environment runs Java 8 or 11, this will immediately fail to compile. Additionally, the parser heavily relies on manual `move()` and `back()` calls for token lookahead, which is structurally fragile and prone to `IndexOutOfBoundsException` or infinite loops at runtime, but the switch syntax is the direct compilation blocker.

                case RBRACK -> this.errorList.addError(errLineNum, "k");
            }

        }
