Based on the problem description and code review, the main issue appears to be in the lexical analysis error handling. The test case input shows "/bin/sh: 1: javac: not found" which suggests a compilation error, but the real issue is in how lexical errors are handled.

Looking at the Lexical.java file, when encountering single '&' or '|', it adds error 'a' but still creates an AND/OR token. However, the bigger issue is in the lexical_print() method - it only writes to lexer.txt when there ARE errors, and writes to error.txt (empty) when there are NO errors. This is backwards.
