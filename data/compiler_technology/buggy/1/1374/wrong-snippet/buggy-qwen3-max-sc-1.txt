Based on the problem description and code review, the main issue appears to be in the `compUnit()` method in Parser.java. The method uses lookahead with hardcoded offsets (`curPos + 3`, `curPos + 2`) to determine when to stop parsing declarations and function definitions, which is error-prone and doesn't properly handle all cases. This can cause incorrect parsing behavior when analyzing the structure of the compilation unit.

    // CompUnit → {Decl} {FuncDef} MainFuncDef
    public static void compUnit() {
        ArrayList<Decl> declList = new ArrayList<Decl>();
        ArrayList<FuncDef> funcDefList = new ArrayList<FuncDef>();
        MainFuncDef mainFuncDef = null;
        while (!getType(curPos + 3).equals(LexerType.LPARENT)) { //decl
            declList.add(decl());
        }
        while (!getType(curPos + 2).equals(LexerType.MAINTK)) { //fundef
            funcDefList.add(funcDef());
        }
        mainFuncDef = mainFuncDef();
        set(declList, funcDefList, mainFuncDef);
    }
