Based on the problem description and code review, the main issue appears to be in the `compUnit()` method in Parser.java. The method uses lookahead with fixed offsets (`curPos + 3` and `curPos + 2`) to determine when to stop parsing declarations and function definitions, which is incorrect because:

1. The lookahead doesn't account for variable numbers of tokens in declarations/function definitions
2. It assumes a fixed structure that may not match the actual token stream
3. This can cause the parser to either skip valid declarations/functions or try to parse non-existent ones

The error is specifically in these lines:

<Fronted/parser/Parser.java>
...
    // CompUnit → {Decl} {FuncDef} MainFuncDef
    public static void compUnit() {
        ArrayList<Decl> declList = new ArrayList<Decl>();
        ArrayList<FuncDef> funcDefList = new ArrayList<FuncDef>();
        MainFuncDef mainFuncDef = null;
        while (!getType(curPos + 3).equals(LexerType.LPARENT)) { //decl
            declList.add(decl());
        }
        while (!getType(curPos + 2).equals(LexerType.MAINTK)) { //fundef
            funcDefList.add(funcDef());
        }
        mainFuncDef = mainFuncDef();
        set(declList, funcDefList, mainFuncDef);
    }
