Based on the problem description and code review, the key issue appears to be in the error handling for missing semicolons (error type 'i') and missing right parentheses (error type 'j'). The main problem is that the error line number is not being properly tracked in several parsing functions.

Looking at the Parser.java file, multiple functions set `error_line = this.lineNum` at certain points but don't update it when moving through tokens, leading to incorrect error line reporting. The most critical locations are in statement parsing functions where the error line should correspond to the line of the last non-terminal before the expected punctuation.
