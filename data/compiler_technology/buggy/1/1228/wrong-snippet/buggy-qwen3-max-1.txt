Based on the problem description and code review, I can see several issues in the parser implementation. The main problems appear to be:

1. In `parseConstDef()` and `parseVarDef()`, after parsing a potential array dimension (when encountering `[`), the code checks for `]` but doesn't properly handle the case where there's no assignment
2. In `parsePrimaryExp()`, when handling parentheses, it calls `getToken()` before parsing the expression, which skips the opening parenthesis
3. In `parseUnaryExp()`, the function call parsing logic has issues with parameter handling
4. The error checking logic in several places uses the wrong line number

The most critical error appears to be in the `parsePrimaryExp()` method where it handles parenthesized expressions:
