Based on the problem description and code review, the main issue appears to be in the `consume_step` method in Parser.java. This method handles error detection and reporting for missing tokens (like semicolons, parentheses, brackets), but has flawed logic for determining when to report errors and which line number to use.

The key problems are:
1. The error line number is taken from `gettoken(pos - 1)` which may not be correct in all cases
2. There's special handling for error type "i" (missing semicolon) that returns early without adding the error in some cases
3. The method doesn't properly handle end-of-file scenarios

    public void consume_step(Tokentype type, String value) {
        Token token = gettoken(pos);
        if (consume(token, type, value)) step();
        else {
            String err = type.equals(Tokentype.RPARENT) ? "j" : 
                type.equals(Tokentype.RBRACK) ? "k" : 
                type.equals(Tokentype.SEMICN) ? "i" : null;
            errcnt ++;
            int line = gettoken(pos - 1).getline();
            if (errcnt == 1 && gettoken(pos).getline() - line == 2) {
                if (err.equals("i")) {
                    return;
                }
            }

            Error e = new Error(err, line);
            errors.adderror(e);
        }
    }
