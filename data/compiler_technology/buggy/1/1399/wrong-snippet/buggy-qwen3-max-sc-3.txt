Based on the problem description and code review, the main issue appears to be in the `handleTerminal` method in Parser.java, which handles error reporting for missing tokens. The method incorrectly reports error line numbers by using `getLineNumber(-1)` which refers to the previous token's line number, but according to the problem specification, the error line number should correspond to "the line where the last non-terminal before the semicolon/right parenthesis/right bracket is located."

The error handling logic in `handleTerminal` needs to track the correct line number for error reporting. Additionally, there are issues in the expression parsing methods (`handleAddExp`, `handleMulExp`, etc.) where left recursion isn't properly handled.

    // 处理终结符
    private Token handleTerminal(LexType lexType) {
        if (getSymType().equals(lexType)) {
            Token token = getNowToken();
            nextIndex();
            return token;
        }
        is_analyze_success = false;
        if (lexType.equals(LexType.SEMICN)) {
            ErrorHandler.recordError(ErrorType.SEMICN_MISSING, getLineNumber(-1));
        }
        else if (lexType.equals(LexType.RPARENT)) {
            ErrorHandler.recordError(ErrorType.RPARENT_MISSING, getLineNumber(-1));
        }
        else if (lexType.equals(LexType.RBRACK)) {
            ErrorHandler.recordError(ErrorType.RBRACK_MISSING, getLineNumber(-1));
        }
        else {
            throw new Error(ErrorType.OTHER_ERROR, getLineNumber());
        }
        return null;
    }
