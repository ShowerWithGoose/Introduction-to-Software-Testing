Based on the problem description and code review, the main issue appears to be in the parser's handling of error reporting for missing semicolons (error type 'i'). The problem is that when an error occurs, the parser creates a fake token with lineNo = -1, but the error reporting requires the actual line number where the error occurred.

Looking at the VarDecl parsing method, when a semicolon is missing, it creates an error with the correct line number but then creates a fake token with lineNo = -1. This could cause issues in the output formatting.

    private VarDecl parseVarDecl() {
        // VarDecl -> BType VarDef { ',' VarDef } ';'
        BType bType = null;
        List<VarDef> varDefs = new ArrayList<>();
        List<Token> commas = new ArrayList<>();
        Token semicnToken = null;
        bType = parseBType();
        varDefs.add(parseVarDef());
        while (getTokenType() == TokenType.COMMA) {
            commas.add(token);
            nextToken();
            varDefs.add(parseVarDef());
        }
        if (getTokenType() == TokenType.SEMICN) {
            semicnToken = token;
            nextToken();
        } else {
            // error i
            Error error = new Error();
            error.lineNo = varDefs.get(varDefs.size() - 1).lineNo;
            error.type = ErrorType.i;
            Error.errors.add(error);

            // fix error
            semicnToken = new Token(";", TokenType.SEMICN, -1, -1);

        }
        return new VarDecl(bType, varDefs, commas, semicnToken);
    }
