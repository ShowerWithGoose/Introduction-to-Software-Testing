Based on the problem description and code review, the main issue appears to be in the `_CompUnit()` method in Parser.java. The current implementation has incorrect lookahead logic for determining when to stop parsing declarations and function definitions before the main function. The condition checks `tokenList.get(index + 1)` and `tokenList.get(index + 2)` which can cause IndexOutOfBoundsException and doesn't properly handle the grammar structure.
