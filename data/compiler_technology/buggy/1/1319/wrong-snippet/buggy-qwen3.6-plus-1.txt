### Summary of Issues
Based on the provided code and the "Compilation Error" indication, several critical issues have been identified:
1. **Java Version Compatibility (`Lexer.java`)**: The `ident()` method uses modern Java 14+ `switch` expressions (`case "main" -> ...`). If the judging environment uses Java 8 or 11, this will directly cause a **compilation failure**.
2. **Array Index Out of Bounds (`Parser.java`)**: The `for` loop in `parseStmt()` lacks a bounds check (`f < tokens.size()`). If a semicolon is missing or the file ends unexpectedly, `tokens.get(f)` will throw an `IndexOutOfBoundsException`. Additionally, `latestToken()` accesses `tokens.get(pos - 1)`, which crashes if `pos == 0` when `match()` fails at the start of parsing.
3. **Output Format Mismatch (`ASTPrinter.java`)**: `visitCompUnit` writes `<CompUnit>` without a trailing newline (`\n`), violating the requirement to output each component on a new line.
4. **Debug Output Interference (`Parser.java`)**: `System.out.println(pos+currentToken().s);` in `parseUnaryExp()` prints to standard output, which may cause presentation errors or interfere with automated grading.
