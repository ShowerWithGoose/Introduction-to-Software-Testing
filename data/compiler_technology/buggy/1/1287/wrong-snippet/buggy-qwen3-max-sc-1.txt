Based on the problem description and code review, the main issue appears to be in the `MyParser.java` file's `CompUnit()` method. The logic for distinguishing between `MainFuncDef`, `FuncDef`, and `VarDecl` is flawed - it only checks the next token after a BType to determine which production to use, but doesn't properly handle cases where a variable declaration might follow a BType.

The current implementation:
1. Checks if next token is MAINTK → MainFuncDef
2. Else if next token is IDENFR → then checks if next-next is LPARENT → FuncDef, else VarDecl

However, this fails to account for the fact that `CompUnit` can have multiple `Decl` (which includes `VarDecl`) before `FuncDef` and `MainFuncDef`. The grammar specifies `CompUnit → {Decl} {FuncDef} MainFuncDef`, meaning all declarations must come first, then function definitions, then main function.

The parser should first consume all possible `Decl` (both `ConstDecl` and `VarDecl`) before attempting to parse `FuncDef` or `MainFuncDef`.

        while (pos < tokens.size() && (nowRead().equals(LexType.CONSTTK) ||

            if (nowRead().equals(LexType.CONSTTK)) {
                root.addNode(ConstDecl());

            else {
                if (preRead(1).equals(LexType.MAINTK)) {
                    root.addNode(MainFuncDef());
                }
                else if (preRead(1).equals(LexType.IDENFR)) {
                    if (preRead(2).equals(LexType.LPARENT)) {
                        root.addNode(FuncDef());
                    }
                    else {
                        root.addNode(VarDecl());
                    }
                }
                else {
                    error();
                }
            }
        }
        return root;
