Based on the problem description and code review, the main issue appears to be in the **Char.java** and **Number.java** files where the error handling is incorrect. In both files, when the expected token is not found, the `err` flag is set to `false` instead of `true`, which means the parser won't recognize these as errors.

< pac/tool/Char.java >
...
    public Char(Lex lex) {
        if (lex.getFirst() == TOKEN.CHRCON) {
            val = lex.get();
            line = lex.getLine();
            lex.next();
        } else {
            err = false;  // This should be err = true;
        }
    }
