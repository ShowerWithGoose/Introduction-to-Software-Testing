The error is in the multi-line comment end detection: the condition `if(line.charAt(pos) != '*' && ...)` should be `if(line.charAt(pos) == '*' && ...)`. This causes the lexer to never properly close `/* */` comments.

Additionally, after the `if(ch == '/' && ...)` block, there is no `continue` or `else if`, so the code falls through to check `if(Character.isDigit(ch))` etc., potentially double-processing the same character (e.g., after consuming `/` as a single operator, the next iteration will process the same `ch` again because `pos` was already incremented? Actually, after the `else` branch, `pos++` is done, but then the outer `if` ends, and the code continues to the next `if(Character.isDigit(ch))` which will process the character at the new `pos`. That is fine because the character was already consumed. However, for the comment cases (`//` and `/*`), after the while loops, the outer `if` ends without any further action, and then the subsequent `if(Character.isDigit(ch))` might incorrectly try to process a character that is now either a newline or a character within the comment? Since `pos` has been advanced past the comment, the next character should be the first after the comment, so it should be fine. But the missing `continue` is not the main issue; the main issue is the logical error in the multi-line comment end condition.
