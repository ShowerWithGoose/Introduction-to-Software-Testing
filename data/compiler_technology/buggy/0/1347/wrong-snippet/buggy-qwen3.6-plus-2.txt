### Summary of Information
- **Problem**: Implement a lexical analyzer that reads `testfile.txt`, identifies tokens, and outputs them to `lexer.txt` for correct programs. For erroneous programs (specifically containing standalone `&` or `|`, error type `a`), it should output the line number and error code to `error.txt`.
- **Test Input/Output**: The provided test input shows an environment error (`javac: not found`), indicating the grading system failed to compile/run the Java code. However, analyzing the source code reveals logical flaws that would cause incorrect tokenization or output behavior.
- **Key Issues Identified**:
  1. **`digitConst` in `Lexer.java`**: Incorrectly treats `'0'` as a complete token immediately. This breaks multi-digit integers starting with `0` (e.g., `01`, `007`), splitting them into multiple tokens instead of a single `INTCON`.
  2. **Unconditional Output in `Compiler.java`**: Both `rightProcedure()` and `errorProcedure()` are called unconditionally. For erroneous programs, `lexer.txt` will still be generated, which may conflict with the requirement to only output to `error.txt` when errors exist.
  3. **Resource Leak in `errorProcedure`**: The `FileWriter` is not closed when `errors1` is empty, potentially leaving the file handle open.
