### Summary of Issues
Based on the provided source code and problem description, the primary logical error lies in `frontend/Lexer.java` within the comment and division operator handling block. After reading the `/` character and incrementing `pos`, the code checks `if (ch == '/')`. However, `ch` still holds the original `/` character and was never updated to reflect the new `pos`. This makes the condition **always true**, causing the lexer to incorrectly treat every `/` as a line comment (`//`), rendering the block comment (`/* ... */`) and division operator (`/`) branches completely unreachable. Additionally, multiple `source.charAt(pos)` calls throughout the lexer lack bounds checking (`pos < source.length()`), which will trigger `StringIndexOutOfBoundsException` when tokens or comments appear at the very end of the input file.
