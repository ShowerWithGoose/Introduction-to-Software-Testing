**Summary of the Issue:**
The lexical analyzer contains a logical error in the `next()` method when handling the specific lexical error for individual `&` and `|` symbols (error type `a`). At the beginning of the `else` block, `token` is already appended with `currentChar` (`token += currentChar;`). When the code detects that `|` or `&` is not followed by a matching symbol, it incorrectly appends the symbol again (`token += "|";` and `token += "&";`). This results in a duplicated token string (e.g., `"||"` or `"&&"` instead of `"|"` or `"&"`), which corrupts the token value and may lead to incorrect type mapping or output formatting.
