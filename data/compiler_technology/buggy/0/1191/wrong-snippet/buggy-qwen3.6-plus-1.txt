- **Problem**: Implement a lexical analyzer that outputs tokens or lexical errors.
- **Error Type**: Compilation Error.
- **Test Input/Output**: The input `/bin/sh: 1: javac: not found` indicates the grading environment failed to compile the Java source code.
- **Root Cause**: In `frontend/Lexer.java`, there is a **naming conflict** between the instance field `private HashMap<String, String> type;` and the nested enum `public enum type { ... }`. Java does not allow a field and a nested type (class/enum/interface) to share the same identifier within the same class scope, which directly causes a compilation failure. Additionally, the enum `type` is never actually used in the logic (string literals are used instead), making it redundant dead code that triggers the compiler error.
