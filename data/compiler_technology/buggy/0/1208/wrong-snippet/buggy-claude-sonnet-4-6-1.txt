## Summary

**Problem:** Lexical analyzer for a simple language. The test fails with a compilation error: `javac: not found` — meaning the Java compiler isn't available in the test environment. However, looking past the build environment issue, there are also logical bugs in the source code itself.

**Test case:** The program failed to compile/run (`javac: not found`), so there's no output to compare.

**Logical errors identified in the source code:**

1. **Single-line comment handling** — The loop reads characters but the condition checks `source.charAt(curPos) != '\n'` while the loop body advances `curPos` without checking the character it just read. The `\n` is consumed and `lineNum++` happens, but then `curPos` is advanced again, skipping the character after the newline.

2. **Block comment handling** — The `while` condition checks `source.charAt(curPos) != '*'`, but the `else if` inside the loop also checks for `c == '*'` — these are contradictory. The block comment parser is broken and will not correctly skip to `*/`.

3. **Missing bounds check** on `source.charAt(curPos)` in several operator handlers (`!`, `<`, `>`, `=`) — if these appear at the very end of the file, `curPos` would be out of bounds.

**Key issues:**

- **Single-line comment:** The `do-while` loop skips past the `\n` check incorrectly — after the loop, `curPos` points at `\n`, then the `if` block reads past it and increments `lineNum`, but the character after `\n` is skipped entirely (it will be missed on the next `next()` call since `curPos` already moved past it).

- **Block comment:** The `while` condition `source.charAt(curPos) != '*'` exits the loop when a `*` is found, but the `else if (c == '*' && ...)` inside the loop can never trigger because `c` is set *before* the `*` check in the condition — the loop exits before `c` can ever equal `*`. The `*/` terminator is never properly consumed, leaving `curPos` in the wrong position.

- **Operator lookahead:** `source.charAt(curPos)` is accessed without a bounds check (`curPos < length`) for `!`, `<`, `>`, and `=`, which will throw `StringIndexOutOfBoundsException` if any of these appear as the last character in the file.
