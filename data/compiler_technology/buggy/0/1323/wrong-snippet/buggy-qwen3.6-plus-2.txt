**Summary:**
The program specification explicitly requires that lexical errors be written to `error.txt`, but the current implementation incorrectly writes them to `lexer.txt`. Additionally, the error handling for single `&` and `|` characters in `lexer.cpp` logically fails: it appends a second character to `content` (creating `&&` or `||`), emits an `AND`/`OR` token, and does not properly isolate the error. The `while (!input.eof())` loop in `main.cpp` is also a known C++ anti-pattern that can cause an extra, unintended call to `nextToken()`. The primary issue violating the assignment requirements is the incorrect output file path for errors.
