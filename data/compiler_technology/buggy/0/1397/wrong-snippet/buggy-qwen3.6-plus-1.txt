### Summary
The assignment requires implementing a lexical analyzer that reads a source program, identifies tokens (keywords, identifiers, constants, operators, etc.), and outputs them to `lexer.txt`. If lexical errors occur (specifically error type `a` for standalone `&` or `|`), it should output the line number and error code to `error.txt`. 

Upon reviewing the provided code, a critical semantic error exists in `lexical/Lexer.java` within the `parseIdent()` method. The code uses the `==` operator to compare the parsed identifier string against keyword literals. In Java, `==` compares object references, not string content. Since `ident` is created via `StringBuilder.toString()`, it will always be a new object, causing `==` to evaluate to `false` for all keywords. Consequently, every keyword will be incorrectly classified as a generic identifier (`IDENFR`), breaking the lexical analysis for correct programs.
