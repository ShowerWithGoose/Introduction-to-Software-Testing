Based on the problem description and code review, the main issues are:

1. In Token.java, there's a typo in the enum: "COSNTTK" should be "CONSTTK"
2. The lexer doesn't properly handle the conversion of keywords to their specific token types (like MAINTK, CONSTTK, etc.) - it just marks them as KEYWORD
3. The output class tries to convert KEYWORD to uppercase + "TK" but this won't match the expected token names due to the enum typo

The most critical error is in the Token.java enum definition where "COSNTTK" is misspelled.
