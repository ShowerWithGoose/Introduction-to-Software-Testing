### Summary of the Issue
The program fails to compile primarily due to a **variable scope and definite assignment error** in the `switch` statement within `frontend/Lexer.java`. The variable `nextCh` is declared inside the `case '/':` block, but it is subsequently used in multiple other `case` blocks (`<`, `>`, `=`, `!`, `&`, `|`) without being declared or initialized in those paths. In Java, a variable declared in one `case` is not automatically visible or initialized in other `case` blocks unless declared at the `switch` level or enclosed in braces per `case`. This causes a `java: variable nextCh might not have been initialized` (or `cannot find symbol`) compilation error.

Additionally, there is a logical mismatch between the problem specification and the code: the token for `getint` should be `GETINTTK` according to the table, but the code defines and uses `GETINTK`. This will cause output mismatches during testing.
