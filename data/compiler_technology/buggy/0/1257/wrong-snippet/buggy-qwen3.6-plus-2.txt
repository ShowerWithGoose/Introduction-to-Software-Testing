### Summary
The program is a lexical analyzer that must correctly identify tokens and report lexical errors. Specifically, error type `a` should be triggered when the symbols `&` or `|` appear individually instead of as `&&` or `||`. 

The bug is located in `<mainSys/Parser.java>` within the `parseSigns()` method. The code **prematurely adds the `AND` or `OR` token to the output list before checking if the next character actually forms a valid `&&` or `||` sequence**. As a result, when an individual `&` or `|` is encountered, the program incorrectly records a valid token alongside the error, violating the expected output format.
