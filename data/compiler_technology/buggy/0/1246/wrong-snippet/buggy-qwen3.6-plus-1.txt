### Summary
The provided lexical analyzer is designed to tokenize source code and detect lexical errors, specifically error type `a` (individual `&` or `|` symbols). The test environment reports a compilation/runtime issue (`javac: not found`), but focusing on the source code logic reveals a critical bug in the `Lexer.java` file. 

When the lexer encounters `&` or `|` at the **very end of the input file** (i.e., when `pos == maxLength`), the condition `if (pos < maxLength && TokenType.doubleCharList.contains(nowChar))` evaluates to `false`. Since `&` and `|` are not in `singleCharList`, the lexer skips the entire symbol-handling block, leaves `tokenType` as `null`, and fails to increment `pos`. Consequently, the character is silently ignored instead of triggering error type `a`.
