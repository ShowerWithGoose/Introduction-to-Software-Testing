Based on the problem description and test case input, the key issue is that the program fails to handle **error type 'a'** correctly: detecting standalone `&` or `|` symbols as lexical errors.

Looking at the provided code, the main logic for handling this error is in the `next()` method of the `Lexer` class. The current implementation has a flaw in how it checks for error type 'a'. Specifically:

1. It only checks for error type 'a' **after** attempting to match double-character tokens
2. The condition `if (isErrorA(inputText.charAt(pos)))` is inside the block that checks `TokenType.doubleCharList.contains(nowChar)`, but it only triggers when the double-character token lookup fails
3. However, if a single `&` or `|` appears, it should immediately be flagged as an error without waiting for double-character matching logic
