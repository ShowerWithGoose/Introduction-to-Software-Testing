### Summary
- **Problem Description**: Implement a lexical analyzer that tokenizes a source program into specified token types or detects lexical errors (specifically, single `&` or `|` symbols). Outputs tokens to `lexer.txt` or errors to `error.txt`.
- **Test Case Input**: `/bin/sh: 1: javac: not found` (Indicates the program failed to compile or crashed at runtime during testing, resulting in no output).
- **Buggy/Expected Output**: Empty (due to runtime crash).
- **Error Location**: The crash is caused by **missing bounds checking** before accessing `srccode.charAt(p + 1)`. If the source code ends with a comparison operator (`<`, `>`, `=`, `!`) or a logical operator (`&`, `|`), `p + 1` will exceed the string length, throwing an `IndexOutOfBoundsException`. This prevents the lexer from finishing tokenization and writing to the output files.
