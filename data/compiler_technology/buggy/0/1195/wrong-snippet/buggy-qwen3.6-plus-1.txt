### Summary
The assignment requires implementing a lexical analyzer that reads a source program from `testfile.txt`, tokenizes it, and outputs results to `lexer.txt` (for correct programs) or `error.txt` (for programs with lexical errors like standalone `&` or `|`). The provided code attempts to read the entire file into a `StringBuilder`, then passes it to the `Lexer` for processing. 

A critical logical error exists in how the input file is read: **newline characters (`\n`) are stripped during file reading**. This causes all lines to be concatenated into a single continuous string. As a result:
1. `curLine` in `Lexer.java` will never increment, breaking line number tracking for error reporting.
2. Single-line comment handling (`//`) relies on detecting `\n` to terminate; without it, comments will consume the rest of the file.
3. Token boundaries across lines may be incorrectly merged.

Additionally, in `Lexer.java`, the `print()` method does not append a newline after each error entry and does not close the `PrintStream` objects, which can lead to empty or malformed output files due to unflushed buffers.
