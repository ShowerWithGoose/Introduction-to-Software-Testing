### Summary
- **Problem Description**: Implement a lexical analyzer that tokenizes source code according to a specified grammar. For valid programs, output token types and values to `lexer.txt`. For programs with lexical errors (specifically isolated `&` or `|`), output the line number and error code `a` to `error.txt`.
- **Test Case & Outputs**: The test input shows a shell execution error (`javac: not found`), indicating the program failed to compile or crashed at runtime. Both buggy and expected outputs are empty due to this failure.
- **Identified Errors**: The core issues reside in `frontend/Lexer.java` within the `analyse` method:
  1. **Missing Boundary Checks**: Multiple `while` loops and `if` conditions access `code.charAt(i)` without verifying `i < len`. This causes a `StringIndexOutOfBoundsException` when the lexer reaches the end of the file.
  2. **Incorrect Token Type for Character Constants**: Character literals (`'...'`) are incorrectly assigned `TokenName.STRCON` instead of the required `TokenName.CHRCON`.
  3. **Comment Handling Flaws**: The `//` comment loop lacks an EOF check and fails to update `lineNumber`. The `/* */` loop also lacks an EOF check, risking infinite loops or index-out-of-bounds crashes.
  4. **Rigid Char Constant Parsing**: The logic for `'...'` assumes a fixed length and does not safely handle EOF or malformed inputs.
