### Summary of Information
- **Problem Description**: Implement a lexical analyzer that outputs tokens for correct programs or detects lexical errors (specifically single `&` or `|` marked as error type `a`) for erroneous programs.
- **Test Case Input**: `/bin/sh: 1: javac: not found` (Indicates a runtime/compilation failure in the test environment, typically caused by an unhandled exception in the student's code).
- **Buggy/Expected Output**: Empty, consistent with a crash before any output is generated.
- **Error Identification**: In `lexer/Lexer.java`, the `readNextToken()` method repeatedly accesses `source.charAt(curPos)` **without verifying if `curPos < source.length()`**. This causes a `StringIndexOutOfBoundsException` when the source code ends with a symbol (`&`, `|`, `!`, `<`, `>`, `=`, `/`), or when parsing comments/strings at the end of the file. Additionally, the comment-parsing `while(true)` loops lack EOF checks, which can lead to infinite loops if the file ends prematurely.
