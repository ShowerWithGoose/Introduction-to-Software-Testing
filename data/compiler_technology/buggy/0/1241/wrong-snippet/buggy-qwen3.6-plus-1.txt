**Summary of Errors:**
Based on the problem description and code review, there are three primary logical/semantic errors in `frontend/Lexer.java`:
1. **Incorrect Token Generation for `!`**: In the `isExclamation` handling block, when `!` appears at the end of a line, the code mistakenly creates an `ASSIGN` token with the string `"="` instead of a `NOT` token with `"!"`. This is a clear copy-paste error.
2. **Missing Bounds Check for `/`**: In the `isDiv` block, the code accesses `line.charAt(++index)` without verifying if `index + 1 < line.length()`. This will cause a `StringIndexOutOfBoundsException` at runtime if `/` is the last character on a line.
3. **Incorrect Token String for `&` and `|` Errors**: When handling error type 'a' (individual `&` or `|`), the code incorrectly outputs the token string as `"&&"` or `"||"` instead of preserving the actual single character `"&"` or `"|"`. The problem requires preserving the exact character/string representation.
