Based on the problem description and test case input showing a compilation error due to missing javac, but more importantly analyzing the lexical analyzer code for error handling of individual '&' and '|' symbols (error type 'a'), I can identify the issue in the Lexer class.

The problem is in the handling of single '&' and '|' characters. According to the error specification, when '&' or '|' appear individually (not as '&&' or '||'), it should trigger error type 'a'. However, the current implementation has several bugs:

1. In the `isExclamation` case, when there's a single '!', it incorrectly creates an ASSIGN token ("=") instead of a NOT token ("!")
2. In the `isAnd` and `isOr` cases, it always adds an AND/OR token even for single characters, and then reports an error, but it should only report the error and not create a valid token

Looking specifically at the test case requirement for error type 'a' with input like "if(1 & 2)", the lexer should detect the single '&' and report error 'a' without producing an AND token.
