**Summary of the Issue:**
The program incorrectly formats the output for lexical errors. According to the problem description, the output format for an erroneous source program should be `error line number error category code` (e.g., `5 a`). However, the current `Token.toString()` method unconditionally outputs `type.toString() + " " + content`, which results in `a 5` for error tokens. This violates the specified output format and causes the test case to fail.
