**Summary**
The lexical analyzer is required to detect error type `a`, which occurs when the symbols `&` or `|` appear individually instead of as `&&` or `||`. In `frontend/Lexer.java`, the conditional statement intended to catch this error contains a logical typo: it checks `peekChar() == '|'` for the pipe character instead of `peekChar() != '|'`. Because `||` is already handled in a preceding `else if` block, this incorrect condition will never trigger for a standalone `|`, causing the program to miss the required error report.
