Based on the problem description and test case, the issue is related to error type 'a', which occurs when '&' or '|' appear individually (not as '&&' or '||'). The test input likely contains a single '&' or '|', and the program should detect this as an error and output the line number with error code 'a'.

Looking at the Lexer code, specifically the `addComparatorToken()` method, there are logical errors in how it handles single '&' and '|' characters.
