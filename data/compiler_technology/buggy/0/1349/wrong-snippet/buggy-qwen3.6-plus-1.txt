**Summary**
The lexical analyzer is required to detect error type `a` when the symbols `&` or `|` appear individually. In `frontend/Lexer.java`, the `addComparatorToken()` method processes these characters. The bug lies in the control flow: after determining that the next character is not `&` (or `|`), the code correctly logs the error and adds a single-character token, but then **unconditionally** executes the line that adds a double-character token (`&&` or `||`) immediately afterward. This happens because the `else` keyword is missing before the double-character token addition, causing incorrect token output and violating the expected lexical analysis behavior.
