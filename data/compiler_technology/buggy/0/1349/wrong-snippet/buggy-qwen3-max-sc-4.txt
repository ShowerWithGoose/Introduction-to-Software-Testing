Based on the problem description and test case, the issue is related to error type 'a', which occurs when '&' or '|' appear individually (not as '&&' or '||'). The test input likely contains such a case (e.g., `if(1 & 2)`), and the lexer should detect this and output an error.

Looking at the `addComparatorToken()` method in `Lexer.java`, there are logical errors in handling the '&' and '|' cases. Specifically:

1. For '&': When a single '&' is found (not followed by another '&'), the code adds an AND token (which is wrong - it should not add a valid token) and then also adds the error. But more critically, even after detecting the error, it still proceeds to add the "&&" token unconditionally.

2. Same issue exists for '|'.

The error is in the `addComparatorToken()` method in `Lexer.java`.
