Based on the problem description and test case input, the issue appears to be related to handling of the '&' and '|' operators. The error type 'a' is triggered when these symbols appear individually instead of as '&&' or '||'. 

Looking at the Lexer code, specifically the `addComparatorToken` method, there's a logic error in how it handles single '&' and '|' characters. When it encounters a single '&', it first adds an AND token for "&", then later adds another AND token for "&&", which is incorrect. The same issue occurs for '|'.
