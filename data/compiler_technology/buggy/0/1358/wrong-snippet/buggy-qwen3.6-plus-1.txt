### Summary of Identified Errors
Based on the problem description and code analysis, there are two primary logical/semantic errors in `Lexer.java`:
1. **Incorrect Token Type for Character Constants**: When parsing a character constant (enclosed in single quotes), the code assigns `Symbol.CHARTK` to `symbol`. According to the token category table, `CHARTK` corresponds to the `char` keyword, while character constants should be assigned `Symbol.CHRCON`.
2. **Missing Loop Continuation After Lexical Error Reporting**: When a single `&` or `|` is encountered (error type `a`), the program correctly calls `Error.lexErrorA(...)`. However, it fails to `continue` the loop. As a result, execution falls through to the end of the `while` loop where `reader.getChar()` is unconditionally called. This causes the lexer to incorrectly skip the character immediately following the erroneous `&` or `|`, leading to misaligned parsing for the rest of the file.
