### Summary
- **Problem Description**: The lexical analyzer should output tokens to `lexer.txt` for correct programs and error information (line number + error code) to `error.txt` for erroneous programs.
- **Test Case & Outputs**: The provided test case input appears to be an environment error (`javac: not found`), but the core issue lies in the program's logic. The buggy and expected outputs are empty, indicating the program fails to write to the correct files or crashes before output.
- **Identified Error**: In `Compiler.java`, the output streams are **swapped**. When `lexer.isError()` is true, `printError()` writes to `outputStream` (which points to `lexer.txt`). When false, `printTokens()` writes to `errorStream` (which points to `error.txt`). This directly violates the output format specification, causing correct programs to write to the error file and erroneous programs to write to the token file.
