Based on the problem description and test case, the program should detect missing semicolons (error type 'i') on lines 2, 9, and 13. The issue appears to be in the parser's error handling for missing semicolons.

Looking at the test input:
- Line 2: `printf("Hello SysY!\n")` is missing a semicolon
- Line 9: `int a` is missing a semicolon 
- Line 13: `return 0` is missing a semicolon

The error handling for missing semicolons is implemented in the `getToken` method in the Parser class. When a required token is not found, it checks if it's a semicolon and adds error type 'i'. However, the line number being reported might be incorrect because it uses `before.line` (the previous token's line) rather than the current token's line or the actual line where the statement ends.

                return new Token(TokenType.RPARENT,")", before.line);

                ErrorHandler.add(before.line, ErrType.k);
                return new Token(TokenType.RBRACK,"]", before.line);
