```java
// CompUnit -> {Decl} {FuncDef} MainFuncDef
private CompUnitNode CompUnit()   {
    List<DeclNode> declNodes = new ArrayList<>();
    List<FuncDefNode> funcDefNodes = new ArrayList<>();
    MainFuncDefNode mainFuncDefNode = null;
    int count = 0;

    while(tokens.get(currentTokenIndex + 2).getLexType() != LexType.LPARENT) {
        DeclNode declNode = Decl();
        declNodes.add(declNode);
    }
    while (tokens.get(currentTokenIndex + 1).getLexType() != LexType.MAINTK) {
        // break;
        funcDefNodes.add(FuncDef());
        count ++ ;
        if (count == 9) {
            break;
        }
    }
    for(int i = currentTokenIndex; i < tokens.size(); ++i) {
        if (tokens.get(i).getLexType() == LexType.INTTK && tokens.get(i+1).getLexType() == LexType.MAINTK) {
            currentTokenIndex = i;
            break;
        }
    }
    mainFuncDefNode = MainFuncDef();

    return new CompUnitNode(declNodes,funcDefNodes,mainFuncDefNode); // 返回构建的编译单元节点
}
```

The problem is in the second while loop condition: `while (tokens.get(currentTokenIndex + 1).getLexType() != LexType.MAINTK)`. This condition is not correctly identifying when to stop parsing function definitions. Additionally, there's an artificial limit of `count == 9` which is causing the parser to stop prematurely before parsing all functions.

The correct approach should be to parse function definitions until we encounter the main function, but the current logic is flawed and misses the last two functions (`judgeEqual` and `judgeTrue`).
