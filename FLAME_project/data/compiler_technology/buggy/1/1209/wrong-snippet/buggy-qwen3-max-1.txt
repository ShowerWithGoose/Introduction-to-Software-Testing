The problem is that the parser is adding the same error multiple times. Looking at the code structure, I can see that in the `ASTNode.expect()` method, when an expected token is not found, it adds an error to the local `errors` list. However, this error gets added again when `addErrors()` is called in the `toString()` method.

The key issue is in the `expect()` method in `ASTNode.java` - it adds errors immediately, but then these errors are also added again through the `addErrors()` call in `toString()`. This causes duplicate error reporting.

Looking more carefully at the test case:
- Line 1: `void f(int a {}` - missing closing parenthesis after parameter list
- Line 3: `f(a + 10;` - missing closing parenthesis in function call  
- Line 5: The `main` function has `int main( {` which is missing the closing parenthesis

The duplicate errors suggest that the error collection mechanism is flawed. The `expect()` method adds errors to the local `errors` list, and then `addErrors()` is called in `toString()`, but if `toString()` is called multiple times or if there are multiple paths that lead to the same error being recorded, duplicates occur.
