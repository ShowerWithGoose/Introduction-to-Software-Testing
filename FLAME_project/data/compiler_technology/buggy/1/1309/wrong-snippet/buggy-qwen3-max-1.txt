The issue is in the condition `peekSymbol(3).getType() != SymbolType.LPARENT` for detecting VarDecl. This condition is flawed because it's looking ahead 3 tokens to see if there's no LPARENT, but this doesn't correctly identify variable declarations.

For a variable declaration like `int a = 0;`, the tokens are: `INTTK`, `IDENFR`, `ASSIGN`, ... so `peekSymbol(3)` would be `ASSIGN`, not `LPARENT`. But for a function definition like `int func()`, the tokens are: `INTTK`, `IDENFR`, `LPARENT`, so `peekSymbol(3)` would be `LPARENT`.

However, the logic is reversed - it should be checking if `peekSymbol(3)` IS `LPARENT` to distinguish function calls/definitions from variable declarations.
