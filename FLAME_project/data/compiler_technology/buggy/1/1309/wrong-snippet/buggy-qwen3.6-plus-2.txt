**Summary of Analysis:**

1.  **Problem Description**: The task is to implement a recursive descent parser that outputs lexical tokens and specific non-terminal syntax components (like `<Stmt>`, `<VarDecl>`, etc.) in the order they are recognized.
2.  **Test Case Input**: A valid C-like source code containing `main` function, `printf`, variable declarations, and nested blocks.
3.  **Buggy Output vs Expected Output**:
    *   The Buggy Output contains a duplicate sequence of tokens and syntax components at the beginning. Specifically, it prints the tokens for `int main() { printf("21373119"); ...` twice.
    *   The first occurrence lacks the `<Stmt>` tag after `printf(...)`.
    *   The second occurrence includes the `<Stmt>` tag and subsequent correct parsing structure.
    *   This indicates that the parsing logic is executing twice or printing tokens twice for the initial part of the program.
4.  **Root Cause Identification**:
    *   In `Compiler.java`, the `Lexer` is instantiated: `Lexer lexer = new Lexer(file);`.
    *   Inside the `Lexer` constructor, `this.doLexer()` is called to tokenize the input. Crucially, at the end of the constructor, `if(shouldPrint) this.print();` is executed. The `print()` method iterates through `symbolList` and calls `symbol.print()`, which outputs `System.out.println(this.type + " " + this.value);`. This dumps all lexical tokens to `parser.txt`.
    *   Then, `Parser parser = new Parser(lexer.getSymbolList());` is created.
    *   Inside the `Parser` methods (e.g., `nextSymbol`), when a symbol is consumed, `ans.print()` is called, which *again* outputs `System.out.println(this.type + " " + this.value);`.
    *   Therefore, every token is printed once by the `Lexer` constructor and once by the `Parser` during analysis. This explains the duplication in the output.
    *   The `Lexer` should not print the tokens if the `Parser` is responsible for printing them in the context of syntax analysis output, or the `Parser` should not print them if the `Lexer` already did. Given the requirement to output tokens in the order recognized by lexical analysis *during* syntax analysis (and the structure of the expected output which interleaves tokens and non-terminals), the `Lexer`'s automatic printing is incorrect. The `Parser` controls the flow and output.

**Error Location:**

The error is in `frontend/Lexer.java`, where the `print()` method is called in the constructor, causing premature and duplicate output of tokens.
