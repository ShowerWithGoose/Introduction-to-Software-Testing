Looking at the problem description and comparing the buggy output with the expected output, I can see that there are extra `<Stmt>` outputs in the buggy version.

Specifically, in the buggy output:
- After the first `printf` statement, there are **two** `<Stmt>` lines instead of one
- After the inner block (before `RBRACE }`), there are **two** `<Stmt>` lines instead of one  
- After the `return` statement, there are **two** `<Stmt>` lines instead of one

The issue is in the `parseStmt()` method in the Parser.java file. The method already outputs `<Stmt>` at the end of each specific statement parsing method (like `parseEmptyStmt()`, `parseBlockStmt()`, etc.), but then it outputs another `<Stmt>` at the very end of `parseStmt()`.

Looking at the code, each individual statement parsing method (like `parseEmptyStmt()`, `parseBlockStmt()`, `parseIfStmt()`, etc.) already calls `outputGrammar("<Stmt>")` at the end. However, the main `parseStmt()` method also calls `outputGrammar("<Stmt>")` after the switch statement, causing duplicate outputs.

    private void parseStmt() throws IOException {
        int currentLine = getPreviousTokenLineNumber();
        switch (currentToken.getType()) {
            case SEMICN ->
                // 空语句
                    parseEmptyStmt();
            case LBRACE ->
                // 复合语句块
                    parseBlockStmt();
            case IFTK ->
                // 处理 if 语句
                    parseIfStmt(currentLine);
            case RETURNTK ->
                // 处理 return 语句
                    parseReturnStmt(currentLine);
            case PRINTFTK ->
                // 处理 printf 语句
                    parsePrintfStmt(currentLine);
            case IDENFR ->
                // 可能是赋值语句或函数调用语句
                    parseAssignmentOrFunctionCallStmt(currentLine);
            case GETINTTK, GETCHARTK ->
                // 处理 getint() 或 getchar() 函数调用语句
                    parseGetStmt(currentLine);
            case FORTK ->
                // 处理 for 循环语句
                    parseForLoopStmt(currentLine);
            case CONTINUETK ->
                // 处理 continue 语句
                    parseContinueStmt(currentLine);
            case BREAKTK ->
                // 处理 break 语句
                    parseBreakStmt(currentLine);
            default ->
                // 其他情况，可能是表达式语句
                    parseExpressionStmt(currentLine);
        }
        outputGrammar("<Stmt>");
    }
