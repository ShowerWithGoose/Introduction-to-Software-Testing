Looking at the test input:
```
void f(int a {} // error 1
int main( { // error 2
    int a = 10;
    f(a + 10; // error 3
    return 0;
}
```

The errors should be:
- Line 1: missing ')' after parameter list → error "j"
- Line 3: missing ')' in main function declaration → error "j" 
- Line 5: missing ')' in function call → error "j"

But the buggy output shows "5 j" twice, indicating that the parser is reporting the same error twice for line 5.

Looking at the `expect` method in `Parser.java`, this is where errors are added. The issue appears to be in the `parseStmt` method where it tries to parse different statement types and may call `expect` multiple times for the same missing token.

Specifically, in the `parseStmt` method, when handling function calls like `f(a + 10;`, the parser likely attempts to parse it as different statement types and ends up calling the `expect(TokenType.RPARENT)` method multiple times, adding the same error twice.

The problem is in the `parseStmt` method's logic for determining statement types. It uses a trial approach by parsing an expression first, then backing up and re-parsing based on what follows. This can cause multiple error reports for the same missing token.

        } else if (this.token.getTokenType() == TokenType.BREAKTK) {
            return parseBreakStmt();
        } else if (this.token.getTokenType() == TokenType.CONTINUETK) {
            return parseContinueStmt();
        } else if (this.token.getTokenType() == TokenType.RETURNTK) {
            return parseReturnStmt();
