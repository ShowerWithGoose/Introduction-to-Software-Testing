In the test case `if(i==2)`, the expression `i==2` is an Equality Expression (`EqExp`). The grammar defines:
`EqExp → RelExp { ('==' | '!=') RelExp }`
`RelExp → AddExp { ('<' | '>' | '<=' | '>=') AddExp }`

When parsing `i==2`:
1. `EqExp` calls `RelExp`.
2. `RelExp` calls `AddExp`.
3. `AddExp` parses `i` (an identifier).
4. Control returns to `RelExp`. It checks if the next token is a relational operator (`<`, `>`, `<=`, `>=`). The next token is `==`, which is **not** a relational operator.
5. Therefore, `RelExp` should finish and return control to `EqExp`.
6. `EqExp` then checks for `==` or `!=`. It finds `==`, consumes it, and parses the right-hand side `RelExp` (which parses `2`).
