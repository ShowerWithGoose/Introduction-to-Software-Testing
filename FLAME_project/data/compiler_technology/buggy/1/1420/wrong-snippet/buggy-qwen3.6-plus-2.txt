    ```java
        if (getCurrentToken().getValue().equals(")")) {
            mainFuncDefNode.addChild(new SyntaxTreeNode("RPARENT", ")"));
            advance(); // 跳过 ')'
        } else {
            // 缺少右小括号，记录错误 (错误类型 j)
            recordError(getPreviousToken().getLineNumber(), "j");
        }

        // 解析函数体 Block
        mainFuncDefNode.addChild(parseBlock());

        return mainFuncDefNode;
    }
    ```
    
    If `)` is missing, it records error but **does not advance**.
    Then it calls `parseBlock()`.
    
    `parseBlock()` expects `{`.
    
    ```java
    private SyntaxTreeNode parseBlock() {
        // ...
        if (getCurrentToken().getValue().equals("{")) {
            // ...
            advance();
        } else {
            System.out.println("Error: Expected '{' in Block");
            // System.exit(currentTokenIndex);
        }
        // ...
    }
    ```
    
    If `{` is missing, it prints error and presumably exits or throws? The code says `System.exit` is commented out. It just prints. It does **not** advance.
    
    Then `parseBlock` continues to `while (!getCurrentToken().getValue().equals("}"))`.
    
    If the token is NOT `}`, it calls `parseBlockItem`.
    
    If the token is stuck, `parseBlockItem` -> `parseStmt` -> ...
    
    If `parseStmt` cannot consume the token, it might return without advancing.
    
    If `parseBlock`'s loop condition `!getCurrentToken().getValue().equals("}")` remains true because the token never advances to `}`, and `parseBlockItem` doesn't advance, we have an infinite loop.

    In our specific test case, `main()` is correct. `int main() {`. So `parseMainFuncDef` advances correctly.

    However, there is a known issue in student parsers: **Left Recursion** or **Failure to Advance on Error**.

    Look at `parseBlockItem`:
    ```java
    if (getCurrentToken().getTokenType() == LexType.INTTK || ...) {
        blockItemNode.addChild(parseDecl());
    } else {
        blockItemNode.addChild(parseStmt());
    }
    ```
    
    If `parseDecl` or `parseStmt` returns without advancing (due to error), `parseBlock` loops infinitely.

    In the provided test case, are there any errors that cause non-advancement?
    
    Error 1: `printf(...)`. Missing `;`.
    `parseStmt` (printf case):
    Matches `printf`, `(`, `Str`, `)`.
    Checks `;`. Not found.
    Records error.
    Returns.
    Did it advance? Yes, it advanced past `)`. The current token is now `if`.
    
    Error 2: `int a`. Missing `;`.
    `parseVarDecl`:
    Matches `int`, `a`.
    Checks `;`. Not found (next is `for`).
    Records error.
    Returns.
    Did it advance? Yes, it advanced past `a`. The current token is now `for`.
    
    Error 3: `return 0`. Missing `;`.
    `parseStmt` (return case):
    Matches `return`.
    Parses `Exp` (`0`). Advances past `0`.
    Checks `;`. Not found (next is `}`).
    Records error.
    Returns.
    Did it advance? Yes, it advanced past `0`. The current token is now `}`.

    So in all error cases, the parser **did** advance past the last valid token.

    **Is it possible the TLE is due to `Lexer`?**

    In `Lexer.java`:

    If `Compiler.java` is run, it creates a Lexer.

    **Conclusion:**
    
    The most likely cause of TLE in these types of assignments, when the logic seems correct for the happy path and simple errors, is often an **infinite loop in the parser when encountering an unexpected token that isn't handled by advancing**.
