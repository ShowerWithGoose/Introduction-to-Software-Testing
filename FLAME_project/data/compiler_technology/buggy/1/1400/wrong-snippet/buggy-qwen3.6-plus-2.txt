2.  **Root Cause Identification**:
    The discrepancy in line numbers for the first two errors suggests that the **Lexer** is incorrectly calculating the `lineNum` for tokens. Specifically, it seems to be skipping lines or incrementing the line counter incorrectly when handling comments or blank lines.

    Let's look at `TokenLexer.java`:
    *   The `paresToken` method iterates through `source` (lines read from file).
    *   It has logic to handle multi-line comments (`/* ... */`) and single-line comments (`//`).
    *   Inside the loop `for (String line : source)`, it increments `lineNum` at the end of the loop iteration: `lineNum++;`.
    *   However, there are `continue` statements inside the loop that skip the rest of the processing but *do not* skip the final `lineNum++`? No, `continue` jumps to the next iteration, so `lineNum++` at the bottom is executed.
    *   Let's trace the logic for blank lines and comments carefully.

    ```java
    public void paresToken() {
        readLines(); 
        lineNum = 1;
        boolean flag = false; 
        for (String line : source) {
            if (!flag) { 
                if (line.contains("/*")) {
                    flag = !line.contains("*/");
                    if (!flag) { 
                        line = removeDoubleComment(line); 
                    } else {
                        line = removeOnceComment(line); 
                    }
                } else { 
                    if (line.contains("//")) {
                        line = removeSingleComment(line);
                    }
                }
                if (isBlank(line)) {
                    lineNum++; // Increment here
                    continue;  // Skip to next line, skipping the bottom lineNum++
                }
            } else if (line.contains("*/")) {
                flag = false;
                line = removeEndComment(line); 
                if (isBlank(line)) {
                    lineNum++; // Increment here
                    continue;  // Skip to next line, skipping the bottom lineNum++
                }
            } else {
                lineNum++; // Increment here
                continue;  // Skip to next line, skipping the bottom lineNum++
            }
            // ... tokenization logic ...
            lineNum++; // Increment at the end of successful processing
        }
    }
    ```

    The issue lies in how `lineNum` is managed.
    - If a line is blank or fully commented out, `lineNum` is incremented inside the `if/else` blocks, and then `continue` is called. This means the `lineNum++` at the very end of the loop is **skipped**. This part seems correct for those cases (increment once per line).
    - However, look at the `else` block for `flag == true` (inside multi-line comment):
      ```java
      } else {
          lineNum++;
          continue;
      }
      ```
      This handles lines strictly inside a multi-line comment that do *not* contain `*/`. It increments `lineNum` and continues. Correct.

    Let's look at the `if (!flag)` block again.
    If `line.contains("/*")` is true, and it does NOT contain `*/` (so `flag` becomes true), it calls `removeOnceComment(line)`.
    `removeOnceComment` returns the substring before `/*`.
    Then it checks `if (isBlank(line))`. If the part before `/*` is blank, it increments `lineNum` and continues.
    If the part before `/*` is NOT blank, it proceeds to tokenize the remaining part. Then it hits the final `lineNum++`.
    
    Wait, let's look at `removeOnceComment`:
    ```java
    public String removeOnceComment(String line) {
        int commentStart = line.indexOf("/*");
        return line.substring(0, commentStart);
    }
    ```
    
    And `removeDoubleComment`:
    ```java
    public String removeDoubleComment(String line) {
        int commentStart = line.indexOf("/*");
        int commentEnd = line.indexOf("*/");
        return line.substring(0, commentStart) + line.substring(commentEnd + 2);
    }
    ```

    The logic seems generally okay for line counting *if* every line in `source` corresponds to exactly one line in the file. `readLines` reads line by line.
