    *   Why is it missing?
    *   Let's check `CompUnit.parseCompUnit()` or `Stmt.getStmt()`.
    *   The error `a` is a lexical error. The `Lexer` detects it and sets `flag = true` and adds to `wrongs`.
    *   In `Compiler.main`:
        ```java
        Lexer lexer = new Lexer();
        lexer.getFile();
        lexer.recognize();
        CompUnit compUnit = new CompUnit(lexer.getWords());
        compUnit.parseCompUnit();
        compUnit.printWords();
        ```
    *   `lexer.recognize()` populates `lexer.wrongs` and `lexer.flag`.
    *   `compUnit.printWords()` checks `WrongFile.getInstance().getFlag()`.
    *   **Crucial Point**: `Lexer` has its own `wrongs` map and `flag`. `WrongFile` is a separate singleton used by the *Parser* (syntax analysis) to report syntax errors (like missing semicolons).
    *   The `Lexer` reports lexical errors (like `a`) to its internal `wrongs` map.
    *   However, `CompUnit.printWords()` only checks `WrongFile.getInstance().getFlag()`. It does **not** check `lexer.flag` or merge `lexer.wrongs` into `WrongFile`.
    *   If `WrongFile` is empty (no syntax errors detected *before* the lexical error check matters, or if the parser doesn't trigger `WrongFile`), the lexical errors from `Lexer` are ignored in the final output because `printWords` relies solely on `WrongFile`.
    *   Wait, `Lexer.printWords()` handles lexical errors. But `Compiler` calls `compUnit.printWords()`, not `lexer.printWords()`.
    *   `CompUnit.printWords()`:
        ```java
        public void printWords() {
            if(WrongFile.getInstance().getFlag()) {
                WrongFile.getInstance().getPrint();
            } else {
                file.printWords();
            }
        }
        ```
    *   The `Lexer`'s errors are stored in `lexer.wrongs`, NOT in `WrongFile`.
    *   The `Parser` stores errors in `WrongFile`.
    *   Therefore, lexical errors (type `a`) are never printed because `WrongFile` doesn't know about them.
    *   To fix this, the lexical errors need to be transferred to `WrongFile`, or `CompUnit` needs to check `lexer`'s errors.
    *   Given the structure, the most likely intended fix is that `Lexer` should add its errors to `WrongFile` instead of (or in addition to) its local map, OR `Compiler` should merge them.
    *   Looking at `Lexer.addWords()`:
        ```java
        Wrong wrong = new Wrong("WRONGAND", posLine, "a");
        wrongs.put(posLine + 1, wrong); // Stores in local map
        flag = true;
        ```
    *   It should likely call `WrongFile.getInstance().addWrong(wrong)` instead.

    *   Let's verify if `WrongFile` is used elsewhere. Yes, in `ConstDecl`, `FuncDef`, etc., for syntax errors.
    *   So, `Lexer` is isolated from `WrongFile`.
    *   The error is in `Lexer.java` where it fails to register the lexical error with the global `WrongFile` singleton that `CompUnit` checks.
