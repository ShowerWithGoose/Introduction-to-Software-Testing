    Why would `cur` not be reset?
    
    ```java
    cur = peek;
    stmt.addChildNode(parseAssignStmt());
    ```
    
    This line is executed.
    
    Unless `peek` was modified?
    `peek` is a member variable.
    
    Inside `parseLVal`, is `peek` modified?
    No.
    
    Inside `parseExp`?
    No.
    
    Inside `parsePrimaryExp`?
    No.
    
    Inside `parseUnaryExp`?
    ```java
    peek = cur + 1;
    ```
    YES! `parseUnaryExp` modifies `peek`!
    
    Trace:
    1. `parseStmt` sets `peek = cur` (points to `x`).
    2. `parseStmt` calls `parseLVal()`.
    3. `parseLVal` consumes `x`.
    4. `parseLVal` sees `[`.
    5. `parseLVal` calls `parseExp()`.
    6. `parseExp` calls `parseAddExp()`.
    7. `parseAddExp` calls `parseMulExp()`.
    8. `parseMulExp` calls `parseUnaryExp()`.
    9. `parseUnaryExp` sees `i` (IDENFR).
    10. `parseUnaryExp` sets `peek = cur + 1`.
        At this point, `cur` points to `i`.
        So `peek` now points to the token AFTER `i`, which is `]`.
    11. `parseUnaryExp` checks `doPeek(peek)`. It is `]`. Not `(`.
    12. `parseUnaryExp` calls `parsePrimaryExp()`.
    13. `parsePrimaryExp` calls `parseLVal()` for `i`.
    14. `parseLVal` consumes `i`.
    15. Returns.
    16. `parseUnaryExp` returns.
    17. ... `parseExp` returns.
    18. Back in outer `parseLVal` (for `x`).
    19. `cur` is at `]`.
    20. `parseLVal` handles `]`.
    21. `parseLVal` returns.
    22. Back in `parseStmt`.
    23. `cur` is at `=` (because `parseLVal` consumed `]`).
        WAIT.
        In step 20, `parseLVal` consumes `]`.
        So `cur` is now at `=`.
    24. `parseStmt` executes `cur = peek`.
        But `peek` was modified by `parseUnaryExp` to point to `]`!
    25. So `cur` is set to `]`.
    26. `parseStmt` calls `parseAssignStmt()`.
    27. `parseAssignStmt` calls `parseLVal()`.
    28. `parseLVal` sees `]`.
    29. `parseLVal` creates `LeafNode(IDENFR, "]")`.
    
    This matches the bug!
    
    The error is that `peek` is a global/member variable, and `parseUnaryExp` modifies it, interfering with the `peek` saved in `parseStmt`.
