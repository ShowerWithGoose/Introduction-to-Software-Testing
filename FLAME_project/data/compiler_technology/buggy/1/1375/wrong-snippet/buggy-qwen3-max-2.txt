Based on the problem description and test case, the program should detect three missing semicolon errors (error code "i") at lines 2, 9, and 13. However, the buggy output shows "No output file", indicating that error detection and reporting is not working properly.

Looking at the test input:
- Line 2: `printf("Hello SysY!\n")` is missing a semicolon
- Line 9: `int a` is missing a semicolon  
- Line 13: `return 0` is missing a semicolon

The issue appears to be in the statement parsers that handle these constructs. Specifically, the `StmtPrintParser`, `VarDeclParser`, and `StmtReturnParser` classes are responsible for detecting missing semicolons in printf statements, variable declarations, and return statements respectively.

Examining the code, I can see that in several parser classes, when a semicolon is missing, they create an error token with content "i" but use `TokenType.SEMICN` as the token type instead of a proper error token type. This might cause issues in the error handling pipeline.

However, the most critical issue is in the main `Compiler.java` file where the output logic is flawed. The program only writes to `parser.txt` if parsing succeeds, and only writes to `error.txt` from lexer errors, but doesn't handle parser-detected syntax errors properly.
