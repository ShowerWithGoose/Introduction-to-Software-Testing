            ```java
}
            ```

        Let's re-read `StmtReturnParser.parseStmtReturn()`:
        ```java

            this.semicn = this.iterator.readNextToken(); // Reads '0' ? No.
        ```

        So, for `return 0 }`, it parses `exp` = `0`, and `semicn` = `}`. It does **not** report an error for the missing semicolon here. It just consumes `}` as the semicolon token in the AST node.
        
        Then control returns to `BlockParser` which called `StmtParser` (via `BlockItemParser`).
        `BlockParser` loop:
        ```java
        Token token = this.iterator.readNextToken(); // Expects next token after Stmt
        while (!token.type.equals(TokenType.RBRACE)) {
           // ...
        }
        ```
        The iterator is now at EOF or after `}`?
        In `StmtReturnParser`, we read `}` as `semicn`. The iterator is now past `}`.
        So `BlockParser` reads next token. If `}` was the last token before EOF, `readNextToken` might fail or return null/EOF token.
        
        Actually, if `StmtReturnParser` consumed `}`, the `BlockParser` will try to read the next token. If there are no more tokens, `iterator.hasNext()` in `TokenListIterator` might return false, or `readNextToken` throws exception.
        
        However, the main issue is that **Error 3 (missing semicolon after `return 0`) is not detected**.
        
        Why? Because `StmtReturnParser` assumes that if the first token after `return` is not `;`, it must be an `Exp`. It parses the `Exp`, then reads the *next* token and blindly assigns it to `semicn` without checking if it is `;`. If it's not `;` (e.g., `}`), it should report an error.

        Let's verify the other errors.
        Error 1: `printf(...)`. `StmtPrintParser` checks for `;`. If not found, reports error. Correct.
        Error 2: `int a`. `VarDeclParser` checks for `;`. If not found, reports error. Correct.
        Error 3: `return 0`. `StmtReturnParser` does NOT check for `;` after parsing `Exp`.

        Therefore, the bug is in `StmtReturnParser.java`.
