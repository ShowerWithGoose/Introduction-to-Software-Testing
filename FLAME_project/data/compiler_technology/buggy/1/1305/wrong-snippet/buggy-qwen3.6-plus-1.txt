Based on the comparison between the buggy output and the expected output, the error occurs when parsing the function call `func_p()` inside the `printf` statement.

In the **Buggy Output**:
```text
COMMA ,
LPARENT (
RPARENT )
<UnaryExp>
...
```
The parser outputs `LPARENT` and `RPARENT` directly, skipping the identifier `func_p`. This indicates that the parser recognized a function call structure but failed to consume or output the function name token (`IDENFR`).

In the **Expected Output**:
```text
COMMA ,
IDENFR func_p
LPARENT (
RPARENT )
<UnaryExp>
...
```
The identifier `func_p` is correctly output before the parentheses.

Looking at the `unaryExpAnalyze` method in `Frontend/Parser.java`:
```java
public UnaryExpNode unaryExpAnalyze(){

        Token Ident = findToken(TokenTypes.IDENFR); // Consumes IDENFR
        Token leftParent = findToken(TokenTypes.LPARENT); // Consumes LPARENT
        FuncRParamsNode funcRParamsNode = null;
        if(currentToken.type != TokenTypes.RPARENT){
            funcRParamsNode = funcRParamsAnalyze();
        }
        Token rightParent = findToken(TokenTypes.RPARENT); // Consumes RPARENT
        return new UnaryExpNode(Ident, leftParent, rightParent, funcRParamsNode);
    }

}
```
The parsing logic correctly consumes the `IDENFR` token into the variable `Ident`.

public void print() throws IOException {
