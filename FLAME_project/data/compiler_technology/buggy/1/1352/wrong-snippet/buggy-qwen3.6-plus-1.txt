        **The Fix:**
        The Lexer should **not** write to `error.txt` directly. It should only collect errors. The final error output should be handled centrally by `IOUtils.writeError()` after parsing is complete, ensuring all errors (lexical and syntactic) are collected, sorted, and written once.

        Alternatively, `Lexer.printLexicalResult()` should not write errors if `isParser` is true, or should not write to `error.txt` at all, leaving it to the central handler.

        Given the structure, the most likely intended fix is to prevent `Lexer` from writing to `error.txt` when syntax analysis is also running, or to clear the file before the final write. But since `IOUtils.writeError` appends, and `Lexer` wrote first, we get duplication and bad ordering.

        The specific code snippet causing the issue is in `Lexer.java`'s `printLexicalResult` method writing to `error.txt` prematurely, and/or `IOUtils.writeError` appending instead of overwriting (or `Lexer` not clearing).

        However, usually, in these compilers, if `isParser` is true, `Lexer` shouldn't output errors to `error.txt` directly.
