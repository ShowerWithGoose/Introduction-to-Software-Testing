        **The Error**:
        1. `Lexer.lexicalAnalysis()` calls `printLexicalResult()`, which writes lexical errors to `error.txt`.
        2. `Lexer` also adds these errors to `IOUtils.compileErrors`.
        3. `Parser` adds syntax errors to `IOUtils.compileErrors`.
        4. `Parser.printSyntaxResult()` calls `IOUtils.writeError()`, which sorts `IOUtils.compileErrors` and APPENDS them to `error.txt`.
        
        This results in duplicate errors (once from Lexer, once from Parser's append) and incorrect ordering (Lexer's output is unsorted relative to Parser's).
        
        **Fix**:
        The Lexer should NOT write to `error.txt` directly if the Parser is going to handle all error reporting via `IOUtils.compileErrors`. Or, `IOUtils.compileErrors` should be cleared before Parser starts, and Lexer should not write to file.
        
        Given the structure, the most likely intended fix is to prevent `Lexer` from writing to `error.txt` when `isParser` is true, or to ensure `IOUtils.compileErrors` is the single source of truth and written only once at the end.
