The issue is in the `parseStmt` method in the `printf` handling section. When parsing printf arguments, the code calls `parseExp()` but doesn't add it as a child to the node, which means the parsed expression tree isn't properly built and the print statements for the non-terminals aren't executed.

Looking more closely at the `parseStmt` method, I can see the problem is in the printf handling section:

```java
while (curType == LexType.COMMA) {
    node.addChild(new GrammarNode(","));
    getWord();
    parseExp();  // <-- This is the bug! Should be node.addChild(parseExp());
}
```

The `parseExp()` call should be `node.addChild(parseExp())` to properly build the AST and trigger the print statements.
