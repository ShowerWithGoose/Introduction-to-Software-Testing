The condition `tokens.get(index + 1).getType() != TokenType.MAINTK && tokens.get(index + 2).getType() != TokenType.LPARENT` is risky. It assumes that if the next token is not `main` AND the token after that is not `(`, it's a `Decl`.
For `int ig1;`, `index` points to `int`. `index+1` is `ig1` (IDENFR). `index+2` is `;` (SEMICN). This works.
For `int fuc1(){...}`, `index` points to `int`. `index+1` is `fuc1` (IDENFR). `index+2` is `(` (LPARENT). The condition `tokens.get(index + 2).getType() != TokenType.LPARENT` is FALSE. So the loop terminates. This means `int fuc1()` is NOT parsed as a `Decl`. It falls through to the `FuncDef` loop.
In the `FuncDef` loop: `while (tokens.get(index + 1).getType() != TokenType.MAINTK)`.
Current token is `int`. `index+1` is `fuc1`. Not `MAINTK`. So it calls `funcDef()`.
`funcDef()` parses `FuncType Ident '(' ... ')' Block`.
This seems correct for distinguishing Decls and FuncDefs based on the lookahead.
