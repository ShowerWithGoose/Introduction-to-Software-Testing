Looking at the test case input and expected output, I need to identify where the parser should detect missing right parentheses (error 'j').

Test Case Input:
```
void f(int a {} // error 1
int main( { // error 2
    int a = 10;
    f(a + 10; // error 3
    return 0;
}
```

Expected Output:
```
1 j
3 j
5 j
```

The errors are:
1. Line 1: Missing ')' in function parameter list `void f(int a {}`
2. Line 3: Missing ')' in main function declaration `int main( {`
3. Line 5: Missing ')' in function call `f(a + 10;`

Looking at the parser code, I can see issues in the `parseFuncDef` method and potentially in expression parsing for function calls.

In the `parseFuncDef` method, there's problematic logic for handling function parameters and detecting missing right parentheses:

    public FuncDef parseFuncDef() {
        FuncDef funcDef = new FuncDef(tokens.next(), tokens.next());
        funcDef.setLparent(tokens.next());//(
        if (!tokens.peek().is(TokenType.RPARENT)) {
            if (tokens.peek().is(TokenType.LBRACE)) {
                //error j
                addError("j");
                funcDef.setBlock(parseBlock());
                return funcDef;
            } else {
                funcDef.setFuncFParams(parseFuncFParams());
            }
        }
        //error j
        if (!tokens.peek().is(TokenType.RPARENT)) {
            addError("j");
        } else {
            funcDef.setRparent(tokens.next());
        }
        funcDef.setBlock(parseBlock());
        return funcDef;
    }
...

The issue is that when encountering `void f(int a {}`, the parser sees `LBRACE` after the parameter list and adds error 'j', but then continues parsing without properly handling the missing `RPARENT`. Similarly, for `int main( {`, it should detect the missing `RPARENT` before the `LBRACE`.
