In `Lexer::next_assert`, when a token is missing (e.g., missing semicolon, parenthesis, or bracket), the code creates an error object and returns a *newly created* Token object to allow parsing to continue. This causes an infinite loop or incorrect state because the lexer stays at the same position, and subsequent calls to `next_assert` or `next` will likely fail or behave unexpectedly because the "virtual" token isn't part of the stream logic properly handled by `cur_index`.
