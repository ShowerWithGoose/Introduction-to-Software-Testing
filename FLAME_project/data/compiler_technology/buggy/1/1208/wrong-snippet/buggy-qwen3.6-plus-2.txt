     ```java
     public Token look(int num) {
         if (curPos + num >= length) {
             return null;
         }
         return tokens.get(curPos + num);
     }
     ```
     
     If `curPos` points to the NEXT token (i.e., `next()` increments `curPos` then gets token)?
     
     ```java
     public void next() {
         if (curPos >= length) {
             curToken = null;
         } else {
             curToken = tokens.get(curPos++);
         }
     }
     ```
     `curPos` is incremented AFTER getting the token.
     So `curToken` is `tokens.get(old_curPos)`.
     `curPos` now points to the NEXT token.
     
     So `look(0)` returns `tokens.get(curPos + 0)` which is the NEXT token.
     
     AHA!
     
     If `next()` has just been called, `curToken` is the current token, but `curPos` is the index of the NEXT token.
     
     So `look(0)` returns the token AFTER `curToken`.
     
     Therefore, `if (look(0).getType() == LexType.LPARENT)` checks if the NEXT token is `(`.
     
     If `curToken` is `IDENFR func_p`, the next token IS `LPARENT`.
     So the condition is TRUE.
     The code ENTERS the function call block.
     
     So the logic for entering the block is CORRECT given the `curPos` behavior.
     
     So why is the output wrong?
     
     Because `parseFuncRParams` is called unconditionally.
     
     ```java
                 unaryExp.addChildNode(parseFuncRParams());
     ```
     
     For `func_p()`, `curToken` is now `LPARENT`? No.
     Inside the block:
     1. `createEndNode()` consumes `IDENFR`. `curPos` advances. `curToken` becomes `LPARENT`.
     2. `createEndNode()` consumes `LPARENT`. `curPos` advances. `curToken` becomes `RPARENT`.
     3. `parseFuncRParams()` is called.
     
     Inside `parseFuncRParams`:
     `curToken` is `RPARENT`.
     `funcRParams.addChildNode(parseExp());`
     
     `parseExp` tries to parse `RPARENT` as an expression.
     It fails to match terminals.
     It returns an empty/partial tree.
     
     This results in the extra `<UnaryExp><MulExp>...` nodes in the output, and the `<FuncRParams>` node itself.
     
     The Expected Output does NOT have `<FuncRParams>` for empty params.
     
     So the error is that `parseFuncRParams` is called even when empty.
     
     Fix:
     ```java
                 if (curToken.getType() != LexType.RPARENT) {
                     unaryExp.addChildNode(parseFuncRParams());
                 }
     ```
     
     This matches the pattern used in `parseFuncDef`:
     ```java
         if (curToken.getType() != LexType.RPARENT) {
             funcDef.addChildNode(parseFuncFParams());
         }
     ```
     
     So the error location is in `Parser.java`, inside `parseUnaryExp`.
