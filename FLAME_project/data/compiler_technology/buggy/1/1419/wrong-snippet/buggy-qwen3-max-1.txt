Based on the test case and expected output, the program should detect three missing semicolon errors (error type 'i') at lines 2, 9, and 13. The issue appears to be in the parser's error handling mechanism, specifically in how it detects and reports missing semicolons.

Looking at the code, the problem is in the `expect` method in `Parser.java`. When a semicolon is expected but not found, it creates a synthetic SEMICN token and reports an error. However, the line number used for the error is `tokenList.get(idx - 1).getLine()`, which may not be correct in all cases.

More critically, examining the `parseStmt` method, when parsing an expression statement (`[Exp] ';'`), if there's no semicolon, the error should be reported at the line of the last token in the expression. But the current implementation doesn't properly handle this case.

The main issue is in the `expect` method's handling of SEMICN tokens and how the parser recovers from missing semicolons.
