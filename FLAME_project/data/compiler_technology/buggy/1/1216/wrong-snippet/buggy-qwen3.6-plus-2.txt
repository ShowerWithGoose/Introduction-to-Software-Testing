The logic in `parseCompUnit` consumes tokens until `reachEnd()`. It should only run once.

Let's re-examine `Compiler.java`. Is it possible that `Test.java` is being run? No, `Compiler` is the main class specified.

Is it possible that `CompUnit.toString()` is iterating over `functions` AND `mainFuncDef` separately, and `mainFuncDef` is also inside `functions`?

In `CompUnit`:
```java
    public void setMainFunc(FuncDef mainFunc) {
        this.functions.add(mainFunc); // Adds to functions list
        this.mainFuncDef = mainFunc;  // Sets mainFuncDef reference
    }
```

In `toString()`:
```java
        for (FuncDef func: functions) {
            res.append(func);
        }
        res.append(mainFuncDef); // Appends mainFuncDef AGAIN
```

Since `setMainFunc` adds the main function to the `functions` list, the loop `for (FuncDef func: functions)` will print the main function. Then `res.append(mainFuncDef)` prints it **again**. This causes the duplication of the main function's output. Since the test case only has `main`, the entire output is effectively duplicated (plus the `<CompUnit>` tag appearing twice if the structure implies it, but here `<CompUnit>` is appended at the very end).

Yes, the `MainFuncDef` is printed twice. Once in the loop over `functions`, and once explicitly via `mainFuncDef`.

The error is in `CompUnit.toString()`. It should not append `mainFuncDef` separately if it's already included in the `functions` list, OR `setMainFunc` should not add it to `functions` if it's handled separately. Given the grammar `CompUnit → {Decl} {FuncDef} MainFuncDef`, it distinguishes between regular `FuncDef`s and `MainFuncDef`.

If the intention is to print all functions in the list and then the main function, `setMainFunc` should probably NOT add it to `functions`, or `toString` should exclude `mainFuncDef` from the `functions` loop.

However, usually `functions` holds non-main functions and `mainFuncDef` holds the main one.
In `Parser.parseCompUnit`:
```java
                        compUnit.setMainFunc(FuncParser.parseFuncDef());
```
vs
```java
                        compUnit.addFunc(FuncParser.parseFuncDef());
```

If `setMainFunc` adds it to `functions`, then `functions` contains the main function.
The `toString` iterates `functions` (printing main) and then appends `mainFuncDef` (printing main again).

To fix this, we should either:
1. Change `setMainFunc` to NOT add to `functions`.
2. Change `toString` to not append `mainFuncDef` if it's already in `functions` (but this breaks ordering if `main` isn't last in `functions`).
3. Change `toString` to iterate only non-main functions in the loop, or handle the separation correctly.

Given the grammar structure `{FuncDef} MainFuncDef`, it's cleaner if `functions` list contains only non-main functions, and `mainFuncDef` is separate.

Let's look at `setMainFunc`:
```java
    public void setMainFunc(FuncDef mainFunc) {
        this.functions.add(mainFunc);
        this.mainFuncDef = mainFunc;
    }
```

If we remove `this.functions.add(mainFunc);`, then the loop `for (FuncDef func: functions)` will only print non-main functions, and `res.append(mainFuncDef)` will print the main function. This matches the expected structure.
