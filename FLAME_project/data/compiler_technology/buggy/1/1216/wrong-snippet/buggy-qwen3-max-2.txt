Looking at the problem description and comparing the buggy output with the expected output, I can see that the buggy output contains **duplicate content** - it appears to be printing the entire parse tree twice, followed by `<CompUnit>`.

The expected output shows the correct sequence once, ending with `<CompUnit>`, but the buggy output shows the same sequence repeated twice, then `<CompUnit>`.

Looking at the `CompUnit.toString()` method in the source code, I can see the issue:

```java
@Override
public String toString() {
    StringBuilder res = new StringBuilder();
    for (Decl decl: globalDecls) {
        res.append(decl);
    }
    for (FuncDef func: functions) {
        res.append(func);
    }
    res.append(mainFuncDef);
    res.append("<CompUnit>\n");
    return res.toString();
}
```

The problem is that `mainFuncDef` is already included in the `functions` list (as seen in the `setMainFunc` method which does `this.functions.add(mainFunc)`), so when the code iterates through `functions` and then separately appends `mainFuncDef`, it's printing the main function twice.
