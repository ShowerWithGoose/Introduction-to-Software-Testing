Looking at the problem description and comparing the buggy output with the expected output, I can see that the buggy output contains **duplicate content** - it appears to be printing the entire parse tree twice, followed by `<CompUnit>`.

The expected output shows the parse tree once, ending with `<CompUnit>`, but the buggy output shows the same content repeated twice before `<CompUnit>`.

Looking at the `CompUnit.toString()` method in the source code, I can see the issue:

<frontend/parser/CompUnit.java>
...
    @Override
    public String toString() {
        StringBuilder res = new StringBuilder();
        for (Decl decl: globalDecls) {
            res.append(decl);
        }
        for (FuncDef func: functions) {
            res.append(func);
        }
        res.append(mainFuncDef);
        res.append("<CompUnit>\n");
        return res.toString();
    }
...

The problem is that `mainFuncDef` is already included in the `functions` list (as seen in the `setMainFunc` method which adds it to `functions`), so when the `toString()` method iterates through `functions` and then separately appends `mainFuncDef`, it's duplicating the main function output.

In the `setMainFunc` method:
```java
public void setMainFunc(FuncDef mainFunc) {
    this.functions.add(mainFunc);  // mainFuncDef is added to functions list
    this.mainFuncDef = mainFunc;
}
```

Then in `toString()`:
```java
for (FuncDef func: functions) {  // This includes mainFuncDef
    res.append(func);
}
res.append(mainFuncDef);  // This duplicates mainFuncDef
```

This causes the entire function list (including main) to be printed, then main is printed again separately, leading to the duplication observed in the output.
