The issue is likely in how `CompUnit` stores the main function.
In `Parser.parseCompUnit`, when `main` is detected:
`compUnit.setMainFunc(FuncParser.parseFuncDef());`

It adds the main function to the `functions` list AND sets `mainFuncDef`.

```java

```

It iterates through `functions` (which includes the main function because `setMainFunc` added it) and appends each. Then it **also** appends `mainFuncDef`. This causes the main function definition to be printed twice.

The expected output shows `<MainFuncDef>` only once at the end, followed by `<CompUnit>`.
The buggy output shows the entire sequence of tokens and non-terminals for the main function twice.

Therefore, the error is in `CompUnit.java` in the `setMainFunc` method or the `toString` method. The standard pattern for these compilers is usually that `functions` contains non-main functions, and `mainFuncDef` contains the main function. Or `functions` contains all functions and `mainFuncDef` is just a reference, but `toString` should handle it correctly.

If `setMainFunc` adds to `functions`, then `toString` should not iterate `functions` and then append `mainFuncDef` separately if `mainFuncDef` is already in `functions`. Alternatively, `setMainFunc` should not add to `functions`.

This implies MainFuncDef is separate from the list of FuncDefs.

So `functions` should probably only contain non-main `FuncDef`s.
