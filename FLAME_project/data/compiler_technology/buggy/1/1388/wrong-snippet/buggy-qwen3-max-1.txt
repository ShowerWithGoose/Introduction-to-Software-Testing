```java
private GramNode BlockItem() {
    // BlockItem → Decl | Stmt
    if(curPos >= tokens.size() - 1 ){
        return null;
    }

    int startPos = curPos;
    GramNode blockItem = new GramNode();
    blockItem.gramType = GramType.BlockItem;

    GramNode child = Decl();
    if(child != null){
        blockItem.children.add(child);
    } else {
        child = Stmt();
        if (child != null) {
            blockItem.children.add(child);
        } else {
            curPos = startPos;
            return null;
        }
    }

    child.parent = blockItem;

    System.out.println("blockItem");
    return blockItem;
}
```

The problem is that when `Decl()` or `Stmt()` fails completely (returns null), it resets `curPos` to `startPos` and returns null. However, in cases where `Decl()` or `Stmt()` partially succeeds but encounters a missing semicolon error, they return a valid node but the parser may not advance `curPos` correctly to allow parsing the next `BlockItem`.

Looking at the `VarDecl()` method again, when it encounters a missing semicolon, it adds the error and returns the `varDecl` node, but it doesn't advance `curPos` past the point where the semicolon should be. This means the next token is still whatever came after the missing semicolon, which could cause parsing issues for subsequent statements.
