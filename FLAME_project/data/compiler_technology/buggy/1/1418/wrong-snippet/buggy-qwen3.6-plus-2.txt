Based on the problem description and the test case, the program fails to output any error file (`error.txt`) when it encounters syntax errors (missing semicolons). The expected output indicates three "missing semicolon" errors (type `i`) at lines 2, 9, and 13.

The root cause lies in how the `Compiler.java` handles the flow between lexical analysis, parsing, and error output. Specifically:

1.  In `Compiler.java`, if `GlobalData.Errorflag` is true (which happens when `ErrorSave.error` is called), the code block that writes to `parser.txt` is skipped because of the `if(GlobalData.Parserflag)` check (since `ErrorSave` sets `Parserflag` to false).
2.  However, the critical issue is that the `ErrorPrint.run(GlobalData.Errors)` is only called if `GlobalData.Errorflag` is true.
3.  Looking at `Lexer.java`, when a single `&` or `|` is encountered, it calls `ErrorSave.error`. This sets `GlobalData.Errorflag = true` and `GlobalData.Lexerflag = false`.
4.  In `Compiler.java`, after the lexer runs, if `GlobalData.Lexerflag` is false, it still proceeds to run the parser.
5.  The parser (`Parser.java`) runs `CompUnitParser`. If the parser encounters an error (like missing semicolon), it calls `ErrorSave.error("Parser", ...)` which sets `GlobalData.Errorflag = true` (already true) and `GlobalData.Parserflag = false`.
6.  The `Compiler.java` then checks `if(GlobalData.Parserflag)` to write `parser.txt`. Since it's false, it skips writing `parser.txt`.
7.  Then it checks `if(GlobalData.Errorflag)` to write `error.txt`. This should be true.
