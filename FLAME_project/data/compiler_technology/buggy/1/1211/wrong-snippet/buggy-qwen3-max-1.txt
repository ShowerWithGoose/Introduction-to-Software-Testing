Based on the problem description and test case, the program should detect three missing semicolon errors (error type 'i') on lines 2, 9, and 13. The issue appears to be in the error handling logic for missing semicolons.

Looking at the test input:
- Line 2: `printf("Hello SysY!\n")` - missing semicolon
- Line 9: `int a` - missing semicolon  
- Line 13: `return 0` - missing semicolon

The error detection for missing semicolons is handled by the `handleIError()` method in the Parser class. The problem is that this method uses `getPrevTkLineno()` which returns the line number of the previous token, but according to the requirements, "the error line number corresponds to the line where the last non-terminal before the semicolon is located."

However, examining the parsing methods more closely, the real issue is in the `parseStmt()` method's handling of the default case. When encountering an unexpected token that doesn't match any statement pattern, it creates a `NullStmt` with `handleIError()`, but this doesn't properly handle cases where a statement is incomplete (like missing semicolon after printf, variable declaration, or return).

The main issue is in the statement parsing logic where incomplete statements aren't being properly detected and reported with the correct line numbers.
