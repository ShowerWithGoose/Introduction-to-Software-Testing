        If we see `CHARTK`, we need to look ahead.
        If `CHARTK` is followed by `IDENFR` and then `(`, it's a `FuncDef`.
        If `CHARTK` is followed by `IDENFR` and then `[` or `=` or `,` or `;`, it's a `Decl`.
        
        The current code in `parseCompUnit` for the `Decl` loop:

        This blindly assumes any `CHARTK` starts a `FuncDef`.

        The fix requires looking ahead when `currToken` is `INTTK` or `CHARTK` to decide if it's a `Decl` or `FuncDef`.
        
        Specifically:
        If `currToken` is `INTTK` or `CHARTK`:
