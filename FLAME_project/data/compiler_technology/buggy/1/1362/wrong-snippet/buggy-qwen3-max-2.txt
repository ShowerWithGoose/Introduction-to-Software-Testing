But the `assertToken` method already adds the RPARENT component internally. So this results in RPARENT being added twice.

Looking at the `assertToken` method again:
- It calls `parseSupporter.checkToken(type)` 
- Then if type is RPARENT, it calls `parseSupporter.addComponent("RPARENT" + " " + ...)`

Then back in `parseFuncDef`, it calls `parseSupporter.addComponent("RPARENT" + " " + ...)` again.

This double-addition causes the extra RPARENT in the output.
