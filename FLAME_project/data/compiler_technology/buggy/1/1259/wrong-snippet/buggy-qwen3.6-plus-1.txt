Based on the problem description and the provided code, the issue is that the program produces **no output file** (`parser.txt` or `error.txt`) for a correct input program.

Looking at `Compiler.java`:
1. It runs the Lexer to tokenize the input.
2. It runs the Parser (`parser.Process()`).
3. It runs the Visitor (`visitor.Process()`).
4. It checks `ErrorManager.GetInstance().isError()`.
   - If there are errors, it writes to `error.txt`.
   - If there are **no errors**, it writes to `symbol.txt` by calling `symbolManager.PrintSymbol()`.

The expected output for a correct program is the syntax analysis result (tokens and non-terminals) written to `parser.txt`. However, the `Compiler.java` main method **never calls the code to print the parse tree** when there are no errors. It only prints symbols. The code to print the tree exists in `Parser.java` (`PrintTree()`) and `GrammarTreeNode.java` (`PrintFromRoot`), which uses `IOManager.Write` to output to the current writer. But `Compiler.java` initializes the writer for `symbol.txt` instead of `parser.txt`, and doesn't invoke the tree printing logic.

Specifically, in the `else` block of `Compiler.java`:
```java
}else{
    IOManager.InitIOWriter(new BufferedWriter(new FileWriter("symbol.txt")));
////                lexer.PrintTokens(bufferedWriter);
////                parser.PrintTree();
    symbolManager.PrintSymbol();
}
```

The lines `lexer.PrintTokens` and `parser.PrintTree` are commented out. Even if they weren't, the file is named `symbol.txt`, but the problem requires `parser.txt` for correct programs. Furthermore, the `Visitor` phase runs before this check, but the Visitor doesn't seem to produce the required output format either (it handles semantic checks and symbol table construction). The required output (tokens and non-terminals in order) is generated by `GrammarTreeNode.DFS` which calls `IOManager.Write`.

Therefore, the error is in `Compiler.java` where it fails to initialize the writer for `parser.txt` and call `parser.PrintTree()` (or equivalent) for correct programs.
