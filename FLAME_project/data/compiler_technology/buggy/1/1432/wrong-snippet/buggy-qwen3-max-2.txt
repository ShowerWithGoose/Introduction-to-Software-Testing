Looking at the lexer code, the `getLogicalOperator()` method handles '&' and '|' tokens. It currently only adds error "a" when a single '&' or '|' is found, but it still returns a token (either AND/OR with single char or double char). However, the issue is that for a single '&', it logs error "a" but then tries to return an AND token with just "&", which is incorrect.

Also examining the parser's Stmt class, in the printf handling section, it doesn't properly handle the error detection for missing right parenthesis.

The main issues appear to be:

1. In Lexer.java - getLogicalOperator() method incorrectly handles single '&'/'|'
2. In Stmt.java - printf statement parsing doesn't correctly detect missing ')'
