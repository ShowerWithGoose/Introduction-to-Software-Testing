    If `isAnnotation` is true at the start of a line, it processes it.
    
    Is it possible `isAnnotation` gets stuck?

    Let's look at `CompUnit()` again.
    
    ```java
    while (!peek(2).getType().equals(TokenType.LPARENT)) {
        //CompUnit → {Decl}
        Node tmp = Decl();
        result.addChild(tmp);
    }
    ```
    
    If `Decl()` fails to advance (e.g. error), this loop will be infinite.
    
    When does `Decl()` fail to advance?
    `Decl()` calls `ConstDecl()` or `VarDecl()`.
    `VarDecl()` calls `BType()`.
    `BType()` calls `matchToken(INTTK, CHARTK)`.
    If current token is NOT `INTTK` or `CHARTK`, `matchToken` fails.
    `BType()` returns fake node. Index does NOT advance.
    `VarDecl()` continues. Calls `VarDef()`.
    `VarDef()` calls `matchToken(IDENFR)`.
    If current token is NOT `IDENFR`, `matchToken` fails.
    `VarDef()` returns.
    `VarDecl()` returns.
    `Decl()` returns.
    
    So if `Decl()` is called with a token that is not `CONSTTK`, `INTTK`, `CHARTK`, it will NOT advance.
    
    In `CompUnit()`, the loop condition is `!peek(2).equals(LPARENT)`.
    If `Decl()` doesn't advance, `peek(2)` remains the same.
    Infinite loop.

    But `BlockItem()` protects `Decl()` by checking:
    ```java
    if(getCurToken().getType().equals(TokenType.INTTK)||getCurToken().getType().equals(TokenType.CHARTK)||getCurToken().getType().equals(TokenType.CONSTTK)){
        node = Decl();
    }
    ```
    So `Decl()` is only called if it CAN advance (at least `BType` will match).

    However, `CompUnit()` calls `Decl()` WITHOUT this check!
    
    ```java
    while (!peek(2).getType().equals(TokenType.LPARENT)) {
        //CompUnit → {Decl}
        Node tmp = Decl();
        result.addChild(tmp);
    }
    ```
    
    If the input starts with something that is NOT a `Decl` (e.g. `MAINTK`), `peek(2)` check prevents entry.
    
    But what if there is a syntax error in a `Decl` that causes `Decl()` to not advance?
    As established, `Decl()` enters `VarDecl()` -> `BType()`.
    `BlockItem()` ensures `BType()` will match `INTTK`/`CHARTK`.
    So `BType()` advances.
    So `Decl()` advances.

    So `CompUnit()` loop is safe IF `BlockItem()` logic is consistent.
