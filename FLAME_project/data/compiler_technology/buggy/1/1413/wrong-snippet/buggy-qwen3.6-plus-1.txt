So `UnaryExpParser` FAILS to report error `j`. It incorrectly accepts `;` as the closing token for the function call parameters/parentheses.

Then control returns to `StmtExpParser`.
`Token token=iterator.readNextToken();`

The input is `f(a + 10;`.
After `;`, there might be a newline or the next statement `return 0;`.
If the next token is `return` (RETURNTK), then:
`if(token.getValue().equals(";"))` is FALSE.
`else { iterator.unReadToken(1); ErrorP errorP=new ErrorP("i",this.errorLine); ... }`
This reports error `i` (missing semicolon) at `errorLine` (which was set to 5 in `StmtExpParser`).

This matches the buggy output: `5 i`.

**Conclusion:**
The error is in `UnaryExpParser.java`. When parsing function arguments, if the closing parenthesis `)` is missing, the parser consumes the subsequent token (in this case `;`) and assigns it to `rightParenthesis` without checking if it is actually `)`. It should check if `next` is `)` and report error `j` if it is not.

The specific location is in `Parser/Exp/UnaryExpParser.java`, inside the `else if` branch handling function parameters.
