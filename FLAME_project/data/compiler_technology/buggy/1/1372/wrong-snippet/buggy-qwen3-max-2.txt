Based on the test case input and expected vs actual output, the issue is with error line number reporting for missing right parentheses (error type 'j'). The program reports lines 1, 4, 7 but should report 1, 3, 5.

Looking at the test input:
```
void f(int a {} // error 1 (line 1)
              ^
              Missing ) - should be reported as line 1 ✓

int main( { // error 2 (line 3)
         ^
         Missing ) - should be reported as line 3 ✗ (program reports line 4)

    int a = 10;
    f(a + 10; // error 3 (line 5)
           ^
           Missing ) - should be reported as line 5 ✗ (program reports line 7)
```

The problem is in how `getErrLine()` determines the error line number. It returns `tokens.get(index-1).getLineNum()`, but when we're expecting a terminal that's missing, the current token might not be the one that should determine the error line.

Looking at the `handleTerminal` method in Parser.java, when a required terminal is missing, it calls `tokens.getErrLine()` which uses the previous token's line number. However, for missing ')' errors, we should report the line number of the last successfully parsed token in the current context, not necessarily the immediately preceding token.
