Based on the test case and expected vs actual output, the issue is that the program is missing detection of a missing semicolon on line 13 (`return 0 // error 3`). The parser correctly identifies missing semicolons in some cases but fails to detect it for return statements without semicolons at the end of a block.

Looking at the code, the problem appears to be in the `parseReturnStmt` method in `StmtParser.java`. When parsing a return statement with an expression, it checks if the next token is a semicolon, but doesn't properly handle the error case when it's missing.

<frontend/Parser/Stmt/StmtParser.java>
...
    // 'return' [Exp]
    public ReturnStmt parseReturnStmt() {
        LexType returnToken = tokens.getAndMove();
        if (tokens.getToken().getType() == LexType.TokenType.SEMICN) {
            return new ReturnStmt(returnToken);
        } else if (!tokens.getToken().isFirstExp()) {
            errorLog.add(tokens.getlastToken(1).getLine(), 'i');
            return new ReturnStmt(returnToken);
        } else {
            return new ReturnStmt(returnToken, new ExpParser(tokens, errorLog).parserExp());
        }
    }
