The issue is that the class name `Char` is being used instead of `Character` for the non-terminal symbol. Looking at the grammar, the correct non-terminal name should be `<Character>`, not `<Char>`.

Examining the code, I can see that in the `Char.java` file, the class is named `Char`, which causes the toString() method in the Symbol class to output `<Char>` instead of `<Character>`.

```java

```

Since the class is named `ast.Char`, `getClass().getName().substring(4)` gives `"Char"`, but it should give `"Character"`.
