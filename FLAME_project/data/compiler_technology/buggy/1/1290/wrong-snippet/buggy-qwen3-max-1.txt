Looking at the problem description and comparing the buggy output with the expected output, I can see the issue is in the character handling.

In the buggy output, for character constants like `'a'` and `'b'`, the output shows:
```
<Char>
<PrimaryExp>
```

But in the expected output, it should show:
```
<Character>
<PrimaryExp>
```

The issue is that the `Char` class in the AST is using the wrong name. According to the grammar, the non-terminal for character constants should be `<Character>`, not `<Char>`.

Looking at the code, I can see that in `ast/Char.java`, the class is named `Char`, which when converted to a string via the `Symbol.toString()` method (which uses `getClass().getName().substring(4)`), produces `<Char>` instead of the expected `<Character>`.

The grammar defines:
- `Character → CharConst`

So the class should be named `Character` to produce the correct output `<Character>`.

<ast/Char.java>
package ast;

public class Char extends Symbol {

    public static Symbol of() {
        Symbol character = new Char();
        Symbol symbol;
        if ((symbol = CharConst.of()) == null) {
            return null;
        }
        character.add(symbol);
        return character;
    }

}
