The error lies in the `ErrorLog.java` file. The `WriteErrorLogs` method writes the errors to the file in the order they were added to the list. However, the problem requires the errors to be sorted by line number in ascending order. The current implementation does not sort the `errorlog` list before writing, which leads to incorrect output order when errors are detected out of sequence (e.g., error on line 18 detected before error on line 16).
