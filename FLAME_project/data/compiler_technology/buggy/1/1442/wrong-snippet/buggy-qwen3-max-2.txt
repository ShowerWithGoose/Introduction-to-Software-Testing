Looking at the test case input and the expected vs actual output, I can see the issue:

Test Case Input:
```
void f(int a {} // error 1
int main( { // error 2
    int a = 10;
    f(a + 10; // error 3
    return 0;
}
```

Expected Output:
```
1 j
3 j
5 j
```

Buggy Output:
```
1 j
2 j
3 j
```

The problem is that the error on line 2 (missing right parenthesis in `int main( {`) is being reported as line 2, but it should be reported as line 5. Looking at the grammar and error reporting rules:

- Error type 'j' is "Missing right parenthesis ')'. The error line number corresponds to the line where the last non-terminal before the right parenthesis is located."

For the `main` function declaration `int main( {`, the missing right parenthesis should have its error line number correspond to where the last non-terminal before the expected ')' is located. However, looking at the code structure, the `main` function spans multiple lines, and the parser should report the error at the end of the function definition.

But more importantly, looking at the third error: `f(a + 10;` - this is missing a closing parenthesis for the function call. The error should be reported on line 5 (where this statement ends), not line 3.

The key issue appears to be in how the error line numbers are being determined. In multiple places in the code, when reporting error 'j' (missing right parenthesis), the code uses:

```java
Errorlist.getErrorlist().AddError(new Errors(tokenlist.getForwardToken().getLine(), 'j'));
```

The method `getForwardToken()` returns the token at position `cur_pos - 1`, which is the previous token. However, according to the error specification, for missing semicolons and parentheses, the error line should correspond to "the line where the last non-terminal before the semicolon/parenthesis is located."

Looking at the specific case of the function call `f(a + 10;` on line 5, the parser processes tokens and when it expects a `RPARENT` but finds `SEMICN`, it should report the error on line 5, not on an earlier line.

The main issue seems to be in the `UnaryFunc.java` file in the `ParseUnaryfunc` method, where it reports the missing right parenthesis error.
