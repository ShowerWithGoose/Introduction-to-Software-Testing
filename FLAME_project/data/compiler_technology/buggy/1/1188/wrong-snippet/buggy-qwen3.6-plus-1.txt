The core issue is that `Lexer.getLine()` returns the *current* line being scanned, which is often the line *after* the error location when a terminator is missing. To fix this, the parser needs to track the line number of the *last successfully parsed token* or the *start of the current syntactic construct* and report that line number instead of the current lexer line.

However, looking closely at the provided code, there isn't an explicit mechanism in `Parser` to store the "last valid line". Since `Token` doesn't store line numbers, we must rely on the Lexer's state *before* it skips too much forward, or modify the Lexer/Token to store line info.
