Based on the problem description and test case, the issue is that error messages are not being output in the correct line number order. The expected output shows errors in order 16, 18, 19, but the buggy output shows 18, 16, 19.

Looking at the code, the `Compiler.java` file collects errors in an `ArrayList<Err>` called `errorList`, but it doesn't sort them by line number before writing to the error file. However, I notice there's commented-out code that does sort the errors:

```java
Collections.sort(errorList, new Comparator<Err>() {
    @Override
    public int compare(Err e1, Err e2) {
        return Integer.compare(e1.getLineNum(), e2.getLineNum());
    }
});
```

This sorting code is commented out, which explains why the errors are output in the order they were encountered rather than sorted by line number.
