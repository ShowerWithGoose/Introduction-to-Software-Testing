Based on the comparison between the buggy output and the expected output, the error lies in how `Character` tokens (`CHRCON`) are handled within `PrimaryExp`.

In the expected output, when a `CHRCON` is encountered, it should be wrapped in a `<Character>` non-terminal before being wrapped in `<PrimaryExp>`.
Expected:
```
CHRCON 'a'
<Character>
<PrimaryExp>
...
```

In the buggy output, the `<Character>` tag is missing:
```
CHRCON 'a'
<PrimaryExp>
...
```

Looking at the code:
1.  `Parser.java` calls `PrimaryExp()`.
2.  `PrimaryExp()` checks if the token is `CHRCON` and calls `Character()`.
3.  `Character()` in `Parser.java` creates a `CharacterNode`.
4.  `PrimaryExpNode` is constructed with this `CharacterNode`.
5.  `PrimaryExpNode.print()` calls `characterNode.print()`.
6.  `CharacterNode.print()` in `node/CharacterNode.java` only prints the token itself (`charConst.print()`), but **fails to print the `<Character>` non-terminal tag**.

Comparing this to `NumberNode.java`, which correctly prints `<Number>` after printing the token, `CharacterNode.java` is missing the corresponding line.

<node/CharacterNode.java>
package node;

import token.Token;

import java.io.IOException;

public class CharacterNode {
    private Token charConst;
    
    public CharacterNode(Token charConst) {
        this.charConst = charConst;
    }
    
    public void print() throws IOException {
        charConst.print();
    }
}
