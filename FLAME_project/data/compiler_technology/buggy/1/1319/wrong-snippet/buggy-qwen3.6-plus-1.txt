Based on the problem description, the test case input, the buggy output, and the expected output, the error lies in the `parseStmt` method within `Parser.java`.

**Analysis:**
1.  **Symptom**: The program stops parsing after the first two empty blocks `{ }` inside `main`. It outputs `<Block>` and `<Stmt>` for them but then immediately outputs `<MainFuncDef>` and `<CompUnit>`, ignoring the rest of the code (`printf`, nested blocks, `if`, `return`).
2.  **Cause**: In `parseStmt`, when encountering a `{` (LBRACE), the code calls `match("LBRACE")` which consumes the token. Then it calls `parseBlock()`.
3.  Inside `parseBlock()`, it calls `match("LBRACE")` **again**. Since the `{` was already consumed by `parseStmt`, `match("LBRACE")` in `parseBlock` will fail (return false) because the current token is likely the next one (or it might match incorrectly if logic allows, but typically `match` expects the current token).
    *   Let's look at `match`: `if (currentToken().type.equals(expectedType)) { advance(); return true; } else { ... return false; }`.
    *   In `parseStmt`: `if (match("LBRACE")) { BlockNode block = parseBlock(); ... }`. The `{` is consumed. `pos` advances.
    *   In `parseBlock()`: `match("LBRACE");`. The current token is now whatever follows `{` (e.g., `}` or another statement). It is NOT `{`. So `match` returns `false` and does NOT advance.
    *   Then `while (!match("RBRACE"))`. If the next token is `}`, `match("RBRACE")` returns `true`, advances, and the loop condition `!true` becomes `false`. The loop doesn't run.
    *   `parseBlock` returns a `BlockNode` with `null` or empty `blockItems`.
    *   Back in `parseStmt`, it returns `new BlockStmt(block)`.
    *   This seems to handle empty blocks correctly? Wait.
    *   Let's re-read `parseBlock`:
        ```java
        private BlockNode parseBlock() {
            List<BlockItemNode> blockItemNodes = null;
            match("LBRACE"); // Expects LBRACE. If not present, it records error 'i'/'j'/'k'? No, only SEMICN, RBRACK, RPARENT are handled in match error reporting. For LBRACE, it just returns false and does nothing.
            while (!match("RBRACE")) {
                if (blockItemNodes == null)
                    blockItemNodes = new ArrayList<>();
                blockItemNodes.add(parseBlockItem());
            }
            return new BlockNode(blockItemNodes);
        ```

    *   If `parseStmt` consumes `{`, `parseBlock` tries to match `{` again. It fails. Then it checks for `}`. If the block is empty `{ }`, the next token is `}`. `match("RBRACE")` succeeds. Loop ends. Returns empty block. This works for empty blocks.
    *   However, if the block is NOT empty, e.g., `{ printf... }`.
        *   `parseStmt` consumes `{`.
        *   `parseBlock` fails to match `{`.
        *   `while (!match("RBRACE"))`: Current token is `PRINTFTK`. `match("RBRACE")` fails (returns false). Loop continues.
        *   `parseBlockItem()` is called.
        *   `isDecl()` checks. `PRINTFTK` is not a decl.

            *   Checks `IFTK`, `FORTK`, etc. Fails.

        *   Back in `parseBlockItem`, returns `StmtBlock`.
        *   Back in `parseBlock` loop. Next token is `}`. `match("RBRACE")` succeeds. Loop ends.
        *   Returns Block.
