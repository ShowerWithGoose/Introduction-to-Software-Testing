However, a more subtle issue in student compilers often lies in the **Parser's logic for distinguishing declarations from function definitions**, specifically the `isDecl` method, which uses lookahead.

Let's examine `Parser.java`:

```java
public boolean isDecl(Token token) {
    if (token.getTokenType() == TokenType.CONSTTK) {
        return true;
    }
    Token token1 = tokens.getNext();
    Token token2 = tokens.getNext();
    if (token2.getTokenType() == TokenType.LPARENT) {
        tokens.backLast();
        tokens.backLast();
        return false;
    }
    /*
    if (token1.getTokenType() == TokenType.MAINTK) {
        tokens.backLast();
        tokens.backLast();
        return false;
    }*/
    tokens.backLast();
    tokens.backLast();
    return true;
}
```

This method looks ahead 2 tokens.
- If input is `int main()`, `token` is `int`, `token1` is `main` (MAINTK), `token2` is `(` (LPARENT). It returns `false`. Correct.
- If input is `int a = 1;`, `token` is `int`, `token1` is `a` (IDENFR), `token2` is `=` (ASSIGN). It returns `true`. Correct.
- If input is `int f1(int a[], ...)` (FuncDef), `token` is `int`, `token1` is `f1` (IDENFR), `token2` is `(` (LPARENT). It returns `false`. Correct.

There is a logical flaw in `parseUnaryExp` for `IDENFR`.
If `t` (next token) is `LPARENT`, it backtracks (`tokens.backLast()`).
Then it adds `token` (the ident) to nodes.
Then it gets `t` again (`tokens.getNext()`), which is `LPARENT`.
It checks `if (t.getTokenType() == TokenType.LPARENT)`.
Inside, it parses params.
BUT, if the identifier is followed by `(`, it is a function call.
The code structure here is messy.
If `t` is NOT `LPARENT` initially, it backtracks and parses as `PrimaryExp` (which handles `LVal`). This is correct for `a[i]` or just `a`.

However, consider the case where `t` IS `LPARENT`.
It backtracks.
Adds `token`.
Gets `t` (`LPARENT`).
Enters `if`.
Adds `t`.
Gets next token `t`.
Checks if it starts params.
If it does, parses params.
Then expects `RPARENT`.

What if the function has NO parameters? e.g., `main()`.
`t` after `(` is `)`.
The condition `if (t.getTokenType() == TokenType.LPARENT || ... || TokenType.CHRCON)` will be FALSE for `RPARENT`.
So it skips `parseFuncRParams`.
Then it checks `if (t.getTokenType() != TokenType.RPARENT)`.
`t` is `)`. So it adds `t`.
Returns.
This seems okay for `main()`.
