The problem is that the `parseForStmt()` method is being called unconditionally in the for loop parsing, but according to the grammar, the ForStmt components are optional (indicated by [...] in the grammar: `for' '(' [ForStmt] ';' [Cond] ';' [ForStmt] ')' Stmt`).

In the test case, there are for loops like `for(;;)` and `for(;a < 12 ; a = a+1)` where the first ForStmt is empty. However, the current implementation always tries to parse a ForStmt, which fails when there's nothing to parse, causing the parser to not handle empty ForStmt components correctly.
