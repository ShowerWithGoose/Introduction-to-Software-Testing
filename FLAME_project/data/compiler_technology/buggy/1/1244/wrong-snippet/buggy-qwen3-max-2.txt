Based on the test case input and expected vs actual output, I can see that the error line numbers are incorrect. The program outputs errors on lines 1, 2, and 3, but the expected output shows errors on lines 1, 3, and 5.

Looking at the test input:
```
void f(int a {} // error 1 (line 1)
int main( { // error 2 (line 2)  
    int a = 10;
    f(a + 10; // error 3 (line 4)
    return 0;
}
```

The actual errors should be:
- Line 1: missing ')' in function parameter list → error j
- Line 3: this is actually line 4 in the input, missing ')' in function call → error j  
- Line 5: missing '}' for main function → but this should be detected as missing ')' in main declaration first

The issue is in the `checkRParent()` method in the Parser class. The error line number is being set to `tokens.get(curTokenIndex - 1).getLineno()`, but this doesn't correctly capture the line where the opening parenthesis was declared.

Looking at the parsing logic, when we encounter a missing right parenthesis, we should report the line number of the corresponding left parenthesis or the function/construct that requires it, not the previous token.
