Based on the test case input and expected vs actual output, the issue is with error line number reporting for missing right parentheses (error type 'j'). The program reports errors on lines 1, 2, and 3, but should report them on lines 1, 3, and 5.

Looking at the test input:
```
void f(int a {} // error 1 (line 1)
int main( { // error 2 (line 3)  
    int a = 10;
    f(a + 10; // error 3 (line 5)
    return 0;
}
```

The problem is in the `checkRParent()` method in Parser.java. It uses `tokens.get(curTokenIndex - 1).getLineno()` to determine the error line, but this doesn't correctly track the line number where the opening parenthesis was encountered.

The error occurs because when parsing function calls or definitions, the parser advances through tokens and loses track of the original line number where the '(' appeared.
