Looking at the difference between the buggy output and expected output, I can see the issue is in the unary operator handling. 

In the buggy output, when processing `-5`, it shows:
```
MINU -
<Number>
INTCON 5
```

But in the expected output, it should show:
```
MINU -
<UnaryOp>
INTCON 5
```

The problem is in the `parseUnaryOp` method in the Parser.java file. It's creating a node with `SyntaxVarType.NUMBER` instead of `SyntaxVarType.UNARY_OP`.

<frontend/Parser/Parser.java>
...
    public Node parseUnaryOp() {
        // UnaryOp → '+' | '−' | '!'
        ArrayList<Node> children = new ArrayList<>();
        children.add(NodeCreator.createNode(curToken)); read();
        return NodeCreator.createNode(SyntaxVarType.NUMBER, children);
    }
