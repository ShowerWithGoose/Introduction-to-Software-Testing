If `curPos` is 0, `curPos--` becomes -1.
Next `getChar`: `line.charAt(-1)` -> **Crash!**

Can `curPos` become 0 and `ungetChar` be called?
`curPos` starts at 0.
`getChar` makes it 1.
`ungetChar` makes it 0.
If `ungetChar` is called again?

Where is `ungetChar` called?
1. After reading identifier/digit loop.
   ```java

                token+=ch;
            }else{
                ungetChar();
            }
   ```
   If `isEnd` is false, `ungetChar` is called.
   This puts `curPos` back to the character after the token.
   This is standard.

2. After skipping spaces.
   ```java

                ch=getChar();
            }
            ungetChar();
   ```
   If the line is `" int"`.
   `ch` is space.
   Loop: `getChar` -> `i`. `ch` is `i`. Loop ends.
   `ungetChar`. `curPos` back to `i`.
   Next `next()` processes `i`.

3. In `!`, `<`, `>`, `=`, `/` handling.
   ```java

                ...
            }else{
                ungetChar();
                ...
            }
   ```
   If `ch` is not `=`, `ungetChar` puts it back.
