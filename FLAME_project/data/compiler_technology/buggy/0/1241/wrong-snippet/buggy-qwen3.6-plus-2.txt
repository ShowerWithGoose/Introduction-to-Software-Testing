**Summary of the Problem:**
The program is a lexical analyzer. The test case involves a correct source program containing string literals like `"21373119"` and `"%d"`.
- **Expected Output**: The tokens for these strings should be `STRCON "21373119"` and `STRCON "%d"`.
- **Buggy Output**: The tokens are output as `STRCON ""` (empty strings).
- **Analysis**: This indicates that while the lexer correctly identifies the start and end of the string (producing a `STRCON` token), it fails to capture the characters *inside* the quotes into the `tokenValue` StringBuilder.

**Location of the Error:**
The error is in `frontend/Lexer.java`, specifically within the block handling double quotes (`isDoubleQuote(c)`). The code appends the opening quote, then enters a loop. Inside the loop, it reads the next character. If it's not a backslash or a closing quote, it does nothing with the character before looping again (reading the next one). Consequently, all content characters are skipped and not appended to `tokenValue`.
