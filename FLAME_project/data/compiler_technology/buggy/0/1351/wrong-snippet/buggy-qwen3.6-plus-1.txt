**Summary of the Problem:**
The program is a lexical analyzer. The test case involves a correct source program containing string literals like `"21373119"` and `"%d"`.
- **Expected Output**: The string tokens should include the closing double quote, e.g., `STRCON "21373119"` and `STRCON "%d"`.
- **Buggy Output**: The string tokens are missing the closing double quote, e.g., `STRCON "21373119` and `STRCON "%d`.

**Analysis:**
Looking at the `dealString()` method in `frontend/lexer/Lexer.java`:
1. It initializes `StringBuilder sb` and appends the opening `"` (current char).
2. It calls `readCh()` to move to the first character inside the string.
3. It enters a `while (curChar != '\"')` loop. Inside the loop, it appends the current character and reads the next one.
4. When the loop terminates, `curChar` is the closing `"` (because the condition `curChar != '\"'` became false).
5. However, the code immediately creates the token using `sb.toString()` **without** appending this closing `"` to `sb`.
6. It then calls `readCh()` to move past the closing quote for the next token processing.

The fix is to append the closing `"` to `sb` before creating the token.
