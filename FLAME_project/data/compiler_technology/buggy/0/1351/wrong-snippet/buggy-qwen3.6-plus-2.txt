**Summary of Analysis:**

1.  **Problem**: The lexical analyzer is failing to correctly tokenize string constants. Specifically, the closing double quote `"` is missing from the output token value for strings.
2.  **Evidence**:
    *   Input: `printf("21373119");`
    *   Buggy Output: `STRCON "21373119` (Missing closing `"`)
    *   Expected Output: `STRCON "21373119"` (Includes closing `"`)
    *   Input: `printf("%d",a);`
    *   Buggy Output: `STRCON "%d` (Missing closing `"`)
    *   Expected Output: `STRCON "%d"` (Includes closing `"`)
3.  **Root Cause**: In the `dealString()` method in `frontend/lexer/Lexer.java`, the loop `while (curChar != '\"')` reads characters until it encounters the closing quote. However, when the loop terminates, `curChar` is the closing quote `"` , but this character is **not appended** to the `StringBuilder` `sb`. The code then immediately creates the token using `sb.toString()`, which lacks the final quote. Finally, `readCh()` is called to move past the quote, but it's too late for the token content.
