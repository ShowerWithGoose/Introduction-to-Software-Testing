The error is in the `case '&'` and `case '|'` blocks. When a single `&` or `|` is encountered, the code adds an error to `errorList` but does not return -1 to skip adding a token. As a result, the token variable still holds the previous token (from the previous `getSym()` call), and that stale token gets appended to the token list. The error is correctly reported, but the program should also skip adding the erroneous token to the token list. The fix would be to add `return -1;` after adding the error, similar to how comments are handled.
