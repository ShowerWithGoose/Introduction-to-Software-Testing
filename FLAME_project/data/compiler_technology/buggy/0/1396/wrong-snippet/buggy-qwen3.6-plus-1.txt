**Summary of Analysis:**

1.  **Problem:** The program is a lexical analyzer. It fails on a test case containing a C-style comment `// 输出 1`.
2.  **Expected Behavior:** Comments should be ignored (skipped) during lexical analysis and not appear in the output token list.
3.  **Actual Behavior:** The buggy output shows `DIV /`, `DIV /`, and `INTCON 1` corresponding to the characters in the comment `// 1`. This indicates that the lexer is treating `/` as a division operator and `1` as an integer constant, failing to recognize or skip the comment sequence.
4.  **Root Cause:** In `Frontend/Lexer.java`, the code block responsible for handling comments (specifically starting with `/`) is commented out. Consequently, when the lexer encounters `/`, it falls through to the `default` case in the `switch` statement, which simply adds the character `/` as a token. It does not look ahead to check for a second `/` or `*` to form a comment.
