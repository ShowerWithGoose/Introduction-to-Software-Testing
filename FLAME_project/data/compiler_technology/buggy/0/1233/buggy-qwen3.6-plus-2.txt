If `nextToken` is not `=`, it creates `ASSIGN`, then `positionNum++`.
So if we were at `=` (index k), `positionNum` becomes k+1 (pointing to `0`). `nextToken` is `0`. Not `=`. Creates `ASSIGN`. `positionNum` becomes k+2 (pointing to `;`).
The digit `0` at k+1 was SKIPPED. It was read into `nextToken` but not consumed as a token start. The `positionNum` was incremented past it.

This is the bug. When checking for two-character operators like `==`, `!=`, `<=`, `>=`, `&&`, `||`, the code reads the next character to decide. If it's NOT a two-character operator, it treats the current character as a single-character operator. However, it has already advanced `positionNum` to read the next character. If that next character is not part of the operator (e.g., it's a digit or letter starting a new token), the code simply increments `positionNum` again, effectively skipping the character at `positionNum` (the one read into `nextToken`).

For example, in `i=0`:
1. At `=`. `positionNum` is index of `=`.
2. Enter `=` branch. `positionNum++`. Now points to `0`.
3. `nextToken` = `0`.
4. `nextToken != '='`.
5. Create `ASSIGN`.
6. `positionNum++`. Now points to `;`.
7. The `0` was never processed as a digit token.

The fix should be: if it's not a two-char operator, we should NOT have advanced `positionNum` past the next character, OR we should handle the fact that `positionNum` is now pointing to the next character.
Actually, the standard way is:
1. Peek at next char.
2. If it forms a 2-char token, consume both.
3. If not, treat current char as 1-char token, and do NOT consume the next char (it will be processed in the next iteration).

So the error is in all these two-char operator checks. The `positionNum++` in the `else` branches (where it's a single char operator) is incorrect because `positionNum` was already incremented to peek. It should be decremented, or the peek should be done without incrementing, or the increment should only happen if it's a match.

The easiest fix is to remove the `positionNum++` in the `else` blocks of these operators, BUT since `positionNum` was already incremented at the start of the block, we need to decrement it back so the next loop iteration processes that character. Or, better yet, don't increment `positionNum` until we know what we're doing.
