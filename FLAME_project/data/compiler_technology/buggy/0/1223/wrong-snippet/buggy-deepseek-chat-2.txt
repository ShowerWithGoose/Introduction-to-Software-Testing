The error is that the `nextToken()` method starts with `clear()` and then immediately increments `curPos` with `curPos++`, which skips the first character of the source. This causes the first token `int` to be misread as `nt` (since the initial `i` is skipped), leading to the buggy output `IDENFR nt` instead of `INTTK int`.
