**Summary of the Problem:**
The program is a lexical analyzer. It failed a test case where the input source code contains a single `&` character on line 25 (`if ('a' <= str[i] & str[i] <= 'z')`). This is a lexical error (error type 'a'). The expected output is `25 a`. However, the program output `57 a`. This indicates that the lexer is reporting the error on line 57 instead of line 25.

**Analysis:**
1.  **Line Counting Logic**: In `Lexer.java`, the `wordAna` method reads characters one by one. It increments `curLine` when it encounters a `\n`.
2.  **Symbol Handling**: The `symbol` method handles `&` and `|`. When it encounters `&`, it reads the next character to check if it's another `&`. If not, it reports an error `a` using `this.curLine`.
