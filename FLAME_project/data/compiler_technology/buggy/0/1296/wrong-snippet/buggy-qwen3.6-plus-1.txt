Based on the problem description and the comparison between the buggy output and the expected output, the error lies in the handling of comments. The input code contains a C-style single-line comment `// 输出 1`. The lexer fails to recognize `//` as the start of a comment and instead treats `/` as a division operator (`DIV`), then processes the rest of the line as tokens.

The `Lexer.java` file handles the `/` character but does not check if the next character is also `/` (indicating a comment). It simply returns `DIV` and moves to the next character. Consequently, the second `/` is processed as another `DIV`, and the subsequent text is tokenized incorrectly.
