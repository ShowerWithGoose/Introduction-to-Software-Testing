    The use of `reader.ready()` is problematic. `ready()` returns true if the stream is ready to be read without blocking. For a file reader, this is usually true until EOF. However, if `ready()` returns false temporarily (unlikely for FileReader but possible in some buffered contexts or if implementation details vary), it returns `\0`, terminating the lexer prematurely. But standard `FileReader` backed `PushbackReader` usually behaves predictably.
    
    Actually, the most common student error in these assignments involving `PushbackReader` and `ready()` is that `ready()` might return `false` at the end of file, which is handled, but sometimes it behaves unexpectedly with buffering. However, a more likely candidate for "Empty Output" is an exception being swallowed or the writer not flushing/closing properly before the program exits or is judged.

    Why would `getsym` return `\0`?
    `reader.ready()` returns false.
    
    There is a known issue with `PushbackReader` and `ready()` in some Java versions or specific environments where `ready()` might return false if the underlying buffer is empty, even if more data is available in the file. The correct way to check for EOF is to read and check for `-1`.
    
    Current `getsym`:
    ```java
    private char getsym() {
        try {
            if (reader.ready()) {
                return (char) reader.read();
            } else {
                return '\0';
            }
        } catch (IOException e) {
            System.out.println("Error reading file");
        }
        return '\0';
    }
    ```
    
    If `reader.ready()` is false at the start (e.g., buffer not loaded yet), it returns `\0`, `lex()` terminates immediately, `tokens` is empty. `printTokens` writes nothing.
    
    The fix is to remove the `ready()` check and rely on `read()` returning `-1` for EOF.

    ...
    private char getsym() {
        try {
            int read = reader.read();
            if (read == -1) {
                return '\0';
            }
            return (char) read;
        } catch (IOException e) {
            System.out.println("Error reading file");
        }
        return '\0';
    }
