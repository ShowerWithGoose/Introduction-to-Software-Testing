**Summary of the Problem:**
The program is a lexical analyzer. It reads source code from `testfile.txt` and outputs tokens to `lexer.txt` or errors to `error.txt`.
The specific test case contains a lexical error: a single `&` character used instead of `&&` in the line `if ('a' <= str[i] & str[i] <= 'z')`.
The expected output indicates the error is on line **25**.
The buggy output indicates the error is on line **24**.
This suggests an "off-by-one" error in how line numbers are calculated or stored when an error is detected. In Java, `ArrayList` indices are 0-based, while line numbers in files are 1-based. The code currently uses `i` (the loop index) for the line number in most places, but sets `errorRow = i` when an error is found. Since `i` is 0-based, `errorRow` ends up being one less than the actual line number. Other tokens use `i+1` for their row number, which is correct.

**Location of the Error:**
The error is in `Lexer.java` where `errorRow` is assigned. It should be `i + 1` to match the 1-based line numbering convention used elsewhere in the code (e.g., `new Word(..., i+1)`).
