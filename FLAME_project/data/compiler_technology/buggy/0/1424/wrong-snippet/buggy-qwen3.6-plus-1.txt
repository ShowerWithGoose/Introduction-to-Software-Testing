**Summary of Analysis:**

1.  **Problem Description**: The task is to implement a lexical analyzer that outputs tokens for correct programs or errors for erroneous ones. Identifiers (`IDENFR`) should include letters, digits, and underscores.
2.  **Test Case Input**: The input contains valid C-like code, including variable names like `fun1`.
3.  **Buggy Output vs Expected Output**:
    *   In the buggy output, the identifier `fun1` is incorrectly split into two tokens: `IDENFR fun` and `INTCON 1`.
    *   In the expected output, `fun1` is correctly identified as a single token: `IDENFR fun1`.
    *   This pattern repeats for other usages of `fun1`.
4.  **Error Location**: The issue lies in the logic for recognizing identifiers in `frontend/Lexer.java`. The `while` loop condition for building an identifier token only checks for `Character.isLetter(ch)` or `ch == '_'`, but it fails to check for `Character.isDigit(ch)`. Consequently, when the lexer encounters a digit within an identifier (like '1' in 'fun1'), it stops the identifier recognition and subsequently processes the digit as an integer constant.
