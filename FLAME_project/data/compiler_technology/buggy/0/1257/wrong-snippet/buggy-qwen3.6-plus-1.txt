**Summary of the Problem:**
The program is a lexical analyzer. It failed a test case where the input source code contains a single `&` character (which is a lexical error, type 'a').
- **Expected Output:** `25 a` (Line number first, then error code).
- **Actual Output:** `a 25` (Error code first, then line number).

The issue lies in the `toString()` method of the `Error` class, which formats the output string. The current implementation returns `type + " " + line`, but the required format is `line + " " + type`.
