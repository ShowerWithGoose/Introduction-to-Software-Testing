**Fix:**
`nextToken` should not return `null` for valid end-of-file scenarios after comments. Or `LexicalAnalyse` should handle `null`. Or `nextToken` should handle the end-of-file gracefully without returning null if it's just trailing whitespace/comments.

Actually, the standard fix is to ensure `nextToken` doesn't return `null` unless it's an unrecoverable error, OR to filter out nulls in `LexicalAnalyse`.

But looking at the structure, `nextToken` returning `null` at EOF is a design choice. The caller `LexicalAnalyse` should probably check for null.

```java
                Token token = nextToken();
                if (token != null) {
                    tokenList.addToken(token);
                }
```
