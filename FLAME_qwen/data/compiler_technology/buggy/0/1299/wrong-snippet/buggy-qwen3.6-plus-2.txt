### Summary
The assignment requires a lexical analyzer that reads a source program from `testfile.txt`, identifies tokens, and outputs them to `lexer.txt`. If a lexical error occurs (specifically, a standalone `&` or `|`), it should output the error line number and error code `'a'` to `error.txt`. The provided code correctly identifies tokens and handles comments, but contains a logical flaw in the error reporting mechanism for standalone `&` and `|`. The error line number is recorded *after* `getNewCh()` is called, which increments `nowLine` if the next character is a newline. This causes the error to be reported on the following line when `&` or `|` appears at the end of a line.
