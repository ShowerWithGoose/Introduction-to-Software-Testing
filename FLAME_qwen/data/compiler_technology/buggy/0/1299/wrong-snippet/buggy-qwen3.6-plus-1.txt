### Summary of the Issue
The lexical analyzer contains a logical error in the handling of the `&&` and `||` operators. When the scanner encounters `&` or `|`, it correctly reads the next character to check if it forms a valid double-character operator. However, in the successful case (`c == lastCh`), the code attempts to look up the token type in `KeyWord.keyWordMap` **before appending the second character to the `token` string**. As a result, `token` remains as `"&"` or `"|"`, causing `KeyWord.keyWordMap.get(token)` to return `null`. This leads to incorrect token generation (or potential `NullPointerException` downstream) instead of correctly identifying `AND` or `OR` tokens.
