**Summary**
The assignment requires a lexical analyzer that outputs tokens to `lexer.txt` for correct programs and error details (line number + error code) to `error.txt` for erroneous programs. Error type `a` specifically flags standalone `&` or `|` symbols. The provided code contains a critical logical flaw in error handling: `Lexer.java` returns `1` when it encounters a standalone `&` or `|`, but `Compiler.java` completely ignores this return value. The main loop unconditionally prints token information and never writes to `error.txt`, causing the program to fail the error-reporting requirement. Additionally, `Lexer.java` does not expose the current line number or set an error code for the driver to utilize.
