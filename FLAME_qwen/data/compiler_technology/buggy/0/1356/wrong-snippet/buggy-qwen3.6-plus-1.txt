**Summary of the Issue:**
The program fails to correctly handle lexical errors as specified. When an individual `&` or `|` is encountered, `Lexer.next()` returns `1` but does not record the current line number or the required error code (`a`). Furthermore, `Compiler.java` completely ignores the error return value from `next()`. Instead of intercepting the error, retrieving the line number, and writing `line number error code` to `error.txt`, it continues to print empty/stale token data to `lexer.txt`. The error reporting pipeline between the `Lexer` and `Compiler` is missing.
