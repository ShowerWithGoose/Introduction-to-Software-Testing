### Summary of Identified Issues
1. **Inconsistent Error File Paths**: In the `&` and `|` error handling branches, the code writes to `"src/error.txt"` for `&` but `"error.txt"` for `|`. According to the problem description, all errors must be output to `error.txt`.
2. **Flawed Comment Handling Logic**: When `isComment` is `true` and the current character is `/`, the code incorrectly falls into the `if (curChar == '/')` block instead of ignoring it. This can cause the lexer to prematurely break out of the line (if `//` is found inside a comment) or incorrectly emit a `DIV` token. Comment handling should uniformly skip characters until `*/` is detected.
3. **FileWriter Append Mode**: Both `readLine()` and the error handling blocks use `new FileWriter(..., true)`, which appends to existing files. This can cause duplicate outputs if the program is run multiple times without clearing the output files. It's safer to overwrite (`false` or default constructor) or clear the file at the start.
