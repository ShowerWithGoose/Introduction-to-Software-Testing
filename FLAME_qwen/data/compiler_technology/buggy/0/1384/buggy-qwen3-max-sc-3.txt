@@ [The loop iterates over TokenType.values() in an arbitrary order, which may cause shorter tokens (like "&" or "|") to be matched before longer valid tokens (like "&&" or "||"). This leads to incorrect error reporting for valid "&&" or "||" operators because the single "&" or "|" is matched first and flagged as an error. The token matching order must prioritize longer patterns before shorter ones, but enum iteration order does not guarantee that.]
