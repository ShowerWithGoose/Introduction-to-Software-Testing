### Summary of the Issue
The program contains a logical error in how it records the line number for lexical errors. In `TokenLexer.java`, the method `moveForward()` is called **before** `getLineNumber()`. If the erroneous token (`&` or `|`) appears at the very end of a line, `moveForward()` will increment the internal line counter to point to the next line. Consequently, `getLineNumber()` will return the wrong (next) line number, causing the error to be reported incorrectly. Additionally, `errorMap` is initialized as a `HashMap`, which does not guarantee iteration order. The problem specification requires errors to be output in **ascending order of line numbers**, so a `TreeMap` or explicit sorting should be used.
