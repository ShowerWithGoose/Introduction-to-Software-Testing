**Summary**
- **Problem Description**: Design a lexical analyzer that reads `testfile.txt`, identifies tokens or lexical errors, and outputs results to `lexer.txt` or `error.txt` according to specified category codes and formats.
- **Test Case Input**: `/bin/sh: 1: javac: not found` (Indicates a missing Java compiler in the test environment, but static analysis of the provided source code reveals logical bugs that would cause incorrect behavior even if compiled).
- **Buggy/Expected Output**: Empty due to compilation failure, but code inspection shows critical semantic errors.
- **Error Location**: The most direct logical error is in `frontend/Lexer.java` inside the `keywords` HashMap initialization. The keyword `"getint"` is misspelled as `"getin"`, causing the `getint` keyword to be incorrectly tokenized as an identifier (`IDENFR`). Additionally, the `next()` method lacks bounds checking before accessing `sourceCode.charAt(tokenIndex)` for multi-character operators and comments, which will throw `IndexOutOfBoundsException` at EOF. The `isNewLine()` method also modifies `lineno` as a side-effect within a boolean expression, which is unsafe and can lead to incorrect line tracking.
