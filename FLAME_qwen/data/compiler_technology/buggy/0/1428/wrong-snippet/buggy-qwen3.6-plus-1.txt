### Summary of Identified Issues
1. **Missing Bounds Checking in `Lexer.java`**: When peeking at the next character to identify multi-character operators (e.g., `&&`, `||`, `!=`, `<=`, `>=`, `==`, `/*`, `//`), the code directly calls `string.charAt(endPos)` without verifying `endPos < len`. This will cause a `StringIndexOutOfBoundsException` if the operator appears at the end of the file or line.
2. **Incorrect Data Structure for Error Ordering in `ErrorList.java`**: The problem explicitly requires errors to be output in **ascending order of line numbers**. However, `ErrorList` uses a `HashMap`, which does not guarantee any iteration order. This will result in non-deterministic or incorrectly ordered error output. A `TreeMap` or explicit key sorting is required.
