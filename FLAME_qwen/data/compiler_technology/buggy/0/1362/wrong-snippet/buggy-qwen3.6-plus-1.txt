**Summary of the Issue:**
The program incorrectly routes the output to the specified files. According to the problem description, a correct source program should output its tokens to `lexer.txt`, while an erroneous source program should output error information to `error.txt`. However, in `Compiler.java`, the `printError` method is incorrectly passed `outputStream` (which points to `lexer.txt`), and the `printTokens` method is incorrectly passed `errorStream` (which points to `error.txt`). This swaps the intended output destinations, causing the program to fail the test cases.
