### Summary
The lexical analyzer is required to detect single `&` and `|` symbols as error type `a`. In the provided `Lexer.java`, when a single `&` or `|` is encountered, `getChar()` is called to peek at the next character. However, if the next character is not a matching symbol, the code **fails to call `retract()`**, causing the subsequent character to be permanently skipped and lost during scanning. Additionally, the `token` variable is not assigned in the error branch, which leads to the previous token being incorrectly appended to the token list in the `run()` method. The `ErrorItem.toString()` method also prefixes the output with `"Line "`, which may violate the strict output format requirement (`error line number error category code`).
