### Summary
The assignment requires a lexical analyzer that reads source code, identifies tokens according to a specified grammar, and outputs them to `lexer.txt`. It must also detect lexical errors (specifically error type `a` for standalone `&` or `|` symbols) and output them to `error.txt`. The provided test case input (`/bin/sh: 1: javac: not found`) indicates a compilation environment failure, but static analysis of the source code reveals critical logical and semantic errors:
1. **Missing Multi-Character Operator Handling**: In `Lexer.java`, the `analyzeSymbol()` method only explicitly checks for `&&` and `||`. It completely neglects `<=`, `>=`, `==`, and `!=`. As a result, these valid operators will be incorrectly split into two separate tokens (e.g., `<` followed by `=`).
2. **Missing Token Mapping**: In `Keywords.java`, the mapping for the modulo operator `%` to `MOD` is absent. This causes `%` to fall through to the default case and be incorrectly reported as error `a`.
3. **Overly Broad Error Reporting**: The `else` block in `analyzeSymbol()` reports error `a` for *any* character not found in the `Keywords` map. According to the specification, error `a` should only be triggered for individual `&` or `|` symbols.
