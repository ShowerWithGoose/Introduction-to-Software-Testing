### Summary of the Issue
The primary logical error is located in the `analyzeSymbol` method within `Lexer.java`. The method only explicitly handles `&&` and `||` as multi-character operators. It fails to check for other two-character operators defined in the problem (`<=`, `>=`, `==`, `!=`), causing them to be incorrectly split into two separate single-character tokens (e.g., `<=` becomes `LSS <` followed by `ASSIGN =`). Additionally, the `%` operator is missing from the `Keywords` map in `Keywords.java`, which will cause it to be misidentified or incorrectly flagged as an error. The core flaw resides in the symbol analysis logic.

    // 处理其他符号（如运算符，分隔符等）
    private void analyzeSymbol() {
        if (currentChar == '&' && peekChar() == '&') {
            tokens.add(new Token("AND", "&&", lineNumber));
            nextChar();  // 跳过第二个 &
            currentChar = nextChar();
        } else if (currentChar == '|' && peekChar() == '|') {
            tokens.add(new Token("OR", "||", lineNumber));
            nextChar();  // 跳过第二个 |
            currentChar = nextChar();
        } else {
            String tokenType = Keywords.getTokenType(Character.toString(currentChar));
            if (!tokenType.equals("IDENFR")) {
                tokens.add(new Token(tokenType, Character.toString(currentChar), lineNumber));
            } else {
                errors.add(lineNumber + " a");  // 非法字符
            }
            currentChar = nextChar();  // 推进到下一个字符
        }
    }
