**Summary of the Issue:**
The program contains critical logical and semantic errors in the token definition and keyword mapping. According to the problem description, reserved words (like `int`, `main`, `const`, `if`, etc.) must be recognized as distinct token types (`INTTK`, `MAINTK`, `CONSTTK`, `IFTK`, etc.). However, the `TokenType` enum in `lexer.h` is missing these keyword token definitions entirely. Furthermore, the `keywords` unordered map incorrectly maps every reserved word to `TokenType::IDENFR` instead of their respective specific token types. This will cause the lexical analyzer to misclassify all keywords as identifiers, leading to incorrect output and failing the test cases.
