Based on the problem description and error type "a" (The symbols "&" and "|" appearing individually), the issue is in the lexical analyzer's handling of single '&' or '|' characters. The test case input shows a compilation error related to `javac` not being found, but the actual lexical error would be detected in the `scanSymbol` method of the Lexer class.

Looking at the Lexer code, when it encounters a single '&' or '|', it correctly adds an InvalidToken error but then still tries to create a token for "&&" or "||". This is incorrect - it should not create a token when encountering a single '&' or '|'.
