Based on the problem description and test case, the issue appears to be related to handling of the '&' symbol in lexical analysis. The error type 'a' corresponds to "The symbols "&" and "|" appearing individually", which should be detected as InvalidToken.

Looking at the Lexer.java file, specifically the scanSymbol() method, I can see the logic for handling '&' and '|' characters. The current implementation adds an error when a single '&' or '|' is encountered, but it still proceeds to add a token for "&&" or "||" even when only a single character was found.
