**Summary:**
The lexical analyzer is required to detect individual `&` and `|` symbols as lexical errors (error type `a`). In the `scanSymbol` method of `Lexer.java`, when a `|` or `&` is encountered, the program correctly reads the next character and checks if it forms a valid `||` or `&&` operator. If the second character does not match, it correctly records an `InvalidToken` error and unreads the character. However, the code **unconditionally** calls `addToken` immediately after the `if` block. This causes an invalid `||` or `&&` token to be added to the token stream even when an error has been detected. The `addToken` calls should be placed inside an `else` block (or guarded by a condition) so that tokens are only emitted when the operator is valid.
