### Summary
The provided program implements a lexical analyzer that reads source code line-by-line, tokenizes it, and outputs tokens to `lexer.txt` or lexical errors to `error.txt`. The test case input `/bin/sh: 1: javac: not found` indicates an environment configuration issue (Java compiler not found), but analyzing the source code reveals a critical logical flaw in the tokenization process.

**Primary Error Identified:**
In `frontend/Token.java`, the whitespace handling logic simply skips whitespace characters using `continue` without flushing the accumulated `currentToken`. This causes identifiers, keywords, or integer constants separated by spaces to be incorrectly concatenated into a single token (e.g., `int a` becomes `inta`, `1 2` becomes `12`). Additionally, there is redundant/dead code for character constant (`'`) parsing, but the missing token flush on whitespace is the core semantic error that will cause widespread tokenization failures.
