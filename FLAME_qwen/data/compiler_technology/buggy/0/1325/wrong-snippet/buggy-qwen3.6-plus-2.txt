### Summary
The problem description explicitly states: *"Ensure that the read string is preserved exactly as it appears for output purposes, especially for numerical values."* However, the current implementation of `parseInt()` in `Lexer.java` converts the sequence of digits into a primitive `int` value. This process strips leading zeros and alters the original lexical representation (e.g., `007` becomes `7`). Consequently, `Token.toString()` in `Token.java` outputs this parsed integer value for `INTCON` tokens instead of the original string, violating the required output format.
