### Summary
- **Problem Description**: Implement a lexical analyzer that outputs token sequences to `lexer.txt` for correct programs, and error line numbers with error codes to `error.txt` for erroneous programs. Error type `a` corresponds to standalone `&` or `|`.
- **Test Case Input**: `/bin/sh: 1: javac: not found` indicates the execution environment lacked the Java compiler, so the program never actually ran. However, static code analysis reveals definitive logical flaws that would cause incorrect behavior even if compiled.
- **Buggy/Expected Output**: Both are empty due to the compilation failure, but the source code contains a critical copy-paste error in the error-handling branch and missing boundary checks in the lexer.
- **Error Location**: The primary logical error is in `Compiler.java`'s `else` block. When errors are detected, it incorrectly redirects output to `lexer.txt` (instead of `error.txt`) and prints the `res` list (valid tokens) instead of the `errors` list. Additionally, `frontend/ParseLexer.java` lacks `location + 1` bounds checks when handling `&`, `|`, and comments, which will throw `StringIndexOutOfBoundsException` on certain inputs.
