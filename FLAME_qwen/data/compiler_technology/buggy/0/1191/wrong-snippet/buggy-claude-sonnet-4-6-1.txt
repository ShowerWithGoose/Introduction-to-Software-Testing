## Summary

**Problem:** Lexical analyzer for a custom language, outputting token types or error codes.

**Test failure:** The program fails to compile — `javac: not found` — meaning the Java compiler isn't available in the test environment. However, beyond the environment issue, there are also logical bugs in the source code worth identifying.

**Key issues found:**

1. **`getint` / `getchar` keyword casing mismatch** — The keywords are registered as `"getInt"` and `"getChar"` in the HashMap, but the language spec uses lowercase `getint` and `getchar`. This means these keywords will never be recognized and will fall through as `IDENFR`.

2. **`/` (division) handling — missing `retract()`** — When `/` is followed by a non-comment character, the lexer has already consumed the next character via `getChar()` but never retracts it. That next character is silently skipped.

3. **`error` HashMap iteration order** — `HashMap` does not guarantee insertion order, so errors may not be output in ascending line number order as required.
