### Summary of Information & Error Analysis
- **Problem**: Implement a lexical analyzer that outputs tokens (`category code string`) for correct programs, or errors (`line_number error_code`) for erroneous programs. The only defined lexical error is type `a` for standalone `&` or `|`.
- **Input/Output**: Reads from `testfile.txt`, writes to `lexer.txt` or `error.txt`.
- **Identified Issue**: In `frontend/Lexer.java`, the handling of `&&` and `||` contains a logical flaw. When the second `&` or `|` is successfully matched, the position pointer `curPos` is **not incremented**. As a result, the outer `while` loop's next iteration re-processes the second character, incorrectly treating it as a new standalone `&` or `|`, which triggers a false error report (type `a`) or generates duplicate tokens. Additionally, `Errorlog.WriteError2File()` opens `error.txt` in append mode (`true`), which may cause residual errors from previous test runs to persist if the file isn't cleared by the judging system.
