### Summary of Information
- **Problem**: Implement a lexical analyzer that tokenizes source code into `lexer.txt` or reports lexical errors to `error.txt`. Error type `a` corresponds to standalone `&` or `|` symbols.
- **Key Requirement**: "Ensure that the read string is preserved exactly as it appears for output purposes, especially for numerical values."
- **Test Case Input**: `/bin/sh: 1: javac: not found` (This is a system environment error from the grading runner, not the actual test case content. The analysis focuses on the provided source code logic.)
- **Identified Issues**:
  1. **`getIntConst()`**: Incorrectly handles integers with leading zeros. The condition `if (sourceCode.charAt(charIndex - 1) != '0')` prematurely stops reading digits, causing numbers like `05` to be split into `0` and `5`, violating the "preserve exactly" requirement.
  2. **`getOperator()`**: When detecting a standalone `&` or `|`, it incorrectly adds a token with type `AND` or `OR` to the `tokens` list. Since these are lexical errors, they should only be reported to `error.txt` and not be treated as valid tokens for `lexer.txt`.
  3. **`skipComment()`**: The multi-line comment loop lacks a bounds check for `charIndex + 1`, which will throw an `IndexOutOfBoundsException` if the file ends with an unclosed `/*`.
