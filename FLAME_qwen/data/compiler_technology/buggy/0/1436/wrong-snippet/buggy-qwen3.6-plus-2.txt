### Summary of Issues
1. **`IntConstParser` Logical Error**: In `IntConstParser::parse`, when `buf` is empty and the next character is a digit, it returns `{true, true}`. The `canUse` flag should be `false` because an empty string is not a valid integer constant token. It should return `{false, true}` to indicate it can continue parsing but is not yet a complete token.
2. **`getNext()` Runtime Crash**: When encountering invalid characters like `&` or `|` individually, no parser will accept them as a complete token (`totalCanUse == 0`) and no parser will continue (`totalCanNext == 0`). This causes `curParsers` to become empty after `goNextStatus()`. Accessing `curParsers.back()` in the `else` branch leads to undefined behavior/segmentation fault. A safety check for an empty vector is required.
