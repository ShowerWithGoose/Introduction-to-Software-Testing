**Summary of Issues:**
1. **Typo in Keyword Token:** In `Lexer.java`, the token type for the `continue` keyword is incorrectly assigned as `"CCONTINUETK"`. According to the specification table, it should be `"CONTINUETK"`.
2. **Line Comment State Persistence:** In `Compiler.java`, the `zhushi` (comment state) variable is not reset to `0` after processing a line comment (`//`). Since `zhushi == 1` indicates a line comment, passing this state to the next line's `Lexer` causes the entire next line to be incorrectly skipped.
