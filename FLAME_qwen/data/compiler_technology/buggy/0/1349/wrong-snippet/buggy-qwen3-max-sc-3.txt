Based on the problem description and test case, the issue is related to error type 'a', which occurs when '&' or '|' appear individually (not as '&&' or '||'). The test input likely contains a single '&' or '|', and the program should detect this as a lexical error and output the line number with error code 'a'.

Looking at the Lexer code, specifically the `addComparatorToken` method, there are logical errors in handling the '&' and '|' cases. When a single '&' is encountered, the code currently:
1. Adds an AND token for "&" 
2. Reports error 'a'
3. Then **still adds another AND token for "&&"**

This is incorrect - it should only add the single "&" token and report the error, not proceed to add a "&&" token.

The same issue exists for the '|' case.
