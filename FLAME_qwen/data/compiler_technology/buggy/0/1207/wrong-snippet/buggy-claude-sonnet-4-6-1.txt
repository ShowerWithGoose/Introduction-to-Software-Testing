## Summary

**Problem:** The program fails to compile because `javac` is not found (`/bin/sh: 1: javac: not found`). This is a compilation error, not a logic error.

**Root cause:** The source files are in a package structure (`package frontend;`) and need to be compiled with the correct classpath. However, looking at the code itself, there is also a structural bug in `linesLexer()`: after the word-matching loop fails to match any token, the code **always** falls through to the error-matching loop ‚Äî there's no `else` or flag to prevent this. More critically, if neither loop matches, `start` never advances, causing an **infinite loop**.

Additionally, the error regex in `Error.java` is incorrect.

---

Here are the code snippets where the errors are located:

**frontend/Error.java**
```
...
public enum Error {
    a("^&|\\|");
...
```

The regex `^&|\\|` is parsed as `(^&)` OR `(\\|)` ‚Äî the `^` only anchors the first alternative, so `\\|` matches a bare `|` anywhere in the string, not just at the start. It should be `^[&|]` or `^(&(?!&)|\\|(?!\\|))` to match a single `&` or `|` at the start.
