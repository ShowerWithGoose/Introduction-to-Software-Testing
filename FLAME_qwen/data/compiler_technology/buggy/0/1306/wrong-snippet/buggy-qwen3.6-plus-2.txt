### Summary of Issues
Based on the provided code and problem description, the primary logical errors are located in `frontend/Lexer.java`:

1. **Missing Index Update on EOF in Parsing Loops**: In the inner `for` loops for parsing identifiers, integer constants, and string constants, the index `i` is only updated when a delimiter is found (`i = j - 1` or `i = j`). If the loop reaches the end of the input string (`j == len`) without encountering a delimiter, the loop terminates naturally, but `i` remains unchanged. The outer loop then increments `i` by only 1, causing the lexer to repeatedly re-scan and incorrectly tokenize the remaining characters of the word/number/string.
2. **Incorrect Line Counting in `//` Comments**: The comment handling logic increments `currentLine` when encountering `\t` or `\r`, which is semantically incorrect. Line numbers should only increment on `\n`.
3. **Missing Bounds Check for Character Constants**: The character constant parsing accesses `program.charAt(i + 2)` without verifying `i + 2 < len`, which will throw an `IndexOutOfBoundsException` if the file ends with a single `'`.

The most critical and frequently triggered bug is **#1**, which corrupts tokenization for any identifier, number, or string at the end of a line or file.
