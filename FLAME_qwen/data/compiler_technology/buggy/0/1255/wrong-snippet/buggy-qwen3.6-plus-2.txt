**Summary of the Issue:**
The lexical analyzer crashes with a `StringIndexOutOfBoundsException` because it lacks necessary boundary checks before accessing characters in `fileStr`. Specifically:
1. In the `compile()` method, the comment detection logic accesses `fileStr.charAt(pos + 1)` without verifying that `pos + 1` is within the string bounds. This causes a crash if the input ends with a single `/` or `*`.
2. The parsing methods (`parseAlp`, `parseDigit`, `parseStr`, `parseChar`) use `while` loops that continuously call `fileStr.charAt(pos)` without checking `pos < fileStr.length()`. If the input file ends abruptly (e.g., an identifier or number at EOF without a trailing newline, or an unclosed string/char literal), these methods will read past the end of the string and throw an exception.
