**Summary**
The lexical analyzer contains a logical flaw in the `parseSigns()` method when handling the `&` and `|` symbols. According to the specification, a single `&` or `|` appearing individually should be reported as error type `a`. However, the current code unconditionally adds the `AND` or `OR` token to the token list *before* verifying whether the next character forms a valid `&&` or `||` sequence. This causes the program to incorrectly output valid tokens for erroneous inputs and fails to properly isolate error detection from token generation.
