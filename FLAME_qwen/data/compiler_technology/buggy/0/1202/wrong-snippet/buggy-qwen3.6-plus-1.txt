### Summary
- **Problem Description**: Implement a lexical analyzer that reads `testfile.txt`, identifies tokens (with category codes and values), and outputs them to `lexer.txt`. If lexical errors occur (specifically single `&` or `|`), output the line number and error code `a` to `error.txt`.
- **Test Case Input**: `/bin/sh: 1: javac: not found` (Indicates a compilation environment issue in the grading system, but the source code itself contains logical/runtime bugs that would cause failure even if compiled).
- **Buggy Output**: Empty/None (Program likely hangs or produces empty files).
- **Expected Output**: Correctly formatted token list or error log.
- **Identified Errors**:
  1. `skipSpace()` in `Lexer.java` lacks an end-of-file/line boundary check. When `getChar()` returns `'\n'` at EOF, `Character.isWhitespace('\n')` evaluates to `true`, causing an infinite loop.
  2. `Compiler.java` fails to `close()` or `flush()` the `FileOutputStream` instances. Without closing, buffered data may not be written to disk, resulting in empty `lexer.txt` and `error.txt`.
  3. `skipComment()` in `Lexer.java` does not safely check bounds before accessing `charAt(charNum + 1)`, which can throw `IndexOutOfBoundsException` if `//` or `/*` appears at the very end of a line.
