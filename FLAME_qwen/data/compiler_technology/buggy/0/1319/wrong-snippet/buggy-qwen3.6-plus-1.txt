### Summary
- **Problem Description**: Implement a lexical analyzer that reads `testfile.txt`, identifies tokens (outputting to `lexer.txt`), or detects lexical errors (outputting line number and error type to `error.txt`). Only error type `a` (individual `&` or `|`) is specified for this assignment.
- **Test Case Input**: `/bin/sh: 1: javac: not found` (Indicates a compilation/environment failure in the grading system, but the source code contains a logical flaw that would cause incorrect behavior during execution).
- **Buggy/Expected Output**: Empty (due to compilation failure, but logically the program would produce incorrect line numbers for errors).
- **Error Analysis**: The `advance()` method in `Lexer.java` does not update the `lineno` counter when a newline character (`\n`) is consumed. Since `lineno` is only incremented in the main `tokenize()` loop when whitespace is explicitly handled, any newlines consumed internally by comment skipping (`//` and `/* */`) or other helper methods will not increment the line counter. This results in incorrect line numbers being reported for any lexical errors that occur after comments or multi-line constructs.
