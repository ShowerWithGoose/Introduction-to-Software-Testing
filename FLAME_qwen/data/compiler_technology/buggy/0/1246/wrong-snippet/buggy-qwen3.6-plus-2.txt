### Summary of the Issue
The lexical analyzer fails to correctly detect and report **error type 'a'** (individual `&` or `|` symbols) in certain scenarios, particularly when the symbol appears at the very end of the input file. This occurs because the error detection logic for type 'a' (`isErrorA`) is incorrectly nested inside the `if (pos < maxLength && TokenType.doubleCharList.contains(nowChar))` block. When `pos == maxLength` (i.e., the symbol is the last character in the file), the condition `pos < maxLength` evaluates to `false`, causing the entire block to be skipped. As a result, `tokenType` remains `null`, the loop terminates without adding the error to `ErrorHandler`, and the error goes unreported. The check for error 'a' should be performed independently when no valid single or double-character token has been matched.
