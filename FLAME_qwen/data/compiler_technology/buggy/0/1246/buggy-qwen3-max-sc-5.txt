@@ The logic fails to handle standalone '&' or '|' when they appear at the end of file (pos == maxLength) or when they are not followed by a second character that forms a valid double-char token. The condition `pos < maxLength && TokenType.doubleCharList.contains(nowChar)` skips error checking for '&' or '|' at the last position. Additionally, if a single '&' or '|' is processed as a singleChar (which it shouldn't be, since '&' and '|' are not in singleCharList), but more critically, the error detection only triggers inside the doubleCharList block and only when pos < maxLength. Since '&' and '|' are not in singleCharList, they fall through without being handled, causing a null tokenType and no token added, but also no error reported.
