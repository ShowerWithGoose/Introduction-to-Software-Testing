### Summary of Issues
Based on the provided code and problem description, there are two critical logical errors in `frontend/Lexer/Lexer.java` that will cause incorrect tokenization or infinite loops:

1. **Missing `else`/`continue` after comment/division handling**: The `if(ch == '/' ...)` block is structurally disconnected from the subsequent `if(Character.isDigit(ch))`. After processing a comment or a `/` operator, the code falls through to the next conditions using the stale `ch` value. This causes duplicate token generation or incorrect parsing paths. It should be chained with `else if` or followed by a `continue` statement.
2. **Inverted condition for block comment termination**: Inside the `/* */` comment loop, the condition `line.charAt(pos) != '*'` is used to detect `*/`. This is logically incorrect; it should be `line.charAt(pos) == '*'`. As written, the loop will never correctly match the `*` in `*/`, causing the comment to consume the rest of the file or fail to terminate properly.
