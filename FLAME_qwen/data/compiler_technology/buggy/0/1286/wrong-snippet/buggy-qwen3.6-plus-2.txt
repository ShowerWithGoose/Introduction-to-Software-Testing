### Summary
The lexical analyzer logic in `Lexer.java` correctly parses tokens and detects the specified lexical error (standalone `&` or `|`). However, in `Compiler.java`, when writing error information to `error.txt`, the code uses `"x"` as a separator between the line number and the error category code. According to the problem specification, they must be separated by a **single space**. Additionally, the `BufferedWriter` for the error file is never closed, which may prevent the data from being flushed to disk.
