**Summary of Information:**
- **Problem Description:** Implement a lexical analyzer that outputs tokens for correct programs or error line numbers and codes for erroneous programs. Error type `a` corresponds to standalone `&` or `|` symbols.
- **Test Case Input:** `/bin/sh: 1: javac: not found` (Indicates a compilation/environment failure in the test runner, but the source code itself contains a logical bug that would cause incorrect behavior or runtime exceptions).
- **Buggy/Expected Output:** Empty due to the failure.
- **Identified Error Location:** In `Lexer.java`, the handling of the `|` character contains a copy-paste error. It incorrectly appends `source.charAt(curPos)` (the *next* character) to the `StringBuilder` instead of `tempChar` (the current `|` character). This causes `||` to be misidentified as an error and single `|` to generate malformed tokens.
