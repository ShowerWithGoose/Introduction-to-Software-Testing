The bug is that when a single '&' or '|' is encountered (error type 'a'), the program still outputs the token to `lexer.txt` before later deleting it. This violates the requirement that for an erroneous source program, only `error.txt` should be produced. The error is located in the `getToken` method where, after detecting the error and printing to `error.txt`, the code unconditionally calls `printToLexer` at the end.

                        //error here
                    } else token = "" + c + ch;
