### In `chrcon()`:
```cpp
void Lexer::chrcon() {
    std::string chr = "";
    pos++;
    // 需要考虑转义字符
    if (source[pos] == '\\') {
        std::string chr1 = "";
        chr1.push_back(source[pos++]);
        chr1 = "\'" + chr1 + "\'";
        tokens.push_back(Token{chr1, Token::TokenType::CHRCON, line_number}); // '\''
        chr.push_back(source[pos]);
        pos +=2;
        tokens.push_back(Token{chr, Token::TokenType::CHRCON, line_number});
    } else {
        chr.push_back(source[pos]);
        pos +=2;
        chr = "\'" + chr + "\'";
        tokens.push_back(Token{chr, Token::TokenType::CHRCON, line_number});
    }
}
```

This logic is **fundamentally broken**:

- If it sees `'\\'`, it:
  1. Pushes a token for `'\'` (which is incomplete — it should include the next char like `\n`, `\\`, etc.)
  2. Then reads `source[pos]` (now after `\`) and pushes **another** character as a separate char const.
  3. Does `pos += 2` after already incrementing `pos` once → **over-skipping**
- In the `else` branch: it does `pos += 2`, assuming the next char is the closing `'`, but **does not check** if `source[pos + 1]` is actually `'`.

Moreover, **it never validates that the character constant is properly closed**. For example, input like `'a` (missing closing quote) should be an error, but this code assumes it's always followed by a `'` and blindly does `pos += 2`.
