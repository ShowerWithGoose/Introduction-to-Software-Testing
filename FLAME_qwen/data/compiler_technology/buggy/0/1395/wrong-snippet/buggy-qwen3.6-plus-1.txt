**Summary:**
The lexical analyzer processes the source code character by character, tracking the current line number. When a token contains newline characters (e.g., whitespace tokens or comments spanning multiple lines), the code attempts to update the `line` counter using `token.getContent().split("\n").length - 1`. However, Java's `String.split("\n")` method **discards trailing empty strings by default**. When a token consists solely of a newline (`"\n"`), `split` returns an empty array with length `0`, causing the line counter to incorrectly decrement by `1` instead of incrementing. This leads to inaccurate line numbers for all subsequent tokens and error reports, which will cause the program to fail test cases that verify line number accuracy.
