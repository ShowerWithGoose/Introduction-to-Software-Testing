### Summary of Identified Issues
1. **Line Number Double Increment**: In `Lexer.java`, `lineNum` is incremented twice for every newline character (`\n`). Once in the main `tokenize()` loop (`if(currentChar == 10) { lineNum+=1; ... }`) and again inside the `readNextChar()` method (`if(currentChar==10) { lineNum++; }`). This causes all subsequent tokens and errors to report incorrect line numbers.
2. **Missing Newline in Error Output**: In `Compiler.java`, when writing to `error.txt`, the code `writer.write(token.getLine()+" "+"a");` does not append a newline character. This will cause multiple errors (if any) to be concatenated on a single line, violating standard output formatting expectations.
