### Summary
The provided code implements a lexical analyzer for a specific grammar. Based on the problem description and code analysis, a critical semantic error exists in the `getChar` function within `src/lexer.cpp`. The `switch` statement responsible for parsing escape sequences in character constants is missing `break` statements after most `case` labels. This causes unintended fall-through execution, meaning that for almost any escape character (except `'a'`), the `result` variable will be overwritten repeatedly until it hits the `default` case and is set to `-1`. This corrupts the internal value of the character token and breaks correct lexical analysis for `char` constants. Additionally, the closing single quote `'` is not properly consumed after `getChar` returns, which can lead to an infinite loop or incorrect token stream generation.
