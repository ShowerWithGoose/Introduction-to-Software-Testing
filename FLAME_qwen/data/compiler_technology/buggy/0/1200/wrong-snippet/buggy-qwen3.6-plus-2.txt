### Summary of Information
- **Problem Description**: Implement a lexical analyzer that outputs token types and values for correct programs, or line numbers and error codes for erroneous programs. Specifically, error type `a` corresponds to standalone `&` or `|` symbols.
- **Test Case Input**: `/bin/sh: 1: javac: not found` (Indicates a shell/environment compilation failure, but the code itself contains logical flaws that would cause incorrect behavior or compilation issues on older Java versions).
- **Buggy/Expected Output**: Empty due to the compilation failure.
- **Identified Errors**:
  1. `tokenError()` in `Lexer.java` is completely empty. It is called when a standalone `&` or `|` is detected, but it fails to record the error, set an error flag, or throw an exception as expected in a lexical analyzer.
  2. `Compiler.java` attempts to work around the empty `tokenError()` by checking `lexer.getToken().equals("&")` or `"|"`. This is fragile and mixes lexical error detection with output logic. Error handling should be centralized in the Lexer or use a proper error reporting mechanism.
  3. `Files.readString()` requires Java 11+. If the grading environment uses Java 8, this will cause a compilation error. It should be replaced with `new String(Files.readAllBytes(...))` or `BufferedReader` for compatibility.
