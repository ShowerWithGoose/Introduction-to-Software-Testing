### Summary
- **Problem Description**: Implement a lexical analyzer that reads a source program from `testfile.txt`. For correct programs, output token types and values to `lexer.txt`. For erroneous programs, output line numbers and error codes to `error.txt`. Error type `a` specifically flags standalone `&` or `|` symbols.
- **Test Case & Expected Behavior**: The program should correctly tokenize all valid symbols, including operators, keywords, identifiers, and literals. It must also correctly identify and report lexical errors.
- **Identified Error**: In the `Lexer.java` file, the handling of the `/` character is incomplete. The code correctly identifies `//` (line comments) and `/*` (block comments), but **lacks an `else` branch to handle the division operator `/`**. As a result, when `/` appears as a mathematical operator, it is completely ignored and not added to the token list, causing the lexical analysis to fail or produce incorrect output for any program using division.
- **Secondary Issue**: If `testfile.txt` is missing, `inputFileStream` remains `null`, and `new Scanner(inputFileStream)` will throw a `NullPointerException`. It's recommended to add a null check or `return` statement after the catch block.
