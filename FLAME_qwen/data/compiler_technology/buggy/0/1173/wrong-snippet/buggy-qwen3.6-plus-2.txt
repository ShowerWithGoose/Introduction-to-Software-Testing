### Summary
- **Problem Description**: Implement a lexical analyzer that outputs token types and values for correct programs, or error line numbers and error codes for erroneous programs. The specific lexical error to handle is type `a`: individual `&` or `|` symbols appearing alone.
- **Test Case Input**: `/bin/sh: 1: javac: not found` (This indicates an environment/compilation failure in the grading system, but we will focus on the logical flaws within the provided source code).
- **Buggy/Expected Output**: Not explicitly provided, but code analysis reveals a critical state management bug in the error handling path.
- **Error Location**: In `frontend/Lexer.java`, the branches handling individual `&` and `|` symbols call `error('a', lineNum)` but **fail to call `retract()`**. This causes the lexer to incorrectly consume and discard the character immediately following the erroneous symbol, which disrupts the parsing of subsequent tokens and can lead to incorrect tokenization or infinite loops.
