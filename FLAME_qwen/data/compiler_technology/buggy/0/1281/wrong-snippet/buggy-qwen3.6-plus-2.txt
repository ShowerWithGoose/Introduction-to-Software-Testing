**Summary of the Issue:**
The provided lexical analyzer has two primary logical errors that will cause incorrect tokenization and line number tracking:
1. **Missing Keywords:** The problem specification requires recognizing `char` and `getchar` as keywords with token types `CHARTK` and `GETCHARTK`. However, both the `TokenType` enum and the `specificWords` map in `Lexer.java` are missing these entries. This will cause `char` and `getchar` to be incorrectly tokenized as identifiers (`IDENFR`).
2. **Line Number Tracking Bug in Comments:** The single-line comment (`//`) handler consumes the newline character (`\n`) to exit the comment loop but fails to increment `lineNumber`. This causes all subsequent tokens or errors to be reported on the wrong line number.
