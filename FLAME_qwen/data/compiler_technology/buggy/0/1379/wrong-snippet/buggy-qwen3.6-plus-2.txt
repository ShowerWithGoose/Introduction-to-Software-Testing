### Summary of Identified Errors

1. **Infinite Loop on Lexical Error (`Compiler.java`)**: When `Error_A` is thrown in `lexer.next()`, the catch block sets `existError = true` but does not exit the loop. Since `next()` throws before updating `curToken`, `lexer.peek()` continues to return the previous valid token. The loop then calls `next()` again on the same erroneous character, causing an infinite loop. A `break;` statement is required after catching the error.
2. **Incorrect Line Number Tracking in Comments (`Lexer.java`)**: Inside the `//` and `/* */` comment parsing blocks, newline characters (`\n`) are consumed but do not increment `lineNum`. This causes all subsequent tokens or errors to be reported on incorrect line numbers. `lineNum++` must be triggered when `\n` is encountered within these loops.
