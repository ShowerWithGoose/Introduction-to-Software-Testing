### Summary
- **Problem**: Implement a lexical analyzer that reads `testfile.txt`, identifies tokens, and outputs them to `lexer.txt`. If lexical errors exist, output the line number and error code to `error.txt`.
- **Error Type `a`**: Represents standalone `&` or `|` symbols.
- **Identified Issue**: In `frontend/Lexer.java`, after incrementing the index `i` to check for two-character operators (like `&&`, `||`, `!=`, `<=`, `>=`, `==`, `//`, `/*`), the code directly calls `input.charAt(i)` without verifying if `i` is still within the string bounds. This will cause a `StringIndexOutOfBoundsException` when `&` or `|` (or other operators) appears at the very end of the file or is immediately followed by EOF. The same pattern applies to `!`, `<`, `>`, `=`, and `/` handling blocks.
