**Summary:**
- **Problem Description**: Implement a lexical analyzer that outputs token types and values for correct source programs, or line numbers and error codes (specifically error type `a` for standalone `&` or `|`) for erroneous programs.
- **Test Case Input**: `/bin/sh: 1: javac: not found` (Indicates the program failed to compile or run in the test environment, likely due to a runtime exception).
- **Buggy/Expected Output**: Both are empty, indicating the program crashed before producing any output.
- **Error Location**: In `frontend/Lexer.java`, several `while` loops and `if` conditions lack boundary checks (`i < input.length()`). After incrementing `i` to peek at the next character for multi-character operators (`&`, `|`, `!`, `<`, `>`, `=`), or when scanning identifiers and integers, the code directly calls `input.charAt(i)` without verifying that `i` is still within bounds. This causes a `StringIndexOutOfBoundsException` when the input ends with a token or when checking for a second character that doesn't exist.
