Based on the problem description and test failure, the issue appears to be in the **ErrorChecker.java** file. The test case input shows a compilation error due to missing `javac`, but more importantly, the actual lexical analysis error handling logic has a flaw.

The key issue is that the **ErrorChecker** processes lines independently and checks for single `&` or `|` characters outside of strings and comments. However, it does **not account for cases where `&` or `|` appear inside character constants (e.g., `'&'`) — which are valid and should not trigger an error**.

Looking at the provided code:

- In `<Frontend/ErrorChecker.java>`, the method `processLine` skips string literals (`"..."`) but **does not skip character constants (`'...'`)**.
- Therefore, if a source file contains something like `char c = '&';`, the ErrorChecker will see the `&` outside of a string and report an erroneous "a" error, even though it's inside a valid character constant.

This matches error type **a**: *"The symbols "&" and "|" appearing individually"*, but incorrectly flags them when they're actually inside character literals.
