**Summary of Issues:**
1. **Invalid Import Statements**: The code contains `import utils.IO.*;` and `import frontend.Lexer.*;`. In Java, the `.*` wildcard can only be applied to package names, not to specific class names. These lines are syntactically incorrect and will cause a compilation failure.
2. **File Write Mode**: The `IO.write` method uses `new FileWriter(outputFile, true)`, which opens the file in **append mode**. For a lexical analyzer that is expected to generate fresh `lexer.txt` or `error.txt` outputs per run, this should typically be overwrite mode (`false`). Otherwise, outputs from previous executions will accumulate and cause test case mismatches.
