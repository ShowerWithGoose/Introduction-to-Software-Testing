## Summary of the Problem

**Problem Description**: A lexical analyzer that should:
- For correct programs: output token type and value to `lexer.txt`
- For erroneous programs: output line number and error code 'a' to `error.txt`

**Error Type**: The only lexical error is when '&' or '|' appears individually (not as '&&' or '||')

**Test Case**: The test input shows a compilation error (`g++: not found`), which suggests the program couldn't compile or run.

**Expected vs Actual Output**: 
- Expected output: (empty - suggesting the test file likely contains an error that should be detected)
- Actual output: (empty - but this might be because the program didn't run at all)

**Key Issue**: Looking at the code, when an illegal '&' or '|' is detected, the program:
1. Sets `lexType = ILLEGAL`
2. Calls `reportError()` which writes to `error.txt`
3. BUT then continues to write to `lexer.txt` in the main loop

The problem is that after detecting an error, the program should stop processing and only output to `error.txt`, not continue outputting tokens to `lexer.txt`.
