### Summary
- **Problem Description**: Implement a lexical analyzer that reads `testfile.txt`, identifies tokens, and outputs them to `lexer.txt`. If lexical errors (specifically isolated `&` or `|`) are found, output the line number and error code `a` to `error.txt`.
- **Test Case Input**: `/bin/sh: 1: javac: not found` (This indicates the grading environment failed to compile the Java source code).
- **Buggy Output**: (Empty)
- **Expected Output**: (Empty)
- **Error Analysis**: The empty outputs and shell error message confirm a **compilation failure**. The primary cause is the use of `StringBuilder.isEmpty()` in `TokenList.java`, which was only introduced in Java 15. Most university grading environments run Java 8 or 11, causing a compilation error. Additionally, `TokenLexer.java` contains critical logical/runtime errors where `input.charAt(location)` is accessed without verifying `location < input.length()`, which will trigger `StringIndexOutOfBoundsException` during execution.
