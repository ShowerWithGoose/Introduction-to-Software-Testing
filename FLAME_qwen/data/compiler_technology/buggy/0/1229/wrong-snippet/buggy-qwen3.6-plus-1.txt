**Summary of the Issue:**
The program crashes with a `StringIndexOutOfBoundsException` when processing single `&` or `|` symbols (error type `a`), particularly when they appear at the end of a line or the input file. In `TokenLexer.java`, after detecting `&` or `|`, the code increments `location` and immediately calls `input.charAt(location)` to check for a second matching character. It lacks a boundary check to ensure `location` is still within the valid range of the `input` string before accessing it. This same off-by-one/bounds-checking flaw also exists in the handling of `!`, `<`, `>`, and `=`.
