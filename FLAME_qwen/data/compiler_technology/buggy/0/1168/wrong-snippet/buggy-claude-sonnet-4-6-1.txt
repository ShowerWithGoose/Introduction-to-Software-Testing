## Summary

**Problem:** Lexical analyzer that tokenizes source code and detects errors (specifically lone `&` or `|` characters, error code `a`).

**Failure:** The program fails to compile — `javac: not found` — meaning the Java compiler isn't available in the test environment. However, looking past the environment issue, there's also a logical bug in the error type string.

**Key observation:** The error is recorded with a leading space in the error type string `" a"` instead of just `"a"`, which would produce incorrect output format like `1  a` instead of `1 a`.

## Likely Error Location

<frontend/Lexer.java>

            } else if (matcher.group("ILLEGAL") != null) {

The error type string `" a"` has a leading space. Since `printErrors()` already formats the output as `lineNumber + " " + errorType`, the resulting output would be `1  a` (with two spaces) instead of the expected `1 a`. It should be `"a"`.
