### Summary of Information & Identified Issues
- **Problem Requirement**: The program must read `testfile.txt`, perform lexical analysis, and output tokens to `lexer.txt` (if correct) or errors to `error.txt` (if erroneous). Numerical values must be preserved exactly as they appear in the source.
- **Observed Behavior**: The program produces empty output. The test log shows `/bin/sh: 1: javac: not found`, indicating the code either failed to compile/run or executed without performing any I/O.
- **Primary Error (`Compiler.java`)**: The entire core execution logic (creating `Input`/`Lexer` instances, checking for errors, and writing to output files) is **commented out**. This results in the program doing absolutely nothing. Additionally, the error branch incorrectly writes to `lexer.txt` instead of `error.txt`.
- **Secondary Error (`Lexer.java`)**: Integer constants are parsed by converting characters to a `long` (`curNum`) and then back to a string using `String.valueOf(curNum)`. This **strips leading zeros** (e.g., `0012` becomes `12`), directly violating the requirement to preserve the exact string representation of numerical values.
