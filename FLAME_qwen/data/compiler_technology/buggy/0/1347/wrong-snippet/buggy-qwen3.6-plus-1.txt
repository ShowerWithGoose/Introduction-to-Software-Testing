### Summary
- **Problem Description**: Implement a lexical analyzer that reads `testfile.txt`, identifies tokens, and outputs them to `lexer.txt`. If lexical errors occur (specifically single `&` or `|`), output the line number and error code `a` to `error.txt`.
- **Test Case Input**: `/bin/sh: 1: javac: not found` (Indicates a compilation/environment failure in the grading system, but static analysis of the source reveals logical/runtime bugs).
- **Buggy/Expected Output**: Not generated due to the compilation failure.
- **Error Location**: The primary logical errors are in `<frontend/Lexer.java>`. 
  1. `unGetChar()` is called unconditionally after character-reading loops in `identLexer` and `digitConst`. When the end of the file is reached, `position` becomes `0`, and `unGetChar()` decrements it to `-1`. The subsequent call to `getChar()` will attempt `sourceCode.get(line).charAt(-1)`, causing a `StringIndexOutOfBoundsException`.
  2. In `otherReserveWord`, when a single `&` or `|` is detected, the code correctly records the error but fails to prevent token generation. It falls through to create a token with type `AERROR`, which may conflict with the requirement to only output error information for erroneous programs. Additionally, `unGetChar()` is called here without boundary checks.
