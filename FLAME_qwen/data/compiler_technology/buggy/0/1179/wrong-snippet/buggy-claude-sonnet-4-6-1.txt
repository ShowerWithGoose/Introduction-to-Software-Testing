## Summary

**Problem:** Lexical analyzer that reads `testfile.txt` and outputs either `lexer.txt` (correct programs) or `error.txt` (erroneous programs).

**Test failure:** The compilation itself fails — `gcc: not found` — meaning the code doesn't compile in the test environment. This is a build/environment issue, but looking at the source code for logical errors that would cause incorrect behavior:

**Key issue identified:** In `checkTypeA`, the function only scans up to the first `"` character (`while (line[i] != '\0' && line[i] != '"')`). This means if a `&` or `|` appears **inside a string literal**, it won't be checked — that part is fine. But more critically, if the `&` or `|` appears **after** a string literal on the same line, it will be **missed entirely** because the loop stops at the first `"`.

Additionally, the `checkTypeA` function opens `error.txt` in append mode (`"a"`) directly, bypassing `PrintError`, which means `errorCount` is incremented in `checkTypeA` but the file is also written to separately — this is inconsistent but not the main bug.

The primary logical error is in the loop termination condition in `checkTypeA`:

The condition `line[i] != '"'` causes the loop to stop scanning as soon as it hits a double-quote character. This means:

1. Any `&` or `|` appearing after a string literal on the same line will not be detected.
2. Any `&` or `|` appearing **inside** a string literal should be ignored (that intent is correct), but the current approach stops scanning entirely rather than skipping over the string and continuing.

The correct approach should track whether the scanner is inside a string (toggle on `"`) and skip characters inside strings, but **continue scanning after the closing `"`** rather than stopping at the first `"`.

```c
    while (line[i] != '\0' && line[i] != '"') {  // 只处理到遇到双引号的部分

        // 检查 '&' 是否与另一个 '&' 结合成合法的 '&&'
        if (line[i] == '&') {
            if (line[i + 1] == '&') {
                i += 2;  // 合法的 '&&'，跳过
            }
            else {
                // 单个 '&' 是非法字符，报告 A 类错误

            }
            else {

                errorCount++;
                break;
            }
        }

            // 其他字符默认合法，继续检查下一个字符
            i++;
        }
```
