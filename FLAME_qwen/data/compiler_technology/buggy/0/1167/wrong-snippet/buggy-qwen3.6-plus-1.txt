**Summary of the Issue:**
The lexical analyzer correctly detects the erroneous appearance of individual `&` or `|` symbols and calls the error reporting function. However, the `outputError` method incorrectly generates the error category code. It extracts the first character of the `ErrorType` enum name (`INVALID_CHAR` → `i`), but according to the problem specification, the required error category code for this specific lexical error is `a`. This mismatch causes the program to fail the test cases that check for the correct error output format.
