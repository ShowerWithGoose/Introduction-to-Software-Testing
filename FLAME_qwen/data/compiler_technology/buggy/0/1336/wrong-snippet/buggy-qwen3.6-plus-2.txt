**Summary of Issues:**
The lexical analyzer contains two critical logical errors in `Lexer.java` that lead to infinite loops and incorrect error handling:
1. **End-of-Line Check:** The condition `if (index == line.length())` fails to handle cases where `index` has already been incremented past the line length by `getCharNow()`. This prevents `nextLine()` from being called correctly, causing the lexer to stall or loop infinitely at the end of a line. It should be `index >= line.length()`.
2. **Single `&` / `|` Error Handling:** When a standalone `&` or `|` is detected (error type `a`), the code correctly records the error and calls `retract()`, but it lacks a `return` statement. Since the index is moved back to the erroneous character, the `while` loop will repeatedly encounter the same character, add the same error, and loop infinitely. A `return token;` is required after logging the error to safely exit the tokenization cycle for that character.
