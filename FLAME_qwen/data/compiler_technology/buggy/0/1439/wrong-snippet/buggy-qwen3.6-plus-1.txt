### Summary
**Problem Description**: Implement a lexical analyzer that reads `testfile.txt`, identifies tokens, and outputs them to `lexer.txt` for correct programs. For erroneous programs, it should detect lexical errors (specifically error type `a` for standalone `&` or `|`) and output the line number and error code to `error.txt`.
**Test Case Input**: `/bin/sh: 1: javac: not found` (Indicates a compilation environment issue in the test runner, but we focus on logical errors in the provided Java code).
**Buggy/Expected Output**: Not provided, but based on the specification, the program should conditionally output to only one file depending on error presence, and strictly follow the provided token table.
**Identified Errors**:
1. **`initReservedWords()`**: Registers `while` and `scanf` as keywords (`WHILETK`, `SCANFTK`). These are **not** present in the provided token category table. If they appear in the source, they will be incorrectly tokenized as keywords instead of identifiers or causing mismatches with expected outputs.
2. **`main()` method**: Unconditionally calls both `outputTokens("lexer.txt")` and `outputErrors("error.txt")`. The specification states that correct programs should output to `lexer.txt` and erroneous programs to `error.txt`. Writing to both files unconditionally may cause the judge to fail if it expects only one output file to be populated.
3. **Single-line comment handling**: `lineNum++` is executed unconditionally after skipping a `//` comment. If the comment ends at EOF without a trailing newline, this incorrectly increments the line counter for subsequent (or final) token tracking.
