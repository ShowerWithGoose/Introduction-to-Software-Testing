### Summary
The assignment requires implementing a lexical analyzer that reads a source file (`testfile.txt`), identifies tokens, and outputs them to `lexer.txt`, or detects lexical errors (specifically isolated `&` or `|` symbols) and outputs them to `error.txt`. The provided test case input `/bin/sh: 1: javac: not found` indicates an environment configuration issue in the grading system, but a logical flaw exists in the source code itself. 

The primary error is located in the **single-line comment handling logic**. After skipping characters until the end of the line, the code unconditionally executes `lineNum++`. However, the main `while` loop will subsequently read the newline character `\n` and increment `lineNum` again in the whitespace handling block. This causes the line number counter to be off-by-one for all tokens and errors that appear after any single-line comment. Additionally, the `switch` statement for operators lacks a `case '/':` to properly handle a division operator if it appears at the very end of the file, though this is a secondary edge-case issue.
