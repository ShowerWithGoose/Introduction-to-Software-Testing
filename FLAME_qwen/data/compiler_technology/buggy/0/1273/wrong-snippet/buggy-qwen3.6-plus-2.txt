### Summary
- **Problem Description**: Implement a lexical analyzer that outputs tokens for correct programs or error line numbers/codes for erroneous programs.
- **Test Case Input**: `/bin/sh: 1: g++: not found` indicates that the compilation step failed, meaning the provided C++ code contains a syntax error that prevents `g++` from building it.
- **Buggy/Expected Output**: Empty due to compilation failure.
- **Error Analysis**: The primary cause of the compilation failure is a syntax error in `<token.h>`. The `TOKEN_TYPE` macro definition ends with a trailing backslash `\` on the last line. In C++, a backslash at the end of a line indicates line continuation, but placing it on the final line of a macro definition is invalid and causes a compilation error. Additionally, there is a logical bug in `<main.cpp>` where `ch` is incorrectly used instead of `next_char` in the single-line comment skipping loop, but this does not prevent compilation.
