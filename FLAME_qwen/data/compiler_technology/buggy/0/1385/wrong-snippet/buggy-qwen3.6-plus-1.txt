**Summary of the Issue:**
The primary logical error is located in `Lexer.java` within the error handling blocks for the `|` and `&` operators. The loop index `i` is 0-based, but the problem specification explicitly states that line numbers must start from 1. The code incorrectly assigns `errorRow = i;`, which causes:
1. Errors on line 1 to be recorded as line 0.
2. The `hasError()` method to return `false` for line 1 errors (since it checks `errorRow != 0`), causing the program to incorrectly attempt to output tokens instead of the error message.
The fix requires changing `errorRow = i;` to `errorRow = i + 1;` in both locations.
