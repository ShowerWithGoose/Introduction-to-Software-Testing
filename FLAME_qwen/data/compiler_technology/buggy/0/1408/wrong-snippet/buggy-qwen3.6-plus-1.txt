**Summary of Issues:**
1. **Missing Bounds Checks:** Throughout the `deal()` method, `curPos` is incremented and immediately used to access `source.charAt(curPos)` without verifying `curPos < source.length()`. This will cause a `StringIndexOutOfBoundsException` when the source code ends with certain characters (e.g., operators, comments, or string/char literals).
2. **Incorrect Token Type for Char Constants:** Character constants enclosed in single quotes (`'...'`) are incorrectly assigned `Token.lexToken.STRCON` instead of `Token.lexToken.CHRCON` as specified in the problem description.
3. **Unsafe Comment Parsing:** The `/* ... */` comment handling accesses `source.charAt(curPos + 1)` without checking if `curPos + 1` is within bounds, risking an exception at the end of the file.
