**Summary of Issues:**
1. **Critical Logical Error in `TokenType.getBySymbol()`**: The method iterates over all enum constants and calls `tokenType.getSymbol().equals(symbol)`. For token types like `IDENFR`, `INTCON`, `STRCON`, and `CHRCON`, the `symbol` field is `null`. This will immediately throw a `NullPointerException` when processing any identifier or number, causing the program to crash.
2. **Incorrect Error Handling for `&` and `|` in `Lexer.nextOp()`**: When a single `&` or `|` is encountered, the code correctly logs the lexical error type `a`, but it **unconditionally** returns a token for `&&` or `||` with the string `"&&"`/`"||"`. This corrupts the token stream. The valid token should only be returned when the second character matches; otherwise, the lexer should consume the single character and either return `null` or a token representing the actual erroneous character.
