## Summary

The test case failed with a **compilation error**: `/bin/sh: 1: g++: not found`. This means the build environment doesn't have `g++` installed, so the code never compiled or ran. However, looking at the source code itself for logical/semantic errors that would cause issues once compiled:

The key issue is in `include/lexer.h` — it references `Token::TokenType` and `Token` before those types are declared/included. The header includes `<string>` and `<vector>` but is missing `#include "token.h"`, which defines the `Token` struct used in the method signature `void getToken(Token& token)` and the static member `_keywords_table`.

Additionally, in `src/lexer.cpp`, the single-line comment handling has a logical error: the condition `ch == '\n' && ch != EOF` is redundant (a newline can never be EOF), and more importantly it should be `ch == '\n' || ch == EOF` to properly terminate on end-of-file.
