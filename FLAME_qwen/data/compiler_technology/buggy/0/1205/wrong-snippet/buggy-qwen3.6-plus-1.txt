### Summary
The program contains a critical logical flaw in the lexical analysis of the `&` and `|` operators within `src/lexer.cpp`. According to the problem description, a standalone `&` or `|` should trigger error type `a`. However, after detecting a single `&` or `|` and reporting the error, the code lacks a `return` statement or an `else` branch. This causes the execution to fall through, incorrectly appending the character and overwriting the `ERROR` token with `AND` or `OR`. As a result, the lexer fails to properly flag the lexical error and outputs an incorrect token type instead.
