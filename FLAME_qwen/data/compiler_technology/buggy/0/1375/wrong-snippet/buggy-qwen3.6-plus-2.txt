**Summary of the Issue:**
The lexical analyzer is required to report error line numbers starting from 1, accurately reflecting the original source file. In `readfile.java`, the `readlines()` method explicitly skips empty lines (`if(linenow.length()!=0)`). This causes the internal line counter (`now_line_num`) to become misaligned with the actual file line numbers. When an error occurs after one or more empty lines, the reported line number will be incorrect, failing the test cases. Additionally, the `Compiler.java` correctly adds `+1` to the 0-indexed line number, but the base index is already corrupted by the skipped lines.
