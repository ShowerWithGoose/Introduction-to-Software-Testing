## Summary

The test case failed with `/bin/sh: 1: javac: not found`, meaning the Java compiler isn't available in the grading environment. This is a **build/compilation environment issue** — the code never ran at all.

However, looking at the source code itself for logical/semantic errors independent of the environment issue, there is a real bug in the single-line comment handling inside `isDivi` branch. The loop `while(fileContent.charAt(j) != '\n')` has no bounds check, which will throw a `StringIndexOutOfBoundsException` if the file ends with a `//` comment that has no trailing newline.

The likely error location is:

```
frontend/Lexer.java
...
            else if(isDivi(letter)){
                if(isDivi(next_letter)){
                    //单行注释
                    int j = i + 1;
                    while(fileContent.charAt(j) != '\n'){
                        j++;
                    }
                    i = j - 1;
                }
...
```

The `while` loop lacks a bounds check (`j < fileLen`), unlike the multi-line comment loop below it. If the source file ends with a `//` comment and no trailing newline, `j` will exceed `fileLen` and cause an out-of-bounds exception.

The fix should be:
```java
while(j < fileLen && fileContent.charAt(j) != '\n'){
    j++;
}
```
