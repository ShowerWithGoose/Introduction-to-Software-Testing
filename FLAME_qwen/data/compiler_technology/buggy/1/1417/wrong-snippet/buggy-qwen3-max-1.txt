Based on the problem description and code review, the main issue appears to be in the error handling mechanism. The `Error` class uses a static `isError` flag and static `tokenErrors` queue, but each time an error is created in the lexer or parser, a new `Error` instance is created rather than using a shared instance. This means errors aren't being collected properly.

The critical error is in the `Compiler.java` file where the error handling logic doesn't properly check for errors from the parsing phase, and in multiple parser files where new `Error` instances are created instead of using a shared error handler.

The error handling is flawed because:
1. The `error` object created at the beginning is never passed to the lexer or parser, so it remains empty
2. In the output loop, there's a bug where `post.poll()` is called twice - once in the condition check and once when appending
3. Errors detected during parsing create new `Error` instances that are immediately discarded
