Based on the problem description and code review, the issue appears to be in the error handling for missing semicolons (error type 'i') and missing right parentheses (error type 'j'). The main problem is in the `Stmt()` method in Parser.java where the error line number is not being properly tracked for certain cases.

Looking at the grammar and error specifications:
- Error 'i' (missing semicolon) should report the line number of the last non-terminal before the semicolon
- Error 'j' (missing right parenthesis) should report the line number of the last non-terminal before the right parenthesis

The issue is in the `Stmt()` method where `error_line` is set but not consistently updated for all cases, particularly in the general `[Exp] ';'` case at the end.
