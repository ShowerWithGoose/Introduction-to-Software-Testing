The key issue appears to be in the `parseCompUnit()` method in the `Parser.java` file. The logic for determining when to parse declarations vs function definitions vs main function is flawed.

Looking at this section:

```java
// CompUnit → {Decl} {FuncDef} MainFuncDef
public CompUnit parseCompUnit() {
    ArrayList<Decl> decls = new ArrayList<>();
    ArrayList<FuncDef> funcDefs = new ArrayList<>();
    MainFuncDef mainFuncDef = null;
    while (tokens.hasNext()) {
        if (tokens.get().isOf(TokenType.INTTK, TokenType.CHARTK, TokenType.CONSTTK)
                && !tokens.ahead(2).isOf(TokenType.LPARENT)) {
            decls.add(parseDecl());
        } else if (tokens.ahead(1).isOf(TokenType.MAINTK)) {
            mainFuncDef = parseMainFuncDef();
        } else {
            funcDefs.add(parseFuncDef());
        }
    }
    return new CompUnit(decls, funcDefs, mainFuncDef);
}
```

The problem is with the condition `!tokens.ahead(2).isOf(TokenType.LPARENT)`. This is trying to distinguish between variable declarations and function definitions by looking ahead 2 tokens, but this logic is incorrect.

For example:
- `int x;` - tokens: [INTTK, IDENFR, SEMICN] - ahead(2) would be SEMICN, not LPARENT ✓
- `int func();` - tokens: [INTTK, IDENFR, LPARENT] - ahead(2) would be LPARENT, so it should go to funcDefs ✓

However, the issue is that `tokens.ahead(2)` might be out of bounds or the logic doesn't properly handle all cases. Also, the condition for detecting main function `tokens.ahead(1).isOf(TokenType.MAINTK)` is problematic because it assumes the current token is INTTK and the next is MAINTK, but it should check if the current token is INTTK and ahead(1) is MAINTK.

But more critically, looking at the `hasNext()` method in `TokenList.java`:

```java
public boolean hasNext() {
    return index < tokenLists.size() - 1;
}
```

This is wrong! It should be `index < tokenLists.size()` because when `index == tokenLists.size() - 1`, there's still one token left to process (the last token). The current implementation stops processing one token too early.

This would cause the parser to miss the last token, which could be critical for proper parsing.
