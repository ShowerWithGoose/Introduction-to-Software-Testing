Based on the problem description and code review, the main issue appears to be in the Parser.java file, specifically in the `parseCompUnit()` method. The current implementation has a logic error in how it determines when to stop parsing declarations and function definitions before the main function. The loop condition is not properly handling cases where there might be no declarations or function definitions before main.

    private CompUnit parseCompUnit() {
        ArrayList<Decl> decls = new ArrayList<>();
        ArrayList<FuncDef> funcDefs = new ArrayList<>();
        MainFuncDef mainFuncDef;

        while (true) {
            if (curToken.getType() == TokenType.CONSTTK) {
                decls.add(parseConstDecl());
            } else if (curToken.getType() == TokenType.INTTK) {
                if (tokens.get(curTokenIndex + 1).getType() == TokenType.MAINTK) {
                    mainFuncDef = parseMainFunc();
                    break;
                } else {
                    if (tokens.get(curTokenIndex + 2).getType() == TokenType.LPARENT) {
                        funcDefs.add(parseFuncDef());
                    } else {
                        decls.add(parseVarDecl());
                    }
                }
            } else if (curToken.getType() == TokenType.CHARTK) {
                if (tokens.get(curTokenIndex + 2).getType() == TokenType.LPARENT) {
                    funcDefs.add(parseFuncDef());
                } else {
                    decls.add(parseVarDecl());
                }
            } else if (curToken.getType() == TokenType.VOIDTK) {
                funcDefs.add(parseFuncDef());
            } else {
                // Missing else clause to handle unexpected tokens
                // This can cause infinite loop if none of the conditions match
                break;
            }
        }

        addNode("<CompUnit>");
        return new CompUnit(decls, funcDefs, mainFuncDef);
    }
