Based on the problem description and code review, the main issue appears to be in the error handling and node output logic in the Parser class. The key problems are:

1. The `addNode()` method uses `curTokenIndex` as the key, but according to the problem description, nodes should be associated with line numbers (i+1 for token at index i)
2. The error checking methods (`checkSemicn`, `checkRParent`, `checkRBreak`) don't advance the token pointer when an error occurs, which can cause infinite loops
3. The node output logic doesn't properly handle the ordering requirements specified in the problem
