Based on the problem description and code review, the main issue appears to be in the `parseCompUnit()` method in Parser.java. The current implementation uses lookahead with `movedTkEquals()` which can cause incorrect parsing decisions, particularly when determining whether the next token sequence represents a MainFuncDef or FuncDef.

The problematic logic is in how the parser decides when to stop parsing declarations/functions and start parsing the main function. The condition `movedTkEquals(1, TokenType.MAINTK)` checks if the next token after current is MAINTK, but this doesn't properly handle the grammar requirement that MainFuncDef must come after all Decl and FuncDef.

    public CompUnit parseCompUnit() {
        ArrayList<Decl> decls = new ArrayList<>();
        ArrayList<FuncDef> funcDefs = new ArrayList<>();
        MainFuncDef mainFuncDef = null;

        while (true) {
            if (curToken == null) {
                break;
            } else if (movedTkEquals(1, TokenType.MAINTK)) {
                mainFuncDef = parseMainFuncDef();
                break;
            } else if (movedTkEquals(2, TokenType.LPARENT)) {
                funcDefs.add(parseFuncDef());
            } else if (curEquals(TokenType.CONSTTK) ||
                    curEquals(TokenType.INTTK) ||
                    curEquals(TokenType.CHARTK)) {
                decls.add(parseDecl());
            } else {
                break;
            }
        }

        return new CompUnit(decls, funcDefs, mainFuncDef);
    }
