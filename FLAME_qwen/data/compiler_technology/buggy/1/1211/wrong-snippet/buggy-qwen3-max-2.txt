Based on the problem description and code review, the main issue appears to be in the `parseCompUnit` method in `Parser.java`. The current implementation uses lookahead with `movedTkEquals(1, TokenType.MAINTK)` and `movedTkEquals(2, TokenType.LPARENT)` which can cause incorrect parsing decisions and potentially miss the main function or misclassify declarations/functions.

The specific problematic logic is in how the parser determines when to stop parsing declarations and functions and when to parse the main function. The lookahead approach is fragile and doesn't properly handle all cases according to the grammar `CompUnit → {Decl} {FuncDef} MainFuncDef`.

    public CompUnit parseCompUnit() {
        ArrayList<Decl> decls = new ArrayList<>();
        ArrayList<FuncDef> funcDefs = new ArrayList<>();
        MainFuncDef mainFuncDef = null;

        while (true) {
            if (curToken == null) {
                break;
            } else if (movedTkEquals(1, TokenType.MAINTK)) {
                mainFuncDef = parseMainFuncDef();
                break;
            } else if (movedTkEquals(2, TokenType.LPARENT)) {
                funcDefs.add(parseFuncDef());
            } else if (curEquals(TokenType.CONSTTK) ||
                    curEquals(TokenType.INTTK) ||
                    curEquals(TokenType.CHARTK)) {
                decls.add(parseDecl());
            } else {
                break;
            }
        }

        return new CompUnit(decls, funcDefs, mainFuncDef);
    }
