Based on the problem description and code review, the main issue appears to be in the `CompUnit` parsing logic in the `Praser.java` file. The current implementation doesn't properly handle the grammar rule `CompUnit → {Decl} {FuncDef} MainFuncDef` because it uses a while loop that may not correctly distinguish between declarations, function definitions, and the main function definition.

The specific error is in the `CompUnit` method where it tries to identify different components but doesn't properly ensure that `MainFuncDef` comes last as required by the grammar, and the condition for detecting `FuncDef` is incorrect (checking `tokenList.get(curIndex+2)` without proper bounds checking).
