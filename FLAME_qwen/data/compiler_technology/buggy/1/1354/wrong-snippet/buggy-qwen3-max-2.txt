Based on the problem description and code review, the main issue appears to be in the token recognition for keywords in the lexer, specifically for "getchar" and "printf". Looking at the TokenType enum:

The token patterns are incorrect - "getchart" should be "getchar" and "print" should be "printf". This would cause these keywords to be recognized as regular identifiers (IDENFR) instead of their proper token types, leading to parsing errors.

The logic here first parses an AddExp and then tries to extract an LVal from it, but this approach is flawed because AddExp is a more general expression that may not necessarily be a simple LVal.
