Based on the problem description and code review, the main issue appears to be in the StmtParser class where it handles missing semicolons. According to the error type 'i' definition, "Missing semicolon. The error line number corresponds to the line where the last non-terminal before the semicolon is located." However, in several places in StmtParser, when a semicolon is missing, the error is being reported on the wrong line number.

Looking specifically at the expression statement handling in StmtParser, when a semicolon is missing after an expression, the code reports the error on `semicolonExp.getLine() - 1`, but this may not correctly identify the line of the last non-terminal before the semicolon.
