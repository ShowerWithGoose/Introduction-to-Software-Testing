Based on the problem description and code review, the main issue appears to be in the `parseCompUnit()` method in Parser.java. The logic for determining when to parse Decl vs FuncDef is flawed - it uses `peek(2) != TokenType.LPARENT` which doesn't correctly identify function definitions since function definitions start with a type keyword followed by an identifier and then LPARENT.

The current logic will incorrectly parse function definitions as declarations because it only checks if there's no LPARENT two tokens ahead, but doesn't verify that what comes before LPARENT is actually a valid function signature.

    private CompUnitNode parseCompUnit() {
        //Decl
        ArrayList<DeclNode> declNodes = new ArrayList<>();
        while (peek(2) != TokenType.LPARENT) {
            declNodes.add(parseDecl());
        }

        //FuncDef
        ArrayList<FuncDefNode> funcDefNodes = new ArrayList<>();
        while (peek(1) != TokenType.MAINTK) {
            funcDefNodes.add(parseFuncDef());
        }

        //MainFuncDef
        MainFuncDefNode mainFuncDefNode = parseMainFuncDef();

        return new CompUnitNode(declNodes, funcDefNodes, mainFuncDefNode);
    }
