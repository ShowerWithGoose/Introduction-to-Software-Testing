Based on the problem description and code review, the main issue appears to be in the `parseCompUnit()` method in the Parser.java file. The logic for determining when to parse declarations vs function definitions is flawed - it uses `peek(2) != TokenType.LPARENT` to decide when to stop parsing declarations, but this doesn't correctly handle the grammar structure.

The error is in how the parser distinguishes between Decl (which can be const declarations or variable declarations) and FuncDef (which starts with a type followed by an identifier and then '('). The current condition `peek(2) != TokenType.LPARENT` is insufficient because it doesn't properly check if the next tokens form a function definition pattern.
