### Summary
- **Problem Description**: Implement a recursive descent syntax analyzer for a given C-like grammar. It must output lexical tokens and non-terminal syntax component names for correct programs, or error line numbers and category codes for erroneous programs.
- **Test Case Input**: `/bin/sh: 1: javac: not found` (Indicates a compilation/runtime failure in the testing environment, but points to a logical flaw in the parser code).
- **Buggy Output / Expected Output**: Empty (due to compilation/execution failure).
- **Error Location**: The bug is located in the `parseInitVal()` method within `frontend/Parser.java`. The grammar specifies that the initializer list `'{' [ <Exp> { ',' <Exp> } ] '}'` allows for an empty list `{}`. However, the code unconditionally calls `expNodes.add(parseExp())` immediately after consuming `{`, without checking if the next token is `}`. This will cause a parsing failure or incorrect AST construction when encountering empty initializers like `int a = {};`. This contrasts with `parseConstInitVal()`, which correctly implements the `if (peek(0) != TokenType.RBRACE)` guard.
