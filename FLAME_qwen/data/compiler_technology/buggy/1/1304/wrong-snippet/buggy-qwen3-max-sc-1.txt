Based on the problem description and code review, the main issue appears to be in the `parseCompUnit()` method in Parser.java. The current implementation uses `peek(2) != TokenType.LPARENT` and `peek(1) != TokenType.MAINTK` to determine when to stop parsing declarations and function definitions, but this logic is flawed because it doesn't properly handle the grammar structure.

The CompUnit grammar is: `CompUnit → {Decl} {FuncDef} MainFuncDef`

The current implementation incorrectly assumes that if the next token after 2 positions isn't LPARENT, it's a Decl, and if the next token after 1 position isn't MAINTK, it's a FuncDef. This can lead to incorrect parsing decisions.

    //<Decl>:           [const] int/char <Ident>
    //<FuncDef>:        void/int/char func()
    //<MainFuncDef>:    int main()
    private CompUnitNode parseCompUnit() {
        //Decl
        ArrayList<DeclNode> declNodes = new ArrayList<>();
        while (peek(2) != TokenType.LPARENT) {
            declNodes.add(parseDecl());
        }

        //FuncDef
        ArrayList<FuncDefNode> funcDefNodes = new ArrayList<>();
        while (peek(1) != TokenType.MAINTK) {
            funcDefNodes.add(parseFuncDef());
        }

        //MainFuncDef
        MainFuncDefNode mainFuncDefNode = parseMainFuncDef();

        return new CompUnitNode(declNodes, funcDefNodes, mainFuncDefNode);
    }
