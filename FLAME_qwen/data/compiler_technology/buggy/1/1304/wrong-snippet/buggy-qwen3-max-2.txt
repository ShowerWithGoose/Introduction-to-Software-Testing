Based on the problem description and code review, the main issue appears to be in the `parseCompUnit()` method in Parser.java. The current implementation uses `peek(2) != TokenType.LPARENT` to determine when to stop parsing declarations, but this logic is flawed because it doesn't properly distinguish between declarations and function definitions. A declaration doesn't necessarily have a '(' token at position +2, and this condition could incorrectly terminate the declaration parsing too early or too late.

    private CompUnitNode parseCompUnit() {
        //Decl
        ArrayList<DeclNode> declNodes = new ArrayList<>();
        while (peek(2) != TokenType.LPARENT) {
            declNodes.add(parseDecl());
        }

        //FuncDef
        ArrayList<FuncDefNode> funcDefNodes = new ArrayList<>();
        while (peek(1) != TokenType.MAINTK) {
            funcDefNodes.add(parseFuncDef());
        }

        //MainFuncDef
        MainFuncDefNode mainFuncDefNode = parseMainFuncDef();

        return new CompUnitNode(declNodes, funcDefNodes, mainFuncDefNode);
    }
