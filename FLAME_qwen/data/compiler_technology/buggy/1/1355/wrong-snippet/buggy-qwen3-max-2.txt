Based on the problem description and code review, the main issue appears to be in the CompUnit parsing logic in Parser.java. The current implementation doesn't properly handle the grammar rule `CompUnit → {Decl} {FuncDef} MainFuncDef` because it processes declarations and function definitions in an interleaved manner rather than processing all declarations first, then all function definitions, and finally the main function definition.

The error is in the `CompUnit_func()` method where it tries to parse declarations and function definitions in a single loop without enforcing the correct order specified by the grammar.
