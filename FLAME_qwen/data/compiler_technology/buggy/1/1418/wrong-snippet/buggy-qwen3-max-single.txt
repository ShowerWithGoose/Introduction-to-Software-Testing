Based on the problem description and code review, the issue appears to be in the token output handling for syntax analysis. The key problem is in the `ParserSuccessPrint` class where it's not properly outputting syntax component names before their contents as required by the specification.

The specification states: "Before completing the analysis of any syntax components in the grammar (except for <BlockItem>, <Decl>, <BType>), output the name of the current syntax component on a new line"

However, the current implementation in `ParserSuccessPrint` outputs the children first and then the syntax component name, which is backwards from what's required.

<util/ParserSuccessPrint.java>
package util;

import model.SyntaxTreeNode;

public class ParserSuccessPrint {
    public void run(SyntaxTreeNode root, StringBuilder sb, ParserSuccessPrint psp) {
        if(root.getName().equals("leaf")) {
            sb.append(root.getToken().getType()).append(" ").append(root.getToken().getName()).append("\n");
        }
        else {
            for(SyntaxTreeNode child: root.getChildren()) {
                psp.run(child, sb, psp);
            }
            if(!root.getName().equals("<BlockItem>") && !root.getName().equals("<Decl>") && !root.getName().equals("<BType>")) {
                sb.append(root.getName()).append("\n");
            }
        }
    }
}
