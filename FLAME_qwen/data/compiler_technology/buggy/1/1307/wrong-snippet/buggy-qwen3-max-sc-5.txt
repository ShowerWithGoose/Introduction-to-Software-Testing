Based on the problem description and code review, the main issue appears to be in the error handling logic for missing semicolons and right parentheses/brackets. The error reporting is not properly implemented in several parser components.

The most critical error location is in the `Compiler.java` file where the output logic incorrectly writes both error and parser output simultaneously, and in the parser components where error detection is incomplete.

<Compiler.java>
...
        String output = "symbol.txt";
        String s = SymbolList.sharedSymbolList.toString();//compUnitParser.parseCompUnit().toString();
        if (!ErrorList.isEmpty() && Debug.status){
            output = "error.txt";
            s = ErrorList.errorOutput();
        }
        try {
            OutputStream outputStream = new FileOutputStream(output);
            OutputStream outputStream1 = new FileOutputStream("parser.txt");
            try {
                outputStream.write(s.getBytes());
                outputStream1.write(compUnit.toString().getBytes());
            } catch (IOException e) {
                throw new RuntimeException(e);
            }
        } catch (FileNotFoundException e) {
            throw new RuntimeException(e);
        }
...

The program should only output to either error.txt OR parser.txt, not both. When there are errors, it should only write to error.txt and not generate parser output.

Additionally, several parser components have incomplete error handling. For example:

<parser_part/declaration/constance/ConstDefParser.java>
...
        assign = token;
        if (assign.getType() != Token.TokenType.ASSIGN) {
            System.err.println("expect assign in parseConstDef");
        }
...

This should add an error to ErrorList instead of just printing to stderr.

Similar issues exist in multiple parser files where error conditions are detected but not properly reported through the ErrorList mechanism.
