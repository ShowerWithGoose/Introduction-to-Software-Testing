Based on the problem description and code review, the main issue appears to be in the `parseCompUnit()` method in Parser.java. The current implementation has a logic error in how it determines which parsing function to call, particularly for distinguishing between function definitions and variable declarations. The condition `tokens.preRead(2).getType() == TokenType.LPARENT` is incorrect because it doesn't properly check if the current token is an identifier followed by '(', which would indicate a function definition.

    public Node parseCompUnit() {
        ArrayList<Node> children = new ArrayList<>();
        Node node = null;
        while (true) {
            if (curToken == null) {
                break;
            }
            /* root node*/
            if (tokens.preRead(1).getType() == TokenType.MAINTK) {
                node = parseMainFunDef();
            } else if (tokens.preRead(2).getType() == TokenType.LPARENT) {
                node = parseFuncDef();
            } else if (curToken.getType() == TokenType.CONSTTK) {
                node = parseConstDecl();
            } else if (curToken.getType() == TokenType.INTTK || curToken.getType() == TokenType.CHARTK) {
                node = parseVarDecl();
            } else {
                break;
            }
        }
        children.add(node);
        return new Node(children, SyntaxType.CompUnit);
    }
