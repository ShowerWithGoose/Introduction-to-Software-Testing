Based on the problem description and code review, the main issue appears to be in the `parseCompUnit()` method in Parser.java. The current implementation doesn't properly handle multiple declarations and function definitions before the main function, and has incorrect logic for determining when to parse which component. Specifically:

1. The loop condition doesn't correctly advance through tokens
2. The node assignment overwrites previous nodes instead of collecting all children
3. The logic for distinguishing between FuncDef and VarDecl is flawed

    public Node parseCompUnit() {
        ArrayList<Node> children = new ArrayList<>();
        Node node = null;
        while (true) {
            if (curToken == null) {
                break;
            }
            /* root node*/
            if (tokens.preRead(1).getType() == TokenType.MAINTK) {
                node = parseMainFunDef();
            } else if (tokens.preRead(2).getType() == TokenType.LPARENT) {
                node = parseFuncDef();
            } else if (curToken.getType() == TokenType.CONSTTK) {
                node = parseConstDecl();
            } else if (curToken.getType() == TokenType.INTTK || curToken.getType() == TokenType.CHARTK) {
                node = parseVarDecl();
            } else {
                break;
            }
        }
        children.add(node);
        return new Node(children, SyntaxType.CompUnit);
    }
