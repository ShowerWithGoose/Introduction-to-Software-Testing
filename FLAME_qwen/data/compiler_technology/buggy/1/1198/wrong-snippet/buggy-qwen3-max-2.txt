Based on the problem description and code review, I can see a clear error in the Lexer.java file. In the initialization of the `singles` HashMap, there's a mistake in the mapping for the ">=" token.

Looking at this line:
```java
singles.put(">=", new Single("==", "EQL"));
```

This is incorrect because it maps the string ">=" to a Single object with name "==" and type "EQL". This should instead be mapping ">=" to a Single object with name ">=" and type "GEQ".

The correct line should be:
```java
singles.put(">=", new Single(">=", "GEQ"));
```
