**Summary of Errors:**
1. **`parseInitVal` in `Parser.java`**: The grammar specifies `InitVal → ... | '{' [ Exp { ',' Exp } ] '}' | ...`, meaning the `Exp` inside the braces is **optional**. The current code unconditionally calls `parseExp()` after encountering `{`, which will cause parsing failures or crashes for empty initializer lists like `int a = {};`.
2. **`parseVarDecl` in `Parser.java`**: The method fails to call `root.setEndLine(...)`. This leaves the `endLine` field as `0`, which will cause incorrect error line reporting (e.g., for missing return statements in functions) and break tree traversal logic that relies on line numbers.
3. **`parseCompUnit` in `Parser.java`**: The method calls `peekOffset(1).getLexType()` and `peekOffset(2).getLexType()` without checking if `peekOffset()` returns `null`. When the parser reaches the end of the token list, this will throw a `NullPointerException`. Similar issues exist in `parseStmt` and `parseUnaryExp`.
