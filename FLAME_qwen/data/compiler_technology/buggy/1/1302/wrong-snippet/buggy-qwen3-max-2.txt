Based on the problem description and code review, the main issue appears to be in the `parseCompUnit` method in `Parser.java`. The current implementation uses lookahead tokens to determine whether to parse a MainFuncDef or FuncDef, but this approach is flawed because it doesn't properly handle the case where there are multiple declarations before the main function. The grammar specifies `CompUnit → {Decl} {FuncDef} MainFuncDef`, meaning all declarations come first, then function definitions, then the main function.

The error is in the logic that tries to distinguish between different types of declarations/definitions by looking ahead 1-2 tokens, which can fail in various edge cases.

        while (curToken != null) {
            Node child = null;
            if (peekOffset(1).getLexType() == LexType.MAINTK) {
                child = parseMainFuncDef();
            } else if(peekOffset(2).getLexType() == LexType.LPARENT) {
                child = parseFuncDef();
            } else if (curToken.getLexType() == LexType.CONSTTK) {
                child = parseConstDecl();
            } else if (isBType(curToken.getLexType())) {
                child = parseVarDecl();
            } else {
                System.out.println("Error: parseCompUnit error");
            }

            if (child != null) {
                root.addChild(child);
            }
        }
        root.setEndLine(peekOffset(-1).getLineNum());
        return root;
