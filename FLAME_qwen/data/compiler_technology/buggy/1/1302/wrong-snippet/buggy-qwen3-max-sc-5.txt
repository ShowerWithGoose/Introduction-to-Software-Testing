Based on the problem description and code review, the main issue appears to be in the `parseCompUnit()` method in `Parser.java`. The method incorrectly determines whether to parse a `MainFuncDef` or `FuncDef` by looking ahead at tokens without properly handling the case where a function definition might start with a type keyword followed by an identifier and then '('. This can lead to misclassification of declarations vs function definitions.

    public Node parseCompUnit() {
        Node root = new CompUnit(); root.setBeginLine(curToken.getLineNum());
        while (curToken != null) {
            Node child = null;
            if (peekOffset(1).getLexType() == LexType.MAINTK) {
                child = parseMainFuncDef();
            } else if(peekOffset(2).getLexType() == LexType.LPARENT) {
                child = parseFuncDef();
            } else if (curToken.getLexType() == LexType.CONSTTK) {
                child = parseConstDecl();
            } else if (isBType(curToken.getLexType())) {
                child = parseVarDecl();
            } else {
                System.out.println("Error: parseCompUnit error");
            }

            if (child != null) {
                root.addChild(child);
            }
        }
        root.setEndLine(peekOffset(-1).getLineNum());
        return root;
    }
