Based on the problem description and code review, the main issue appears to be in the `parseCompUnit()` method in `Parser.java`. The current implementation uses lookahead with `peekOffset()` to determine whether to parse a `MainFuncDef` or `FuncDef`, but this logic is flawed because:

1. It checks `peekOffset(1).getLexType() == LexType.MAINTK` to identify main function, but this assumes the current token is 'int', which may not always be the case
2. It checks `peekOffset(2).getLexType() == LexType.LPARENT` to identify function definitions, but this doesn't properly handle all cases
3. The parsing loop doesn't properly handle the end condition when `curToken` becomes null

This can cause incorrect parsing decisions and potential null pointer exceptions.
