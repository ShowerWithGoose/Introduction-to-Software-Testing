Based on the problem description and code review, the issue appears to be in the error handling for missing semicolons, right parentheses, and right square brackets. The error reporting functions `checkSEMICN()`, `checkRPARENT()`, and `checkRBRACK()` in the Parser.java file don't properly handle the case when the expected token is missing - they add an error but don't recover properly, which can cause cascading errors or incorrect parsing.

Looking specifically at the `getPrevTokenLineNum()` method, which is used to determine the line number for error reporting, there's a potential issue with how it calculates the previous token's line number.
