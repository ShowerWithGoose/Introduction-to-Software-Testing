Based on the problem description and code review, the issue appears to be in the error handling for missing semicolons, right parentheses, and right square brackets. The error reporting functions `checkSEMICN()`, `checkRPARENT()`, and `checkRBRACK()` in the Parser.java file don't properly handle the case when the expected token is missing - they add an error but don't consume any token, which can cause parsing to get stuck or produce incorrect output.

    public void checkSEMICN() {
        if (peek().getType() != Token.TokenType.SEMICN) {
            addError(getPrevTokenLineNum(), "i");
        } else {
            next();
        }
    }

    public void checkRPARENT() {
        if (peek().getType() != Token.TokenType.RPARENT) {
            addError(getPrevTokenLineNum(), "j");
        } else {
            next();
        }
    }

    public void checkRBRACK() {
        if (peek().getType() != Token.TokenType.RBRACK) {
            addError(getPrevTokenLineNum(), "k");
