### Summary of the Issue
The provided code implements a recursive descent parser and lexer for a C-like language. While the structure follows the grammar, there are critical **logical and runtime safety issues** that will cause the program to crash (e.g., `NullPointerException`, `IndexOutOfBoundsException`) or behave incorrectly during parsing:
1. **Missing EOF/Null Checks**: Multiple methods in `Parser.java` (e.g., `match`, `parseCompUnit`, `parseBlock`, `parseStmt`) directly call `.getType()` or `.getLineNumber()` on `getTokenByOff(offset)`. When the parser reaches the end of the file, `getTokenByOff()` returns `null`, causing immediate crashes.
2. **Unsafe Error Reporting in `match`**: Using `getTokenByOff(-1)` to report missing symbols will throw an `IndexOutOfBoundsException` if the parser hasn't consumed any tokens yet (`cur == 0`), as `notEnd(-1)` incorrectly evaluates to `true` and `tokens.get(-1)` fails.
3. **Infinite Loop Risk in `parseStmt`**: The lookahead loop `while(getTokenByOff(i).getLineNumber() == thatLine)` lacks a null check. If an assignment is expected but EOF is reached, it will crash.
4. **Flawed Comment Handling**: `handleMultiLineComment` in `Lexer.java` has a logic bug that can skip valid characters or enter an infinite loop if `*/` is malformed or missing.
