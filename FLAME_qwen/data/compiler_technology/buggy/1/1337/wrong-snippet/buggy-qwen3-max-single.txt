Based on the problem description and code review, the key issue appears to be in the error reporting mechanism. The `Util::printErrorInfo` function incorrectly uses `Token::tokenIndex - 1` to get the line number, which doesn't correspond to the actual token causing the error. This would cause incorrect line numbers in error reports.

    std::ofstream errorFile("error.txt", std::ios::app);
    errorFile << Token::tokenList[Token::tokenIndex - 1].lineNum << " " << error << std::endl;
    errorFile.close();
