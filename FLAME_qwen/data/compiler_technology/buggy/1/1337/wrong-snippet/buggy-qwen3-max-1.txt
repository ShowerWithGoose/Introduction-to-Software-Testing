Based on the problem description and code review, the main issue appears to be in the error handling for missing semicolons (error type 'i'). The problem is that when an error is detected, the code reports the current token's line number instead of the line number of the last non-terminal before the semicolon as required by the specification.

Looking at the code, this pattern appears in multiple places, but most notably in:

<parser/declaration/const/ConstDecl.cpp>
...
    if(token.name == ";"){
        token = Token::getNextToken();
    }
    else{
        Util::printErrorInfo(token.lineNum, "i");
    }
...

<parser/declaration/variable/VarDecl.cpp>
...
    if (token.name == ";") {
        token = Token::getNextToken();
    } else {
        Util::printErrorInfo(token.lineNum, "i");
    }
...

<parser/statement/Stmt.cpp>
...
        if (token.name == ";") {
            token = Token::getNextToken();
        } else {
            Util::printErrorInfo(token.lineNum, "i");
        }
...

The issue is that `Util::printErrorInfo` uses the current token's line number, but according to the specification for error type 'i', it should use "the line where the last non-terminal before the semicolon is located." The current implementation passes `token.lineNum` which is the line number of the current token (which might be on a different line), rather than tracking the line number of the last meaningful non-terminal component.
