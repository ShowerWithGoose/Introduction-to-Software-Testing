Based on the problem description and code review, the main issue appears to be in the `parseCompUnit` method in `Parser.java`. The method incorrectly assumes that the last function definition is always the main function, which violates the grammar specification that requires exactly one `MainFuncDef` at the end. This can cause errors when processing valid programs with multiple function definitions.

    //CompUnit → {Decl} {FuncDef} MainFuncDef
    public Ast.CompUnit parseCompUnit() {
        ArrayList<Ast.Decl> decls = new ArrayList<>();
        ArrayList<Ast.FuncDef> funcDefs = new ArrayList<>();
        while (tokens.hasNext()
                && tokens.isType(TokenType.CONSTTK, TokenType.INTTK, TokenType.CHARTK)
                && tokens.ahead(2).getType() != TokenType.LPARENT) {
            decls.add(parseDecl());
        }
        while (tokens.hasNext()
                && tokens.isType(TokenType.VOIDTK, TokenType.INTTK, TokenType.CHARTK)) {
            funcDefs.add(parseFuncDef());
        }
        Ast.FuncDef mainFuncDef = funcDefs.remove(funcDefs.size() - 1);
        return new Ast.CompUnit(decls, funcDefs, mainFuncDef);
    }
