Based on the problem description and code review, the main issue appears to be in the `parseCompUnit` method in `Parser.java`. The logic for distinguishing between declarations and function definitions is flawed - it checks `getNextToken(2)` which may go out of bounds and doesn't properly handle the case where there are no declarations before the main function.

<frontend/Parser.java>
...
    // CompUnit → {Decl} {FuncDef} MainFuncDef
    public CompUnit parseCompUnit() throws IOException {
        CompUnit compUnit = new CompUnit();
        while (getNextToken(1).getType() != TokenType.MAINTK) {
            if (getNextToken(2).getType() == TokenType.LPARENT) {
                compUnit.addFuncDef(parseFuncDef());
            } else {
                compUnit.addDecl(parseDecl());
            }
        }
        compUnit.setMainFuncDef(parseMainFuncDef());
        if (isOutput) {
//        System.out.println("<CompUnit>");
            parserWriter.write("<CompUnit>\n");
        }
        return compUnit;
    }
