Based on the problem description and code review, the main issue appears to be in the error handling implementation in the Parser class. The error handling methods are creating new Error instances and adding them to a static list incorrectly, plus the Lexer's error handling for single '&' or '|' tokens has a logic flaw where it both reports an error AND adds the invalid token to the token stream.

The most critical error is in the Lexer's handling of single '&' or '|' characters (error type 'a'), and in the Parser's error handling methods which don't properly manage the static error list.
