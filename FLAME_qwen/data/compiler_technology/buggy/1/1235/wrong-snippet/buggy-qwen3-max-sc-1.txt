Based on the problem description and code review, the main issue appears to be in the error handling mechanism, specifically in how errors are collected and stored. The `Error` class uses a static list `errors`, but each time an error is created, it's added to this static list through an instance method `addErrors()`. However, in the lexer and parser, new `Error` instances are created and their `addErrors()` method is called, which adds the error to the static list correctly. 

But looking more carefully at the `Lexer.java` file, when handling single '&' or '|' characters (error type 'a'), there's a bug in the error reporting logic:
