Based on the problem description and code review, the main issue appears to be in the `StmtType()` method in Parser.java. This method is responsible for determining the type of statement during parsing, but it has several critical flaws:

1. It consumes tokens during lookahead without properly restoring the parser state in all cases
2. The logic for handling array assignments is flawed - it parses an expression during lookahead which advances the token index
3. The error handling for missing semicolons and parentheses may reference incorrect line numbers

The most problematic section is in the array assignment handling where it temporarily modifies the parser state but doesn't always restore it correctly.
