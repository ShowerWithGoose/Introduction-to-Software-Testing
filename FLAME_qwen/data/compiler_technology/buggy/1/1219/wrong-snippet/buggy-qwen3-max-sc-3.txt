Based on the problem description and code review, the key issue appears to be in the error handling logic for missing semicolons (error code 'i'). The `errorCheckAndOutput` method doesn't properly handle cases where the expected token is missing, particularly for semicolons after statements. This would cause the parser to either miss errors or report them on incorrect line numbers.

The most critical error location is in the `Stmt()` method where it handles expression statements and assignment statements, and in the `errorCheckAndOutput` helper method which determines error line numbers.

    private void errorCheckAndOutput(Token.TokenType checkType, char errorCode) {
        if (curSym.tokenType == checkType) {
            TerminalSym();
        } else {
            errorHandler.addErrorMessage(getNextNSym(-1).lineNumber, errorCode);
