Based on the problem description and code review, the key issue appears to be in the error handling for missing semicolons (error type 'i'). The `getErrorLine` method in Parser.java incorrectly returns the line number of the last token in the nodes list, but according to the problem specification, for missing semicolon errors, "the error line number corresponds to the line where the last non-terminal before the semicolon is located."

Looking at the code, the issue is in how the error line number is calculated. The current implementation just takes the last node's line number, but it should be tracking the actual location where the semicolon is expected.
