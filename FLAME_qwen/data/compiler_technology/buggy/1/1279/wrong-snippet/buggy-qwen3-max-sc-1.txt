Based on the problem description and code review, the main issue appears to be in the error handling for missing semicolons in variable declarations. The error line number should correspond to the line where the last non-terminal before the semicolon is located, but the current implementation uses `getPreviousTokenLineNumber()` which may not correctly capture this.

<frontend/Parser.java>
...
    // 解析变量声明
    private void parseVarDecl() throws IOException {
        // VarDecl → BType VarDef { ',' VarDef } ';' // i
        parseBType();
        int currentLine = getPreviousTokenLineNumber();
        parseVarDef();
        while (currentToken.getType() == TokenType.COMMA) {
            outputToken(currentToken);
            nextToken();
            parseVarDef();
        }
        if (currentToken.getType() == TokenType.SEMICN) {
            outputToken(currentToken);
            nextToken();
        } else {
            addError(currentLine, 'i'); // 缺少分号
        }
        // 需要输出 <VarDecl>
        outputGrammar("<VarDecl>");
    }
