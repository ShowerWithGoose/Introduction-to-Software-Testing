Based on the problem description and code review, the key issue appears to be in the **error handling for missing semicolons** in statement parsing. The test case input shows a compilation error related to javac not being found, but the core issue in the parser is likely in how it handles missing semicolons in statements.

Looking at the grammar requirements and error codes:
- Error code 'i' should be reported for missing semicolons
- The error line number should correspond to the line where the last non-terminal before the semicolon is located

The main problem is in the `parseStmt()` method and its helper methods, particularly in how they handle the semicolon requirement and error reporting. Several statement parsing methods don't properly handle the case where a semicolon is missing after an expression.

    private void parseAssignmentStmt(int currentLine) throws IOException {
        parseLVal(); // 解析左值
        if (currentToken.getType() == TokenType.ASSIGN) {
            outputToken(currentToken); // 输出 '='
            nextToken();

            if (currentToken.getType() == TokenType.GETCHARTK || currentToken.getType() == TokenType.GETINTTK) {
                parseGetExp(); // 解析 getint/getchar 表达式
            } else {
                parseExp(); // 解析表达式
            }

            if (currentToken.getType() == TokenType.SEMICN) {
                outputToken(currentToken); // 输出 ';'
                nextToken();
            } else {
                addError(currentLine, 'i'); // 缺少 ';'
            }
        }
        outputGrammar("<Stmt>");
    }
