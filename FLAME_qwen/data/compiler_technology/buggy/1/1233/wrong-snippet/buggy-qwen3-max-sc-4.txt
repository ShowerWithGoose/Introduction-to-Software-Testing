Based on the problem description and code review, the main issue appears to be in the `CompileUnit.java` file. The parsing logic for declarations and function definitions doesn't properly handle the grammar rules, particularly in how it determines when to stop parsing declarations and start parsing function definitions.

The error is in the `parseDecl()` and `parseFuncDef()` methods where the lookahead logic is incorrect. According to the grammar, `CompUnit → {Decl} {FuncDef} MainFuncDef`, but the current implementation uses flawed lookahead conditions that don't correctly identify the boundaries between declarations, function definitions, and the main function.
