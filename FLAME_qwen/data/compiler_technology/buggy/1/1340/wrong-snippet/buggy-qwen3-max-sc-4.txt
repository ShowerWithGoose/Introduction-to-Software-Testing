Based on the problem description and code review, the main issue appears to be in the `analyseCompUnit` method in `SyntaxMain.java`. The condition for determining when to parse declarations vs function definitions is flawed. Specifically, the condition `!isEqualWordType(2,WordType.LPARENT)` incorrectly assumes that any token sequence without a '(' at position 2 must be a declaration, which can cause misclassification of function definitions.

    private CompUnit analyseCompUnit(){
        CompUnit compUnit = new CompUnit();
        while(isEqualWordType(0,WordType.CONSTTK)
                    ||!isEqualWordType(2,WordType.LPARENT)){
                compUnit.declList.add(analyseDecl());
        }
        while(isEqualWordType(0,WordType.VOIDTK)
                    ||(isEqualWordType(1,WordType.IDENFR)
                        && isEqualWordType(2,WordType.LPARENT))) {
                compUnit.funcDefList.add(analyseFuncDef());
        }
        if(isEqualWordType(0,WordType.INTTK)
            && isEqualWordType(1,WordType.MAINTK)
            && isEqualWordType(2,WordType.LPARENT)){
            compUnit.mainFuncDef = analyseMainFuncDef();
        }
        syntaxRes.add("<"+SyntaxType.CompUnit+">");
        return compUnit;
    }
