Based on the problem description and code review, the issue appears to be in the `analyseCompUnit` method in `SyntaxMain.java`. The condition for detecting declarations is incorrect - it should only check for `CONSTTK` or type keywords (`INTTK`/`CHARTK`) followed by identifiers, not use a negative lookahead for function definitions. The current condition `!isEqualWordType(2,WordType.LPARENT)` is flawed because it assumes all non-function patterns are declarations, which can cause misclassification.

                    ||!isEqualWordType(2,WordType.LPARENT)){

        }
        while(isEqualWordType(0,WordType.VOIDTK)
                    ||(isEqualWordType(1,WordType.IDENFR)
                        && isEqualWordType(2,WordType.LPARENT))) {
                compUnit.funcDefList.add(analyseFuncDef());
        }
        if(isEqualWordType(0,WordType.INTTK)
            && isEqualWordType(1,WordType.MAINTK)
            && isEqualWordType(2,WordType.LPARENT)){
            compUnit.mainFuncDef = analyseMainFuncDef();
        }
        syntaxRes.add("<"+SyntaxType.CompUnit+">");
        return compUnit;
