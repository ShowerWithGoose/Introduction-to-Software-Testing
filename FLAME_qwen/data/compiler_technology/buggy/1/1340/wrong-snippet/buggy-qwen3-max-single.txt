Based on the problem description and code review, the issue appears to be in the syntax analysis logic for handling missing semicolons and other error conditions. The test case input shows a compilation error related to missing javac, but the actual problem is likely in the error handling logic in SyntaxMain.java.

The key issue is in the `analyseStmt()` method where it handles different statement types. Specifically, when processing statements that should end with semicolons (like break, continue, return, etc.), the error reporting line number is taken from `wordList.get(now-1).getRow()`, but this may not correctly correspond to the line where the last non-terminal before the semicolon is located as required by the specification.
