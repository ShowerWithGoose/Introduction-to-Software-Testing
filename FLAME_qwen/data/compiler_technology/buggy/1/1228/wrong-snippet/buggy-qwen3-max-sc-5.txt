Based on the problem description and code review, the main issue appears to be in the error handling for missing semicolons (error type 'i'). The `checkI` method in `ErrorTool.java` is being called with the current token, but it should be checking if the current token is a semicolon. However, looking at the parser code, there are several places where `checkI` is called immediately after parsing a statement component, but the token hasn't been advanced to the expected semicolon yet.

The critical error is in the `parseStmt` method in `Parser.java`, specifically in the handling of assignment statements and other statement types where the semicolon check is performed before actually consuming the semicolon token.
