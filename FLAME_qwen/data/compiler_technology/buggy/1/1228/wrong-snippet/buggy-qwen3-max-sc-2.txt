Based on the problem description and code review, the key issue appears to be in the error checking logic for missing semicolons, parentheses, and brackets. The error checking methods in `ErrorTool` are being called with the wrong token value - they should check the current token *before* advancing the lexer, but the parser is calling `getToken()` before checking.

The most critical error location is in the `parseConstDef` method where it checks for missing right square bracket:
