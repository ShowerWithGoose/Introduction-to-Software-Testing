Based on the problem description and code review, the main issue appears to be in the error handling for missing semicolons and brackets. The error reporting logic incorrectly determines the line number for errors by reading an extra token after detecting the error condition.

The most problematic code is in the declaration parsers where error line numbers are determined:

<frontend/declaration/constant/ConstDeclParser.java>
...
        /* 处理i类错误：缺失; */
        this.semicn = token;
        if (!this.semicn.getType().equals(TokenType.SEMICN)) {
            // 回退两个 Token，获取分号前的非终结符行号
            this.iterator.unReadToken(2);
            Error error = new Error(this.iterator.readNextToken().getLineIndex(),
                    ErrorType.MISSING_SEMICN);
            ErrorTable.addError(error);
        }

The pattern of unreading 2 tokens and then reading one to get the line number is incorrect. It should only unread 1 token to get back to the last valid token before the expected punctuation.
