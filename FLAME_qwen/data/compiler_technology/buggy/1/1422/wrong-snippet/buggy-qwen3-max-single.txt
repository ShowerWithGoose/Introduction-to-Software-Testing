Based on the problem description and code review, the main issue appears to be in the `parse()` method initialization in `Parser.java`. The constructor reads all tokens first, but then `lastWord` is incorrectly initialized to `words.get(0)` before any parsing begins. This causes incorrect error line number reporting since `lastWord` should track the previously processed token during parsing, not be preset to the first token.

Additionally, in the `getWord()` method, the output writing uses `lastWord` which hasn't been properly initialized for the first token, potentially causing missing output or incorrect error line numbers.
