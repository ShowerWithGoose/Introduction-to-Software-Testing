Based on the problem description and code review, the main issue appears to be in the error handling for missing semicolons in declaration statements. The error reporting logic incorrectly identifies the line number for the missing semicolon error (type 'i').

Looking at the test case input showing a compilation error with "javac: not found", this suggests the program is failing during compilation, but the actual syntax analysis error handling has issues in how it reports missing semicolots.

The key problem is in the `ConstDeclParser.java` and `VarDeclParser.java` files where the error line number is determined incorrectly when reporting missing semicolons.
