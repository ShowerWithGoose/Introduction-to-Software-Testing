Based on the problem description and code review, the issue appears to be in the error handling for missing semicolons in statement parsing. The test case shows a compilation error related to javac not being found, but the core issue is likely in how the parser handles missing semicolons.

Looking at the StmtParser class, particularly in the default case and IDENFR case, there are multiple places where semicolon errors are handled. However, the most critical issue appears in the ExpStmtParser class where it handles the [Exp] ';' case.
