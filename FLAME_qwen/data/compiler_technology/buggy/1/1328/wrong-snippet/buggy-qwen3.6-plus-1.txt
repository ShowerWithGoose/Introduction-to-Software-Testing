### Summary of the Issue
The provided code implements a recursive descent parser for a C-like grammar. Although the test case input shows a shell environment error (`javac: not found`), analyzing the source code reveals a critical **logical/semantic error** in how `for` loop statements are parsed and printed. 

In `Parser.java`, the `stmt()` method correctly consumes two semicolons for a `for` loop (`for (init; cond; inc)`), but it stores both in a single `Token semicnToken` variable, causing the first semicolon to be overwritten and lost. Consequently, in `StmtNode.java`'s `print()` method, the same token (the second semicolon) is printed twice, while the first semicolon is completely missing from the output. This violates the grammar specification and will cause incorrect output for any program containing a `for` loop.

Additionally, there is a similar structural flaw in `printf` statement handling where `commaToken` is stored as a single `Token` instead of a list, which would cause incorrect comma output if multiple arguments are passed. However, the `for` loop semicolon bug is the most direct cause of parsing/output failure.
