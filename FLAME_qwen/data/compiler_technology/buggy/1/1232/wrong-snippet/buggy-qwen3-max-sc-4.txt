Based on the problem description and code review, the main issue appears to be in the lexical analysis of single '&' and '|' characters, which should generate error type 'a'. The lexer correctly detects these cases but doesn't properly propagate the error to the output files.

Looking at the Lexer.java file, when a single '&' or '|' is encountered, it sets `errorFlag = lineNum` but still returns an AND/OR token type. This causes the parser to continue processing instead of immediately reporting the error.

The critical issue is in the `outputTokens` method in Lexer.java, which only outputs the error if `errorFlag != 0`, but the main Compiler.java class never calls `lexer.outputTokens()` - it's commented out. Instead, it proceeds to parsing, where the error information is lost.
