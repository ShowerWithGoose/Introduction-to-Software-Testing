Based on the problem description and code review, the issue appears to be in the AddExp parsing logic. The grammar specifies:
AddExp → MulExp | AddExp ('+' | '−') MulExp

However, the current implementation in Parser.cpp builds the AST incorrectly by making the new MulExp the parent of the previous AddExp, when it should be the other way around.

    AddExp *Parser::parseAddExp() {
        auto first = parseMulExp();
        auto addExp = new AddExp(first);
        Token *addExpTK = nullptr;
        while (lexer->has_next() && (lexer->cur_token()->is_type(Token::TokenType::PLUS) ||
                                     lexer->cur_token()->is_type(Token::TokenType::MINU))) {
            addExpTK = lexer->next();
            auto mulExp = parseMulExp();
            addExp = new AddExp(mulExp, addExpTK, addExp);
        }
        return addExp;
    }
