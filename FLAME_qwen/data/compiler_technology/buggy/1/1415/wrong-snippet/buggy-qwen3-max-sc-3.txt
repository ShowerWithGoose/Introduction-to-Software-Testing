Based on the problem description and code review, the main issue appears to be in the lexical analysis error handling. The test case input shows "/bin/sh: 1: javac: not found" which suggests a compilation error, but the real issue is in how the lexical analyzer handles single '&' and '|' characters.

Looking at the Lexical.java file, when encountering a single '&' or '|', the code adds an error but still creates an AND/OR token instead of skipping the invalid token. This causes downstream syntax errors.
