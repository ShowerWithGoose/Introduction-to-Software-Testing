Based on the problem description and code review, the main issue appears to be in the CompUnit parsing logic in Parser.java. The current implementation incorrectly determines when to parse Decl and FuncDef by looking ahead multiple tokens, which can cause incorrect parsing decisions. Specifically:

1. In the Decl parsing loop, it checks `peekSymbol(3).getType() != SymbolType.LPARENT` which is not a reliable way to distinguish between variable declarations and function definitions
2. In the FuncDef parsing loop, it checks `peekSymbol(2).getType() != SymbolType.MAINTK` which similarly doesn't properly handle the grammar rules

These flawed lookahead checks cause the parser to either skip valid declarations/definitions or incorrectly parse them, leading to errors.
