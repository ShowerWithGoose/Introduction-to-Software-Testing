Based on the problem description and code review, the main issue appears to be in the `parseCompUnit()` method in `Parser.java`. The current implementation doesn't properly handle the case where a `MainFuncDef` should be parsed after declarations and function definitions. Specifically, the logic for detecting when to parse `MainFuncDef` is flawed - it only checks if the next token after a type is `MAINTK`, but doesn't properly consume tokens or handle the case where `MainFuncDef` comes after other declarations.

The error is in this section of the parsing logic where it tries to peek ahead but doesn't correctly advance the lexer state when making parsing decisions.

            } else if (nowType == LexType.VOIDTK) {

                    LexType prePreType = lexer.preRead();

                    } else {

                    }
                }
            }
