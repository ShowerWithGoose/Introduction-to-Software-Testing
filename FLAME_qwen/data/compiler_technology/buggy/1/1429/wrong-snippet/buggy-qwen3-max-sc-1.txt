Based on the problem description and code review, the main issue appears to be in the `parseCompUnit()` method in Parser.java. The logic for determining whether to parse a declaration or function definition is flawed - it doesn't properly handle the case where a variable declaration (starting with 'int' or 'char') should be parsed as a Decl rather than a FuncDef when it's not followed by parentheses.

The current implementation checks if the next token after a type keyword is 'main' or '(' to decide between MainFuncDef/FuncDef vs Decl, but this check is done using `nextmatch(currentIndex + 1, TokenType.MAINTK)` and `nextmatch(currentIndex + 2, TokenType.LPARENT)` which may not be reliable since the lookahead might go beyond available tokens or misidentify declarations as function definitions.
