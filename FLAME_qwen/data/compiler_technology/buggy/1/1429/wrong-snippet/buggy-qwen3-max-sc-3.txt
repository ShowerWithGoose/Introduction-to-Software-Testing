Based on the problem description and code review, the main issue appears to be in the `parseCompUnit()` method in Parser.java. The logic for determining whether to parse a declaration or function definition is flawed - it doesn't properly handle the case where a variable declaration (starting with 'int' or 'char') should be parsed as a Decl rather than a FuncDef when followed by an identifier that isn't 'main' and isn't followed by '('.

The current implementation checks if the next token after a type keyword is 'main' to decide between MainFuncDef and FuncDef/Decl, but this check is insufficient. It should first check if what follows a type keyword constitutes a variable declaration (i.e., identifier followed by comma/semicolon/array brackets/assignment) versus a function definition (identifier followed by '(').
