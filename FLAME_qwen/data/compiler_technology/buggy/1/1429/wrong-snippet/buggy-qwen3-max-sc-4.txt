Based on the problem description and code review, the main issue appears to be in the `parseCompUnit()` method in Parser.java. The logic for determining whether to parse a declaration or function definition is flawed - it uses a lookahead with `nextmatch()` but doesn't properly handle the case where a variable declaration (starting with 'int' or 'char') should be parsed as a Decl rather than a FuncDef when it's not followed by 'main' or '(' in the correct positions.

The current implementation incorrectly tries to peek at tokens without properly advancing the parser state, leading to potential misclassification of declarations vs function definitions.
