Based on the problem description and code review, the main issue appears to be in the `parseMulExpTemp` and similar temporary parsing methods where non-terminal symbols are being incorrectly output during recursive descent parsing. According to the problem requirements, only specific syntax components should be output (excluding `<BlockItem>`, `<Decl>`, `<BType>`), but the current implementation outputs non-terminals like `<MulExp>`, `<AddExp>`, etc. in the temporary parsing methods.

The error is most clearly visible in the `parseMulExpTemp` method where it incorrectly outputs `<MulExp>`:

<frontend/Parser/Parser.java>
...
    private MulExpTemp parseMulExpTemp() {
        // MulExp' → ('*' | '/' | '%') UnaryExp MulExp' | ε
        if (currentType == Token.tokenType.MULT
                || currentType == Token.tokenType.DIV
                || currentType == Token.tokenType.MOD) {
            outputList.add("<MulExp>");
            match(currentType);
            Token.tokenType tokenType = currentType;
            next();
            UnaryExp unaryExp = parseUnaryExp();
            if (unaryExp == null) {
                return null;
            }
            int pos = currentIndex;
            next();
            MulExpTemp mulExpTemp = parseMulExpTemp();
            if (mulExpTemp == null) {
                back(pos);
                return new MulExpTemp(tokenType, unaryExp);
            } else {
                return new MulExpTemp(tokenType, unaryExp, mulExpTemp);
            }
        } else {
            return null;
        }
    }
