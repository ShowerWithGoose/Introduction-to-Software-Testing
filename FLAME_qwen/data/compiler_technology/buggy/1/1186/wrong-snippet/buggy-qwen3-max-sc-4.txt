Based on the problem description and code review, the main issue appears to be in the `parseMulExpTemp` and similar parsing methods where non-terminal symbols are being incorrectly output multiple times. The error is in the recursive parsing methods that handle operator precedence.

Looking at the grammar and expected behavior, the non-terminal symbols should only be output once per production rule, but the current implementation outputs them repeatedly during recursive calls.
