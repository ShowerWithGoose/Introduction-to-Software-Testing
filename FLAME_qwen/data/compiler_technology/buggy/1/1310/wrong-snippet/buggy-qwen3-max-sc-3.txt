The `ConstDef` class incorrectly requires an assignment for all constant definitions, but looking more carefully at the grammar:
- `ConstDef → Ident [ '[' ConstExp ']' ] '=' ConstInitVal` - the '=' is mandatory for constants
- `VarDef → Ident [ '[' ConstExp ']' ] | Ident [ '[' ConstExp ']' ] '=' InitVal` - the '=' is optional for variables

Actually, the `VarDef` implementation is correct (it checks if ASSIGN is present), but the `ConstDef` should always have an assignment.
