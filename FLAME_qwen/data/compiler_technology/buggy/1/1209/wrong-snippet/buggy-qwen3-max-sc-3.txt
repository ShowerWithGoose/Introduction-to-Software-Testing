Based on the problem description and code review, the issue appears to be in the CompUnit parsing logic. The current implementation doesn't properly handle the grammar rule `CompUnit → {Decl} {FuncDef} MainFuncDef` because it uses lookahead that may not correctly distinguish between declarations and function definitions.

The main problem is in the `parse()` method of `CompUnit.java` where it tries to determine whether a token sequence represents a declaration or function definition using `tokens.get(2)` (looking ahead 2 tokens). This approach fails when there are consecutive declarations or when function definitions don't follow the expected pattern.

            funcDefNode.parse();
            funcDefNodes.add(funcDefNode);
            token2 = tokens.get(2);
        }

        this.mainFuncDefNode = new MainFuncDefNode(tokens, depth + 1);
