Based on the problem description and code review, the main issue appears to be in the `parseDecls()` method in Parser.java. The method incorrectly checks `iterator.nextToken(2)` (looking ahead 2 tokens) to decide when to stop parsing declarations, but according to the grammar, declarations should continue until we encounter something that isn't a declaration (i.e., not starting with 'const', 'int', or 'char'). The current logic breaks when it sees a '(' which is incorrect.
