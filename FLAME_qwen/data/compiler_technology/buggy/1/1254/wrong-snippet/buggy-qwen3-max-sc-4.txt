Based on the problem description and code review, the main issue appears to be in the `parseDecls()` method in `Parser.java`. The method incorrectly uses `iterator.nextToken(2)` to peek ahead, which can cause incorrect parsing decisions when determining whether to continue parsing declarations or move to function definitions. According to the grammar, `CompUnit → {Decl} {FuncDef} MainFuncDef`, declarations should be parsed as long as we encounter `const`, `int`, or `char` tokens at the current position, not by looking ahead 2 tokens.
