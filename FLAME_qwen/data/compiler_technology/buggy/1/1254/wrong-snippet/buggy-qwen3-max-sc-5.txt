Based on the problem description and code review, the main issue appears to be in the `parseDecls()` method in Parser.java. The method incorrectly checks for `LPARENT` token at position 2 to determine when to stop parsing declarations, but according to the grammar, declarations should continue until we encounter something that's not a declaration (i.e., not starting with 'const', 'int', or 'char'). The current logic will break prematurely in many valid cases.
