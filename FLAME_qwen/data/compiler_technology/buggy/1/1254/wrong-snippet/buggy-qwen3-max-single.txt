Based on the problem description and code review, I can see that the test case input shows a compilation error related to Java not being found (`/bin/sh: 1: javac: not found`), but this appears to be an environment issue rather than a code logic error.

However, looking at the parser implementation, there's a critical issue in the `parseDecls()` method in the Parser.java file. The method is trying to peek ahead by 2 tokens to determine if it should stop parsing declarations, but this logic is flawed. According to the grammar, CompUnit → {Decl} {FuncDef} MainFuncDef, and declarations should continue until we encounter something that's not a declaration (i.e., not starting with 'const', 'int', or 'char').

The current logic checks `iterator.nextToken(2)` which looks at the token 2 positions ahead, but it should be checking the current token to see if it starts a declaration.
