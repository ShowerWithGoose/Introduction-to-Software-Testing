Based on the problem description and code review, the main issue appears to be in the error handling functions `errorI`, `errorJ`, and `errorK` in the Parser.java file. These functions are supposed to detect missing semicolons, right parentheses, and right square brackets respectively, but they have incorrect logic for determining the error line number.

The problem is that when a required token is missing, these functions try to get the line number from `tokens.get(currentTokenIndex - 1)`, but if the current token doesn't match what's expected, `currentTokenIndex` hasn't been advanced, so `currentTokenIndex - 1` might not point to the correct previous token.

Looking at the `match` function, it only advances `currentTokenIndex` when there's a match. So when there's no match (i.e., when an error occurs), `currentTokenIndex` remains at the position of the unexpected token, not the previous one.
