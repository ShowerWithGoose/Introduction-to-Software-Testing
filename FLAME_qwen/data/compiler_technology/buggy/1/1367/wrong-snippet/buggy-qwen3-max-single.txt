Based on the problem description and code review, the main issue appears to be in the error handling functions in Parser.java. The error reporting functions (errorI, errorJ, errorK) are called before attempting to match the expected token, but they report errors even when the token might be present later in the stream. This causes premature error reporting.

The most problematic section is in the `parseMainFuncDef()` method where `errorJ()` is called immediately after matching "LPARENT" without first checking if "RPARENT" exists.

<Parser.java>
...
    private SyntaxNode parseMainFuncDef() {
        // 主函数定义 MainFuncDef → 'int' 'main' '(' ')' Block // 存在main函数
        SyntaxNode mainFuncDefNode = new SyntaxNode("<MainFuncDef>");
        match("INTTK", mainFuncDefNode);
        match("MAINTK", mainFuncDefNode);
        match("LPARENT", mainFuncDefNode);
        errorJ(mainFuncDefNode);//match("RPARENT");
        mainFuncDefNode.addChild(parseBlock());
        return mainFuncDefNode;
    }

Similar issues exist in other parsing methods where error checking is done unconditionally rather than conditionally based on whether the expected token is actually missing.
