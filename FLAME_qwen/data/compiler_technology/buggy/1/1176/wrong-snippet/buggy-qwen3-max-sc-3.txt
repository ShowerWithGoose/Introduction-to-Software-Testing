Based on the problem description and code review, the main issue appears to be in the **CompUnit** parsing logic in Parser.java. The current implementation doesn't properly handle the grammar rule `CompUnit → {Decl} {FuncDef} MainFuncDef` because:

1. It checks for `MainFuncDef` inside the same loop that processes Decl/FuncDef
2. The condition for detecting MainFuncDef is incorrect - it should specifically look for "int main()" pattern
3. The loop structure doesn't guarantee that MainFuncDef comes last as required by the grammar

The error is in the CompUnit method where it tries to detect MainFuncDef within the same while loop that handles declarations and function definitions.

<frontend/Parser.java>
    private void CompUnit(){
        root = new NNode("CompUnit");
        //curNode = root;
        while((isConst() || isFuncType()) && !end()){
            if(isConst()){
                Decl(root);
            } else if (isFuncType()){
                if(lexList.get(curTokenIndex + 1).lexType.equals(Lexer.LexType.IDENFR)){
                    if(lexList.get(curTokenIndex + 2).token.equals("(")){
                        FuncDef(root);
                    } else if (!curLex.lexType.equals(Lexer.LexType.VOIDTK)){
                        Decl(root);
                    }
                }
            }
            if(curLex.lexType.equals(Lexer.LexType.INTTK) && lexList.get(curTokenIndex + 1).lexType.equals(Lexer.LexType.MAINTK)) {
                MainFuncDef(root);
            }
        }
    }
