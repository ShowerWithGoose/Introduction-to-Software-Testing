Based on the problem description and code review, the main issue appears to be in the `parseCompUnit` method in `Parser.java`. The logic for determining when to parse declarations vs function definitions vs main function is flawed. Specifically, the condition `!words.get(pos + 2).getStr().equals("(")` will cause an index out of bounds error when there are no more tokens after `pos`, and the logic doesn't properly handle the grammar structure.
