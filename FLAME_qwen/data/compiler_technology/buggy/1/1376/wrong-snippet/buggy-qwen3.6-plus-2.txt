**Summary of the Issue:**
The provided code contains a logical error in the syntax analysis phase that affects the output format. In the `parseConstDecl()` method, the semicolon token (`TokenType.SEMICN`) is added to the `nodes` list **after** the `ASTNode` for `ConstDecl` is instantiated. As a result, the `ASTVisitor` will not traverse or print the semicolon for constant declarations, causing the output to miss required lexical tokens and fail test cases. The same pattern in `parseVarDecl()` correctly adds the semicolon before node creation, highlighting this as an inconsistency/bug in `parseConstDecl()`.

**Error Location Code Snippet:**
<front_end/parser/Parser.java>

    // 解析常量声明
    private ASTNode parseConstDecl()  {
        expect(TokenType.CONSTTK);
        ArrayList<Object> nodes = new ArrayList<>();
        nodes.add(TokenType.CONSTTK);
        nodes.add(parseBType());
        do {
            nodes.add(parseConstDef());
            nodes.add(TokenType.COMMA);
        } while (match(TokenType.COMMA));
        nodes.remove(nodes.size() - 1);
        ASTNode constDecl = new ASTNode(NodeType.ConstDecl, nodes);
        expect(TokenType.SEMICN);
        nodes.add(TokenType.SEMICN);
        return constDecl;
    }
