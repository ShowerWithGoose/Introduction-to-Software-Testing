Based on the problem description and code review, I can see a critical issue in the `Lexer.java` file in the `scanSymbol()` method. When encountering a single '&' or '|' character (which should be an error), the code incorrectly adds an AND/OR token with "&&"/"||" value instead of the actual single character, and then reports the error. This would cause incorrect tokenization.
