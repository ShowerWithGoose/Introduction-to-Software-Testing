The issue is in the loop conditions for parsing declarations and function definitions. The current implementation uses `tokens.get(index + 1)` and `tokens.get(index + 2)` which can cause IndexOutOfBoundsException when near the end of the token list, and more importantly, the logic for determining when to stop parsing declarations and start parsing function definitions is incorrect.

According to the grammar `CompUnit → {Decl} {FuncDef} MainFuncDef`, we should parse declarations first (which start with 'const', 'int', or 'char'), then function definitions (which start with 'void', 'int', or 'char' followed by an identifier and '('), and finally the main function.
