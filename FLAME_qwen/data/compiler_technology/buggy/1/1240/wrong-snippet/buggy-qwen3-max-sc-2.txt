Based on the problem description and code review, the issue appears to be in the error handling for missing semicolons in statement parsing. The test case shows a compilation error related to missing semicolons (error type 'i'), but the current implementation doesn't properly handle all cases where semicolons are required.

Looking at the StmtParser class, the parseAssignmentOrExpStmt method has logic to detect missing semicolons, but it only checks if the current line differs from the initial token's line. This might not catch all cases, particularly when the expression ends but no semicolon follows.

The main issue is in the Stmt constructor that handles semicolon consumption - it only consumes a semicolon if present but doesn't report an error when it's missing in cases where it should be required.
