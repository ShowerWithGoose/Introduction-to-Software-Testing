Based on the problem description and code review, the main issue appears to be in the parser's handling of syntax component output. According to the requirements, certain syntax components should NOT be output (<BlockItem>, <Decl>, <BType>), but the current implementation outputs all non-terminals.

Looking at the Node.print() method in <frontend/Node/Node.java>, we can see the special handling for certain expression types, but there's no filtering for the components that shouldn't be output.

However, the most critical error is likely in the Parser's parse_CompUnit method where it handles declarations and function definitions. The logic for distinguishing between Decl and FuncDef appears flawed.
