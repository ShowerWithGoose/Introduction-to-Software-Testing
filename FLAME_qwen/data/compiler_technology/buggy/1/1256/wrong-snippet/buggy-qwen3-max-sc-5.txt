Based on the problem description and code review, the main issue appears to be in the `CompUnit.parse()` method in `CompUnit.java`. The current implementation has a flawed lookahead mechanism that doesn't properly handle the parsing of declarations and function definitions before the main function. It incorrectly peeks at three tokens ahead and makes parsing decisions based on that, which can lead to incorrect parsing or infinite loops.
