Based on the problem description and code review, the main issue appears to be in the `handleTerminal` method in `Parser.java`. This method is responsible for handling expected terminal tokens and recording appropriate error types when they're missing. However, the current implementation has a critical flaw in how it determines the line number for error reporting.

Looking at the error types specified:
- For missing semicolon (error 'i'), the line number should correspond to "the line where the last non-terminal before the semicolon is located"
- For missing right parenthesis (error 'j'), the line number should correspond to "the line where the last non-terminal before the right parenthesis is located"  
- For missing right square bracket (error 'k'), the line number should correspond to "the line where the last preceding non-terminal symbol appears before the right square bracket"

The current implementation uses `getLineNumber(-1)` which gets the line number of the previous token, but this may not correctly correspond to the last non-terminal's location as required.

<frontend/Parser.java>
...
    // 处理终结符
    private Token handleTerminal(LexType lexType) {
        if (getSymType().equals(lexType)) {
            Token token = getNowToken();
            nextIndex();
            return token;
        }
        is_analyze_success = false;
        if (lexType.equals(LexType.SEMICN)) {
            ErrorHandler.recordError(ErrorType.SEMICN_MISSING, getLineNumber(-1));
        }
        else if (lexType.equals(LexType.RPARENT)) {
            ErrorHandler.recordError(ErrorType.RPARENT_MISSING, getLineNumber(-1));
        }
        else if (lexType.equals(LexType.RBRACK)) {
            ErrorHandler.recordError(ErrorType.RBRACK_MISSING, getLineNumber(-1));
        }
        else {
            throw new Error(ErrorType.OTHER_ERROR, getLineNumber());
        }
        return null;
    }
