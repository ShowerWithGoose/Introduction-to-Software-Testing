**Summary of Errors:**
1. **Incorrect Unary Operator Restriction in Expressions:** In `parseExp` and `parseConstExp`, the method `parseAddExp` is called with `false` as the first argument (`type`). This flag is propagated down to `judgeUnaryOpFirst`, which incorrectly disables the `!` (NOT) operator for `Exp` and `ConstExp`. According to the grammar, `UnaryOp` includes `!`, and it should be valid in all expressions.
2. **Off-by-One Output Error in `parseOpUnaryExp`:** The loop `for (int i = 0; i < ops.size() - 1; i++)` responsible for outputting `<UnaryExp>` is flawed. The grammar `UnaryExp → UnaryOp UnaryExp` implies that each unary operator creates a new `UnaryExp` node. The current condition skips outputting `<UnaryExp>` for the outermost level (or when only one operator exists), leading to missing syntax component outputs in the parser trace.
