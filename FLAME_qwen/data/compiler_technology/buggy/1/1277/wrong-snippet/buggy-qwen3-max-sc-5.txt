Based on the problem description and code review, the key issue appears to be in the `parseHead()` method in `BlockParser.java`. This method handles statement parsing for assignments and expressions, but has incorrect error handling for missing semicolons in expression statements.

The specific problem is in this section where it handles pure expression statements (not assignments):
```java
} else { // Exp(done) ; // i
    Token t = lexer.getAndNext();
    if (t.getType() == TokenType.SEMICN) {
        children.add(new SynLeaf(t));
    } else {
        lexer.move(-2);
        addError(new ErrorSyntax(lexer.get().getLine(), ErrorType.i));
        lexer.move(1);
    }
}
```

The error line number calculation is incorrect here. According to the problem specification, for missing semicolon errors (type 'i'), "the error line number corresponds to the line where the last non-terminal before the semicolon is located." However, the current implementation uses `lexer.get().getLine()` after moving back, which may not point to the correct line of the last non-terminal in the expression.
