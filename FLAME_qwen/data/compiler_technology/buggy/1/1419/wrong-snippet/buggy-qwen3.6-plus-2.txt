### Summary of Identified Errors

1. **`parseCompUnit` Unsafe Lookahead & Logic**: The method accesses `tokenList.get(idx + 1)` and `tokenList.get(idx + 2)` without checking array bounds, which will throw an `IndexOutOfBoundsException` on short inputs or near the end of the file. Additionally, the loop condition incorrectly assumes any token sequence before `main (` is a `Decl`, failing to properly distinguish between `Decl` and `FuncDef` based on the current token type.
2. **Incorrect AST Print Order for Binary Expressions**: In `AddExp`, `MulExp`, `RelExp`, `EqExp`, `LAndExp`, and `LOrExp`, the `print()` methods output the non-terminal name (`InputOutput.write(getType())`) **before** recursively printing the right-hand child. Syntax analysis output typically requires post-order traversal (children first, then the parent component name), so this causes interleaved and incorrect output formatting.
3. **Flawed Error Recovery in `expect()`**: The `expect()` method's fallback for `TokenType.AND` and `TokenType.OR` unconditionally increments `idx` and returns a synthetic token, even if the current token is completely unrelated. This skips valid tokens, desynchronizes the parser, and masks actual syntax errors.
