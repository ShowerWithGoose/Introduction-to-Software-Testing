Based on the problem description and code review, the main issue appears to be in the **`parseCompUnit()` method in Parser.java**. The logic for determining when to stop parsing declarations and function definitions before the main function is flawed. It uses `readAfterToken(1)` and `readAfterToken(2)` without proper bounds checking and doesn't correctly handle the case where there are no declarations or function definitions before main.

Additionally, there's a critical error in **`parseConstDef()`** where after successfully matching a right bracket `]`, it incorrectly skips another token with `nextToken()` instead of just consuming the `]`.
