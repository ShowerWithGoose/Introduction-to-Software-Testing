Based on the problem description and code review, the main issue appears to be in the Parser's CompUnit processing logic. The current implementation has overly complex and incorrect lookahead logic for distinguishing between Decl, FuncDef, and MainFuncDef. According to the grammar `CompUnit → {Decl} {FuncDef} MainFuncDef`, it should first parse all Decl (const/var declarations), then all FuncDef (function definitions), and finally exactly one MainFuncDef.

The error is in the CompUnit.Process method where it tries to use PreToken() for lookahead but doesn't properly handle the sequence. It should simply parse Decl while possible, then FuncDef while possible, then MainFuncDef.

    static class CompUnit
    {
        static void Process(GrammarTreeNode node) {
            while (true) {
                if (instance.IsFirstOf(instance.token.getType(), NonterminalType.Decl)) {
                    Decl.Process(node);
                }
                else {
                    break;
                }
            }
            while (true) {
                if (instance.IsFirstOf(instance.token.getType(), NonterminalType.FuncDef)) {
                    FuncDef.Process(node);
                }
                else {
                    break;
                }
            }
            MainFuncDef.Process(node);
        }
    }
