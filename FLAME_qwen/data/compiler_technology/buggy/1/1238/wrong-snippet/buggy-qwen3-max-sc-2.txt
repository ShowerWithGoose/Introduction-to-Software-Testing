Based on the problem description and code review, the key issue appears to be in the error handling logic in the Parser.java file. The `handleError` method incorrectly checks for expected tokens and adds errors even when the correct token is present. Specifically:

1. In `handleError`, when checking for semicolon (error type 'i'), it checks if the next token is NOT SEMICN, but then calls `move()` only when the token IS SEMICN. This causes the parser to not consume the semicolon when it's actually present, leading to infinite loops or incorrect parsing.

2. Similar issues exist for error types 'j' (right parenthesis) and 'k' (right bracket).
