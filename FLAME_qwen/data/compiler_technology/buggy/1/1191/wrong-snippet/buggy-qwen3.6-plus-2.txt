### Summary
The reported **Compilation Error** is highly likely caused by the use of **pattern matching for `instanceof`** (e.g., `obj instanceof Type varName`), which is a language feature introduced in **Java 16**. Many academic OJ systems and default university environments still run on Java 8 or Java 11, where this syntax is invalid and will immediately fail during the `javac` compilation phase. The problematic syntax appears multiple times in `Semantic.java`.
