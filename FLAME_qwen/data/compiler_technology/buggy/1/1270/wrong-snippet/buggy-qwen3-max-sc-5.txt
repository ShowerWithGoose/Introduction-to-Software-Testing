Based on the problem description and code review, the main issue appears to be in the error handling for missing semicolons in statement parsing. The test case input shows a compilation error related to javac not being found, but the actual program logic error is in how the parser handles missing semicolons.

Looking at the Parser.java file, in the `parseStmt()` method, when handling assignment statements (LVal '=' Exp ';'), there's a logic flaw in the error reporting line number for missing semicolons. The error should report the line number of the last non-terminal before the semicolon, but the current implementation uses `lookBehind().getLine() + 1` which may not be correct.

The most likely location of the error is in the statement parsing section where semicolon errors are handled:
