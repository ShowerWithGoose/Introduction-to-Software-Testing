Based on the problem description and code review, the key issue appears to be in the error handling for missing semicolons and parentheses. The `lexer.special()` method is used to determine the line number for errors, but it looks back through tokens to find the last non-FAKE token's line number. However, in several places in the Parser, the error is reported before the relevant tokens have been properly added to the tokens list, leading to incorrect line number reporting.

The most critical error location is in the `ConstDecl` method where a missing semicolon error is handled:
