**Summary:**
The program fails primarily due to a critical semantic error in `frontend/Parser.java`. The lookahead methods `prePreRead()` and `preRead()` in `frontend/Lexer.java` return `StringBuilder` objects. In `Parser.java`, these return values are directly compared with `String` literals using `.equals()`. Because `StringBuilder` does not override `equals()`, it falls back to `Object.equals()`, which checks reference equality. This comparison will **always evaluate to `false`**, causing the parser to misclassify function definitions (both `char` and `int` types) as declarations (`Decl()`), which breaks the recursive descent parsing flow. Additionally, there is a minor logical bug in `frontend/Lexer.java` where a single `|` error is incorrectly assigned `LexType.AND` instead of `LexType.OR`.
