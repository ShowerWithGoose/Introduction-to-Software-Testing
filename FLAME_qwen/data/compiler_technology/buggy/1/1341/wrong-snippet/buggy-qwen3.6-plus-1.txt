**Summary of Errors:**
1. **`Compiler.java`**: The condition `!GetSymTable.isError()` checks a static error map intended for semantic analysis (which is currently commented out). Syntax errors are actually stored in the `Parser` instance. This causes the program to always write to `parser.txt` even when syntax errors occur. It should check `parser.isError()` instead.
2. **`frontend/ast/Parser.java`**: In the `parse()` method, `tokens.peek(2).is(TokenType.LPARENT)` lacks a null check. When the token stream has fewer than 3 tokens or reaches the end of input, `peek(2)` returns `null`, triggering a `NullPointerException` and crashing the parser before it can finish or report errors gracefully.
