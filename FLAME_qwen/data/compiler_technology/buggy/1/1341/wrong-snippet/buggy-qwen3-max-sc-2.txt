Based on the problem description and code review, the main issue appears to be in the `parse()` method of the Parser class. The current implementation incorrectly assumes that declarations end when encountering a token followed by '(', but this doesn't properly handle the grammar structure where MainFuncDef comes after FuncDef. The parser fails to correctly identify when to stop parsing declarations and start parsing function definitions.

                compUnit.addDecl(parseConstDecl());
            } else {

        }
        while (tokens.peek() != null) {
            compUnit.addFuncDef(parseFuncDef());
