Based on the problem description and code review, the main issue appears to be in the parsing logic for the main function detection in Parser.java. The current implementation incorrectly assumes that any token sequence followed by '(' indicates a function definition, but doesn't properly handle the MainFuncDef case which has a specific structure ('int' 'main' '(' ')' Block).

The error is in the parse() method of Parser.java where it tries to distinguish between declarations and function definitions. The condition `while (!tokens.peek(2).is(TokenType.LPARENT))` is flawed because it doesn't account for the MainFuncDef which should be parsed separately.
