Based on the problem description and code review, the main issue appears to be in the `parse()` method of the `Parser` class. The current implementation incorrectly assumes that declarations end when encountering a token followed by '(', but this doesn't properly handle the grammar structure where `MainFuncDef` starts with "int main()".

The error is in how the parser separates declarations from function definitions. According to the grammar:
- `CompUnit → {Decl} {FuncDef} MainFuncDef`
- `MainFuncDef → 'int' 'main' '(' ')' Block`

The current condition `while (!tokens.peek(2).is(TokenType.LPARENT))` is flawed because it doesn't specifically check for the "main" function pattern.

    public void parse() {
        //int a
        //const int a
        //int a()
        while (!tokens.peek(2).is(TokenType.LPARENT)) {
            if (tokens.peek().is(TokenType.CONSTTK)) {
                compUnit.addDecl(parseConstDecl());
            } else {
                compUnit.addDecl(parseVarDecl());
            }
        }
        while (tokens.peek() != null) {
            compUnit.addFuncDef(parseFuncDef());
        }
    }
