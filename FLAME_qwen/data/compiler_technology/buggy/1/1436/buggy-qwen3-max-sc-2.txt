@@ [The condition `(target == ComponentEnum::Stmt && form != StmtEnum::STMT_EXP)` incorrectly excludes the STMT_EXP form from semicolon error checking. According to the grammar, even STMT_EXP (which represents an expression statement like "a = b + c") requires a semicolon, and missing semicolon should be reported as error type 'i'. The current logic only reports missing semicolons for non-STMT_EXP statements, causing it to miss semicolon errors in expression statements.]
