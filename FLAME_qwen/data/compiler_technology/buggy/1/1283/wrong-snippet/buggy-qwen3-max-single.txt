Based on the problem description and code review, the key issue appears to be in the `doCompUnit()` method in Grammar.java. The method incorrectly checks for the start of MainFuncDef by looking at `tokens.get(curPos+1)` and `tokens.get(curPos+2)`, which can cause index out of bounds errors and incorrect parsing when there are no declarations or function definitions before main.

The error is in this loop condition:
```java
while(tokens.get(curPos+1).type!=Token.Type.MAINTK&&tokens.get(curPos+2).type!=Token.Type.LPARENT)
```

This can fail when there are fewer than 2 tokens remaining, and doesn't properly handle the case where main() appears immediately.
